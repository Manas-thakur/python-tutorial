# Strings

## Definition

A **string** (`str`) is an immutable sequence of Unicode characters, enclosed in single quotes (`'...'`), double quotes (`"..."`), or triple quotes (`'''...'''` / `"""..."""`). Strings are one of Python's most used data types for representing text.

```python
s1 = 'hello'
s2 = "world"
s3 = '''multi-line
string'''
s4 = """also multi-line"""
```

## Why it exists

Strings exist because programs constantly work with text: user names, file paths, error messages, HTML/JSON data, configuration files, and more. Without a dedicated text type, handling character data would be tedious and error-prone. Python's strings are Unicode-native (Python 3), making them suitable for international text and emoji.

## How it works

### String Creation

```python
# Single quotes
s1 = 'Hello'

# Double quotes (useful when string contains single quote)
s2 = "It's a nice day"

# Triple quotes (multi-line strings)
s3 = """This is a
multi-line
string"""

# Triple single quotes also work
s4 = '''This works too'''

# Raw strings (escape sequences ignored)
s5 = r"C:\Users\name"   # Note: r"C:\Users\name" — backslashes are literal
```

**Raw strings** are useful for regex patterns and Windows file paths:

```python
normal = "C:\new\tab"   # \n and \t are escape sequences!
raw = r"C:\new\tab"     # Literal backslashes
print(normal)  # C:      ew    ab (weird!)
print(raw)     # C:\new\tab
```

### Escape Sequences

| Sequence | Meaning |
|----------|---------|
| `\n` | Newline |
| `\t` | Tab |
| `\\` | Backslash |
| `\'` | Single quote |
| `\"` | Double quote |
| `\r` | Carriage return |
| `\b` | Backspace |

### Indexing

Python uses **0-based indexing**. Negative indices count from the end.

```python
s = "Python"

# Positive indexing:   0  1  2  3  4  5
#                      P  y  t  h  o  n
# Negative indexing:  -6 -5 -4 -3 -2 -1

print(s[0])    # 'P'
print(s[2])    # 't'
print(s[-1])   # 'n'
print(s[-3])   # 'h'
```

### Slicing

Extract a substring using `[start:stop:step]`:

```python
s = "Python Programming"

# Basic slicing
print(s[0:6])        # 'Python' (indices 0 to 5)
print(s[7:18])       # 'Programming' (indices 7 to 17)

# Omitting start/stop
print(s[:6])         # 'Python' (from beginning)
print(s[7:])         # 'Programming' (to end)
print(s[:])          # 'Python Programming' (full copy)

# Step
print(s[::2])        # 'Pto rgamn' (every 2nd character)
print(s[::-1])       # 'gnimmargorP nohtyP' (reversed!)

# Negative step
print(s[5:0:-1])     # 'nohty' (reverse slice)
```

### Common String Methods

```python
s = "  Hello, World!  "

# Case modification
print(s.upper())          # '  HELLO, WORLD!  '
print(s.lower())          # '  hello, world!  '
print(s.title())          # '  Hello, World!  '
print(s.capitalize())     # '  hello, world!  '
print(s.swapcase())       # '  hELLO, wORLD!  '

# Trimming
print(s.strip())          # 'Hello, World!'
print(s.lstrip())         # 'Hello, World!  '
print(s.rstrip())         # '  Hello, World!'

# Searching
print(s.find("World"))    # 7 (index where 'World' starts)
print(s.find("Python"))   # -1 (not found)
print(s.index("World"))   # 7 (like find, but raises ValueError if not found)
print(s.count("l"))       # 3
print(s.startswith("  ")) # True
print(s.endswith("!  "))  # True

# Replacement
print(s.replace("World", "Python"))  # '  Hello, Python!  '
print(s.replace("l", "L"))           # '  HeLLo, WorLd!  '

# Splitting and Joining
csv = "apple,banana,orange"
fruits = csv.split(",")              # ['apple', 'banana', 'orange']
print(fruits)

joined = " - ".join(fruits)          # 'apple - banana - orange'
print(joined)

# Checking content
print("Hello".isalpha())      # True
print("123".isdigit())        # True
print("abc123".isalnum())     # True
print("   ".isspace())        # True
print("Hello".isupper())      # False
```

### Immutability

Strings are **immutable** — once created, they cannot be modified. Any "change" creates a new string:

```python
s = "Hello"
# s[0] = "J"  # TypeError: 'str' object does not support item assignment

# This creates a NEW string
s = s.upper()  # s now refers to "HELLO", old "Hello" is garbage-collected
```

### String Concatenation vs `.join()`

```python
# Inefficient way (creates many intermediate strings)
result = ""
for word in ["hello", "world", "python"]:
    result += word + " "   # Creates new string each iteration!

# Efficient way (join allocates once)
result = " ".join(["hello", "world", "python"])
```

**Why `.join()` is faster:** `+=` creates a new string per operation (O(n²)). `.join()` precomputes the total size and allocates once (O(n)).

## Internal working

### Memory Layout

Python strings are stored as arrays of Unicode code points (flexible representation):

- **ASCII-compatible strings** use 1 byte per character (compact)
- **Other BMP characters** use 2 bytes per character
- **Characters outside BMP** (emoji, rare scripts) use 4 bytes per character

Python optimizes internally (flexible string representation — `PyASCIIObject`, `PyCompactUnicodeObject`).

### String Interning

Python may **intern** short strings (typically identifiers and small literals) — reusing the same object for equal strings to save memory:

```python
a = "hello"
b = "hello"
print(a is b)   # Often True (interning)

# But not for strings created dynamically
a = "hello world!"
b = "hello world!"
print(a is b)   # Often False (not interned)
```

### Slicing Creates New Strings

Each slice operation creates a new string object (copying the sliced characters):

```python
s = "x" * 1000000
sub = s[0:10]  # Creates a new string of length 10
```

## Real-world analogy

A string is like a **string of beads** (hence the name):

- Each **bead** = one character
- The **order** of beads matters (indexing)
- You can **look at** any bead (indexing) or a **section** of beads (slicing)
- You cannot change a single bead without re-stringing (immutability)
- You can make a **copy** with different beads (methods return new strings)
- Raw strings = beads with no special meaning for certain symbols

## Examples (with code)

### Text analysis

```python
text = "Python is amazing and powerful!"

# Word count
words = text.split()
print(f"Word count: {len(words)}")

# Character analysis
print(f"Total characters: {len(text)}")
print(f"Uppercase count: {sum(1 for c in text if c.isupper())}")
print(f"Vowels: {sum(1 for c in text.lower() if c in 'aeiou')}")

# Find and replace
print(text.replace("amazing", "awesome"))
```

### String cleaning

```python
# Clean user input
user_input = "  UsErNaMe 123  "
clean = user_input.strip().lower().replace(" ", "_")
print(clean)  # 'username_123'

# Remove punctuation
import string
text = "Hello!!! How are you??"
clean = text.translate(str.maketrans("", "", string.punctuation))
print(clean)  # 'Hello How are you'
```

### Formatting with strings

```python
name = "Alice"
score = 87

# Padding and alignment
print(f"{name:<10} {score:>5}")    # Alice         87
print(f"{name:>10} {score:<5}")    #      Alice 87

# Truncation
long_text = "Python is great"
print(f"{long_text:.10}...")       # Python is ...

# Zero-padding numbers
print(f"{42:05d}")                 # 00042
```

## Mini Project: Palindrome Checker

```python
# palindrome.py
print("=== Palindrome Checker ===\n")

def is_palindrome(text):
    """Check if text is a palindrome (reads same forwards and backwards)."""
    # Remove non-alphanumeric characters and convert to lowercase
    cleaned = ""
    for char in text:
        if char.isalnum():
            cleaned += char.lower()
    
    return cleaned == cleaned[::-1]

# Test it
word = input("Enter a word or phrase: ")

if is_palindrome(word):
    print(f"\n✅ '{word}' IS a palindrome!")
    # Show the reversal
    cleaned = "".join(c.lower() for c in word if c.isalnum())
    print(f"   Reversed: {cleaned[::-1]}")
else:
    print(f"\n❌ '{word}' is NOT a palindrome.")
    cleaned = "".join(c.lower() for c in word if c.isalnum())
    print(f"   Forward:  {cleaned}")
    print(f"   Reversed: {cleaned[::-1]}")

# Examples
print("\n--- Try these palindromes ---")
examples = [
    "racecar",
    "A man, a plan, a canal: Panama",
    "Was it a car or a cat I saw?",
    "No 'x' in Nixon",
    "madam",
]
for ex in examples:
    result = "✅" if is_palindrome(ex) else "❌"
    print(f"  {result} {ex}")
```

**Sample run:**
```
=== Palindrome Checker ===

Enter a word or phrase: racecar

✅ 'racecar' IS a palindrome!
   Reversed: racecar
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Modifying string in place | `s[0] = "J"` | `s = "J" + s[1:]` |
| Forgetting string is immutable | `s.upper()` doesn't change `s` | Reassign: `s = s.upper()` |
| Off-by-one in slicing | `s[0:5]` for first 5 chars | `s[0:5]` includes indices 0,1,2,3,4 — correct for 5 chars |
| Using `+` in loops | `s += c` in loop (slow) | Use `"".join(list)` |
| Confusing `==` and `is` | `s is "hello"` | Use `s == "hello"` (value comparison) |
| Not handling `.find()` returning -1 | Unchecked -1 in index operation | Check `if s.find(x) != -1` |
| Mixing str and int | `"Age: " + 25` | `"Age: " + str(25)` or f-string |
| Escape sequence gotcha | `"C:\new\tab"` | `r"C:\new\tab"` or `"C:\\new\\tab"` |

## Interview questions + answers

**Q1: Are strings mutable or immutable in Python?**
A1: Immutable. Any operation that appears to modify a string creates a new string object.

**Q2: What is string slicing and how does `[::-1]` work?**
A2: Slicing extracts a substring using `[start:stop:step]`. `[::-1]` means: start at end, step backwards by 1 — this reverses the string.

**Q3: What is the difference between `.find()` and `.index()`?**
A3: Both find a substring's index. `.find()` returns `-1` if not found. `.index()` raises `ValueError` if not found.

**Q4: Why is `"".join(list)` faster than `+=` for string concatenation?**
A4: Strings are immutable, so `+=` creates a new string each iteration (O(n²) total). `.join()` allocates the final string once (O(n)).

**Q5: Explain string interning.**
A5: Python may reuse the same object for identical string literals to save memory. Short strings and identifiers are commonly interned. Use `==` for comparison, not `is`.

**Q6: What are raw strings and when would you use them?**
A6: Raw strings (prefix `r`) treat backslashes as literal characters. Used for regex patterns (`r"\d+"`) and Windows paths (`r"C:\folder"`).

**Q7: How do you reverse a string?**
A7: `s[::-1]` (extended slice with step -1).

**Q8: What is the difference between `.strip()`, `.lstrip()`, and `.rstrip()`?**
A8: `.strip()` removes whitespace from both ends. `.lstrip()` from left only. `.rstrip()` from right only.

**Q9: How do you check if a string contains only digits?**
A9: `s.isdigit()` returns `True` if all characters are digits.

**Q10: What does `split()` do by default?**
A10: Splits on any whitespace (spaces, tabs, newlines) and removes empty strings. `.split(" ")` splits only on spaces and keeps empty strings.

## Revision notes

- Strings are **immutable sequences** of Unicode characters
- Created with `'`, `"`, `'''`, `"""`, or `r""` (raw)
- **Indexing**: 0-based, negative counts from end
- **Slicing**: `[start:stop:step]`, `[::-1]` reverses
- **Methods**: `.upper()`, `.lower()`, `.strip()`, `.split()`, `.join()`, `.replace()`, `.find()`, `.count()`, `.startswith()`, `.endswith()`
- **Immutability**: all "changes" create new strings
- **Concatenation**: prefer `"".join()` over `+=` for multiple operations
- Raw strings (`r"..."`) for regex and paths
- Escape sequences: `\n`, `\t`, `\\`, `\'`, `\"`
- String interning: Python may reuse short string objects
- `\` continuation: backslash at end of line continues to next line
