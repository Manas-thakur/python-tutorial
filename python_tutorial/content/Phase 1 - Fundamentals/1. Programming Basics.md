# Programming Basics

## Definition

**Programming** is the process of writing instructions (code) that a computer can follow to perform specific tasks. A **programming language** like Python is a formal language used to communicate these instructions to a computer.

**Python** is a high-level, interpreted, general-purpose programming language known for its readability and simplicity. Created by Guido van Rossum and first released in 1991, Python emphasizes code readability with its use of significant indentation.

## Why it exists

Computers only understand machine language (binary  -  0s and 1s). Writing programs in binary is impractical for humans. Programming languages exist to bridge this gap  -  they allow humans to write instructions in a readable form, which is then translated into machine code that the computer executes.

Python specifically exists to make programming:
- **Easier to read and write**  -  syntax is clean and intuitive
- **Faster to develop**  -  less code needed compared to C/Java
- **Portable**  -  runs on Windows, macOS, Linux, etc.
- **Extensible**  -  huge standard library and third-party ecosystem

## How it works

### Interpreted vs Compiled Languages

| Feature | Compiled (C, C++, Rust) | Interpreted (Python, JavaScript) |
|---------|-------------------------|----------------------------------|
| Translation | Entire program → machine code before running | Line-by-line at runtime |
| Speed | Faster execution | Slower execution |
| Portability | Must recompile for each OS | Cross-platform if interpreter exists |
| Debugging | Harder (compile first) | Easier (run immediately) |

### Python's Execution Model

Python is **interpreted**, but with an important intermediate step:

```
Source Code (.py) → Bytecode (.pyc) → PVM (Python Virtual Machine) → Output
```

1. **Source Code**  -  You write `.py` files with Python code
2. **Bytecode Compilation**  -  Python compiles your code into low-level **bytecode** (stored in `__pycache__/` as `.pyc` files)
3. **PVM Execution**  -  The Python Virtual Machine (PVM) interprets the bytecode and executes it on the underlying hardware

This hybrid approach gives Python portability (bytecode runs anywhere with a PVM) while keeping the interactive developer experience.

## Internal working

When you run `python script.py`:

1. **Lexing**  -  The source code is broken into tokens (keywords, identifiers, operators, etc.)
2. **Parsing**  -  Tokens are structured into an **Abstract Syntax Tree (AST)**  -  a tree representation of the code's grammatical structure
3. **Compilation**  -  The AST is compiled into **bytecode**  -  platform-independent low-level instructions
4. **Interpretation**  -  The PVM (a stack-based virtual machine) executes each bytecode instruction one by one
5. **Memory management**  -  Python's memory manager handles allocation; garbage collector frees unused objects (reference counting + cycle detection)

```python
# Example bytecode (visible via dis module)
import dis

def greet(name):
    return f"Hello, {name}!"

dis.dis(greet)
```

```
  2           0 LOAD_CONST               1 ('Hello, ')
              2 LOAD_FAST                0 (name)
              4 FORMAT_VALUE             0
              6 BUILD_STRING             2
              8 RETURN_VALUE
```

## Real-world analogy

Think of Python like a **chef following a recipe**:

- **Source code** = the recipe written in English (human language)
- **Compiler/Interpreter** = the chef who reads the recipe
- **Bytecode** = the chef's prep notes (halfway between recipe and cooking)
- **PVM** = the kitchen where cooking happens
- **Computer** = the person eating the meal

Just as a recipe can be followed in any kitchen, Python bytecode can run on any platform with a PVM.

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Forgetting `.py` extension | `python script` | `python script.py` |
| Wrong indentation | `print("hi")` (no indent after def) | 4 spaces inside blocks |
| Using `print` as variable name | `print = 5` (breaks print()) | Use a different name |
| Mixing tabs and spaces | Python 3 disallows mixing | Use 4 spaces consistently |
| Running wrong Python version | `python` vs `python3` | Use `python3` on Linux/Mac |

## Interview questions + answers

**Q1: Is Python compiled or interpreted?**
A1: Both. Python compiles source code to bytecode (a compilation step), then interprets the bytecode in the PVM. It's primarily called an interpreted language because execution happens line-by-line at runtime without a separate compile step.

**Q2: What is bytecode in Python?**
A2: Bytecode is a low-level, platform-independent representation of your Python code. It's stored in `__pycache__/` directory as `.pyc` files. The PVM executes this bytecode.

**Q3: What is the difference between CPython, PyPy, and Jython?**
A3: All are Python implementations. CPython (the standard) compiles to bytecode and interprets it. PyPy uses a JIT compiler for faster execution. Jython compiles Python to Java bytecode for the JVM.

**Q4: What does the `#` character do in Python?**
A4: It starts a single-line comment. Everything after `#` on that line is ignored by the Python interpreter.

**Q5: How do you write multi-line comments in Python?**
A5: Python doesn't have a dedicated multi-line comment syntax. Multi-line strings using `"""` or `'''` are commonly used as multi-line comments, though they are technically string literals.

**Q6: What happens when you run `python -i script.py`?**
A6: It runs the script and then enters interactive mode, allowing you to inspect variables and continue working with the program's state.

**Q7: What is the PVM?**
A7: The Python Virtual Machine is the runtime engine that executes Python bytecode. It's the core component that makes Python portable across platforms.

**Q8: What is the difference between `python` and `python3` commands?**
A8: On some systems, `python` points to Python 2.x (if installed), while `python3` always points to Python 3.x. Python 2 is deprecated  -  always use `python3` or ensure `python` points to Python 3.

**Q9: What is `__pycache__`?**
A9: A directory where Python stores compiled bytecode files (.pyc) to speed up subsequent runs. Python creates it automatically.

**Q10: Can you run Python code without creating a file?**
A10: Yes  -  use interactive mode by running `python` without arguments, or use `python -c 'print("hi")'` to execute inline code.

## Revision notes

- Programming = giving instructions to a computer in a formal language
- Python is high-level, interpreted, dynamically-typed, and garbage-collected
- Python source → Bytecode → PVM → Execution
- Bytecode stored in `__pycache__/*.pyc`
- Interpreted languages: easier debugging, slower execution
- Compiled languages: faster execution, platform-specific binaries
- Run scripts: `python script.py`
- Interactive mode: just `python`
- Comments: `#` for single line, `"""` for multi-line strings used as comments
- Never mix tabs and spaces (use 4 spaces)
- PVM = Python Virtual Machine = bytecode interpreter
