# Variables and Data Types

## Definition

A **variable** is a named container that stores a value in memory. A **data type** defines what kind of data a variable holds (numbers, text, etc.) and what operations can be performed on it.

In Python, variables are **dynamically typed** — you don't need to declare the type; Python infers it from the assigned value.

```python
name = "Alice"     # str
age = 25            # int
height = 5.6        # float
is_student = True   # bool
```

## Why it exists

Variables exist so programs can store, manipulate, and reference data by name. Without variables, you'd have to work directly with raw memory addresses. Data types exist because different kinds of data:
- Use different amounts of memory
- Support different operations (you can divide numbers, not strings)
- Need different internal representations (integers vs floating-point)

Dynamic typing (Python's approach) makes code faster to write and more flexible. Static typing (C/Java) catches type errors earlier and can be more performant.

## How it works

### Variable Assignment

```python
x = 10  # Python creates an integer object 10 and binds name 'x' to it
```

Python uses **reference semantics** — variables refer to objects, not memory boxes that hold values.

### Naming Conventions & Rules

**Rules** (must follow, or Python raises `SyntaxError`):
- Must start with a letter or underscore (`_`)
- Remaining characters can be letters, digits, or underscores
- Case-sensitive (`Age` ≠ `age`)
- Cannot use **reserved keywords** (`if`, `else`, `while`, `class`, `def`, `return`, `import`, `True`, `False`, `None`, etc.)

**Conventions** (should follow, or code is considered bad style):
- **snake_case** for variables and functions: `user_name`, `total_price`
- **UPPER_SNAKE_CASE** for constants: `MAX_SIZE`, `PI`
- **CamelCase** for classes: `Person`, `BankAccount`
- Descriptive names: `student_count` not `sc` or `x`

### Reserved Keywords (35 in Python 3)


### Data Types

#### int (Integer)

- Unlimited precision (can be arbitrarily large)
- Signed (positive and negative)

```python
a = 42
b = -7
c = 0
d = 123456789012345678901234567890  # huge integer, no overflow
```

#### float (Floating-Point)

- Double-precision (64-bit) — follows **IEEE 754** standard
- Approximate representation (not exact for many decimal fractions)

```python
a = 3.14
b = -0.001
c = 1.5e10  # scientific notation: 1.5 × 10^10
d = 0.1 + 0.2  # 0.30000000000000004 (floating-point precision issue!)
```

#### str (String)

- Immutable sequence of Unicode characters
- Can use single quotes, double quotes, or triple quotes

```python
s1 = 'hello'
s2 = "world"
s3 = '''multi-line
string'''
```

- **Unicode support**: Python 3 strings are Unicode by default

```python
emoji = "🔥"
greek = "λ"
chinese = "你好"
```

#### bool (Boolean)

- Only two values: `True` and `False`
- Internally, `bool` is a subclass of `int`: `True == 1`, `False == 0`

```python
is_ready = True
is_done = False
print(True + True)    # 2 (because True is 1)
```

### Type Conversion

#### Implicit Conversion (coercion)

Python automatically converts types when it makes sense:

```python
result = 10 + 3.14    # 13.14 (int → float)
result = True + 5     # 6 (bool → int)
```

#### Explicit Conversion (casting)

You manually convert using type constructor functions:

```python
int("42")        # 42 (str → int)
float("3.14")    # 3.14 (str → float)
str(100)         # "100" (int → str)
bool(1)          # True
bool(0)          # False
bool("")         # False (empty string)
bool("hello")    # True (non-empty string)
int("hello")     # ValueError!
```

### The `type()` Function

Returns the type of any object:

```python
print(type(42))         # <class 'int'>
print(type(3.14))       # <class 'float'>
print(type("hello"))    # <class 'str'>
print(type(True))       # <class 'bool'>
```

## Internal working

### Variable Internals

When you write `x = 10`:

1. Python allocates an **object** in heap memory for the value `10`
2. The object has a **type** (`int`), **reference count** (1), and **value** (`10`)
3. The name `x` is stored in the **namespace** (a dictionary mapping names → references)
4. `x` becomes a **reference** (pointer) to the object

```
  Namespace          Heap
  ┌──────┐          ┌──────────────┐
  │ x ─────────────→│ int object   │
  └──────┘          │ value: 10    │
                    │ refcount: 1  │
                    └──────────────┘
```

### Immutability

`int`, `float`, `str`, `bool`, and `tuple` are **immutable** — once created, they cannot be changed.

```python
x = 10
x = x + 1  # This doesn't modify the 10 object; it creates a new object 11
```

### String Internal: Interning

Python **interns** short strings (auto-optimization for small strings):

```python
a = "hello"
b = "hello"
print(a is b)  # True (same object — Python reused it)
```

### Float Internal: IEEE 754

A float uses 64 bits:
- 1 bit for sign
- 11 bits for exponent
- 52 bits for mantissa (fractional part)

This means many decimal numbers (like 0.1) cannot be represented exactly — they're approximations.

## Real-world analogy

Variables are like **labeled storage boxes**:

- **Variable name** = label on the box (e.g., "Books")
- **Value** = what's inside the box
- **Data type** = what kind of items belong (e.g., "books" not "food")
- **Reassignment** = taking out old contents and putting new ones in
- **Dynamic typing** = a box where you can put books today and clothes tomorrow (flexible but risky)
- **Static typing** = a box permanently labelled "Books only" (safe but rigid)

## Examples (with code)

### Variables in action

```python
# Dynamic typing in action
x = 42
print(type(x))  # <class 'int'>

x = "now I'm a string"
print(type(x))  # <class 'str'> — same variable, different type!
```

### Type checking and conversion

```python
age = input("Enter age: ")      # always returns str
age = int(age)                   # explicit conversion
print(f"Next year you'll be {age + 1}")
```

### Common type operations

```python
# int
print(10 // 3)   # 3 (floor division)
print(10 % 3)    # 1 (modulo)
print(2 ** 10)   # 1024 (power)

# float
print(round(3.14159, 2))  # 3.14
print(0.1 + 0.2)          # 0.30000000000000004

# str
print("hello " + "world")   # "hello world" (concatenation)
print("ha" * 3)             # "hahaha" (repetition)

# bool
print(10 > 5)   # True
print(bool("")) # False
```

### Checking variable type

```python
value = 42.0
print(isinstance(value, int))     # False
print(isinstance(value, float))   # True
print(isinstance(value, (int, float)))  # True
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Using reserved keyword | `class = 5` | `klass = 5` or `class_ = 5` |
| Starting with digit | `1st_name = "A"` | `first_name = "A"` |
| Forgetting `int()` with input | `age = input("Age: ")` then `age + 1` | `age = int(input("Age: "))` |
| Unexpected float precision | `0.1 + 0.2 == 0.3` → `False` | Use `round()` or `Decimal` |
| Confusing `=` and `==` | `if x = 5:` (assignment) | `if x == 5:` (comparison) |
| Assuming type stays same | Reassigning changes type | Be mindful of dynamic typing |
| CamelCase for variables | `myVariable = 5` | `my_variable = 5` |

## Interview questions + answers

**Q1: What is dynamic typing in Python?**
A1: Variables don't have fixed types. The same variable can reference objects of different types during execution. The type is determined at runtime based on the assigned value.

**Q2: What is the difference between `is` and `==`?**
A2: `==` checks value equality (do objects have the same value?). `is` checks identity equality (are they the same object in memory?).

**Q3: Why does `0.1 + 0.2` not equal `0.3`?**
A3: Floating-point numbers are represented in binary (IEEE 754). 0.1 and 0.2 have no exact binary representation, so the result is a tiny approximation error. Use `Decimal` for exact decimal arithmetic.

**Q4: What are Python's reserved keywords?**
A4: Words that have special meaning in Python and cannot be used as identifiers. Examples: `if`, `else`, `while`, `for`, `def`, `class`, `return`, `import`, `True`, `False`, `None`.

**Q5: Are strings mutable or immutable in Python?**
A5: Immutable. Once created, a string cannot be changed. Operations like `.upper()` or concatenation create new strings.

**Q6: What is the difference between `int` and `float` division?**
A6: `/` always returns a float. `//` performs floor division and returns an int (if both operands are ints).

**Q7: What does `type()` do?**
A7: Returns the type of an object as a type object: `type(42)` returns `<class 'int'>`.

**Q8: What is the `None` type?**
A8: `None` is a singleton object of type `NoneType` representing the absence of a value. Similar to `null` in other languages.

**Q9: Can you change the type of a variable after assignment?**
A9: Yes, Python is dynamically typed. `x = 10; x = "hello"` is perfectly valid.

**Q10: What is Python's variable naming convention?**
A10: **snake_case** — lowercase with underscores between words: `user_name`, `total_amount`.

## Revision notes

- Variable = named reference to an object in memory
- Dynamic typing: type inferred from assigned value, can change
- Naming: snake_case, start with letter/underscore, no reserved keywords
- `int`: unlimited precision, signed
- `float`: double-precision IEEE 754 (64-bit), approximate
- `str`: immutable sequence of Unicode characters
- `bool`: subclass of int, `True = 1`, `False = 0`
- Type conversion: implicit (auto-coercion) and explicit (`int()`, `str()`, etc.)
- `type()` returns the type of any object
- `isinstance(obj, type)` checks type safely
- Immutable types: int, float, str, bool, tuple, NoneType
- Use `Decimal` for exact money calculations
- `0.1 + 0.2` is a classic floating-point gotcha
