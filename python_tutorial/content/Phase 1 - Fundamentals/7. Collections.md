# Collections

## Definition

**Collections** are data structures that store multiple values in a single variable. Python's four main built-in collection types are:

| Type | Ordered | Mutable | Allows Duplicates | Syntax |
|------|---------|---------|-------------------|--------|
| **List** | ✅ | ✅ | ✅ | `[1, 2, 3]` |
| **Tuple** | ✅ | ❌ | ✅ | `(1, 2, 3)` |
| **Set** | ❌ | ✅ | ❌ | `{1, 2, 3}` |
| **Dictionary** | ✅ (3.7+) | ✅ | Keys: ❌, Values: ✅ | `{"a": 1, "b": 2}` |

## Why it exists

Real programs need to work with groups of data:
- A list of student names
- A tuple of coordinates (x, y) that shouldn't change
- A set of unique tags for a blog post
- A dictionary mapping user IDs to user profiles

Each collection type serves a specific purpose. Choosing the right one makes your code more efficient, readable, and correct.

## How it works

### 1. Lists (`list`)

Ordered, mutable sequences. The workhorse of Python collections.

```python
# Creation
fruits = ["apple", "banana", "cherry"]
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]
empty = []
nested = [[1, 2], [3, 4]]

# list() constructor
chars = list("hello")   # ['h', 'e', 'l', 'l', 'o']
```

#### List Indexing & Slicing

```python
fruits = ["apple", "banana", "cherry", "date", "elderberry"]

print(fruits[0])       # apple
print(fruits[-1])      # elderberry
print(fruits[1:3])     # ['banana', 'cherry']
print(fruits[::-1])    # Reversed list
```

#### List Methods

```python
numbers = [1, 2, 3]

# Adding elements
numbers.append(4)          # [1, 2, 3, 4] (add to end)
numbers.extend([5, 6])     # [1, 2, 3, 4, 5, 6]
numbers.insert(0, 0)       # [0, 1, 2, 3, 4, 5, 6]

# Removing elements
numbers.remove(3)          # Remove first occurrence of 3
popped = numbers.pop()     # Remove and return last element (6)
popped = numbers.pop(2)    # Remove and return element at index 2

# Other operations
numbers.sort()             # Sort in-place
numbers.reverse()          # Reverse in-place
print(len(numbers))        # Length
print(numbers.count(1))    # Count occurrences of 1
print(numbers.index(2))    # Index of first occurrence of 2
```

### 2. Tuples (`tuple`)

Ordered, **immutable** sequences. Use when data shouldn't change.

```python
# Creation
point = (3, 4)
singleton = (1,)         # Comma required for single-element tuple
empty = ()
coordinates = 10, 20     # Parentheses optional

# tuple() constructor
t = tuple([1, 2, 3])    # (1, 2, 3)

# Access (same as lists)
print(point[0])    # 3
print(point[-1])   # 4

# Immutable — cannot modify!
# point[0] = 5     # TypeError!
```

#### Packing and Unpacking

```python
# Packing
point = 3, 4, 5    # (3, 4, 5)

# Unpacking
x, y, z = point
print(x, y, z)     # 3 4 5

# Swap trick
a, b = 10, 20
a, b = b, a        # a=20, b=10

# Extended unpacking
first, *rest = [1, 2, 3, 4]
print(first)  # 1
print(rest)   # [2, 3, 4]
```

### 3. Sets (`set`)

Unordered collections of **unique** elements. Mutable but elements must be immutable (hashable).

```python
# Creation
colors = {"red", "blue", "green"}
numbers = {1, 2, 3, 2, 1}     # {1, 2, 3} — duplicates removed
empty = set()                  # {} is an empty dict, not set!

# set() constructor
s = set([1, 2, 2, 3])         # {1, 2, 3}

# Adding/Removing
colors.add("yellow")
colors.remove("blue")          # Raises KeyError if missing
colors.discard("blue")         # No error if missing
popped = colors.pop()          # Remove and return arbitrary element
colors.clear()                 # Empty the set
```

#### Set Operations

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a | b)   # Union:          {1, 2, 3, 4, 5, 6}
print(a & b)   # Intersection:   {3, 4}
print(a - b)   # Difference:     {1, 2}
print(b - a)   # Difference:     {5, 6}
print(a ^ b)   # Symmetric Diff: {1, 2, 5, 6}

# Method versions
print(a.union(b))
print(a.intersection(b))
print(a.difference(b))

# Subset/Superset
print({1, 2}.issubset(a))     # True
print(a.issuperset({1, 2}))   # True
```

### 4. Dictionaries (`dict`)

Key-value mappings (hash maps). Keys must be immutable (strings, numbers, tuples of immutables).

```python
# Creation
student = {
    "name": "Alice",
    "age": 25,
    "grade": "A"
}

empty = {}
scores = dict(math=90, science=85)  # {'math': 90, 'science': 85}
pairs = dict([("a", 1), ("b", 2)])  # {'a': 1, 'b': 2}

# Accessing
print(student["name"])      # Alice
print(student.get("phone")) # None (no KeyError)
print(student.get("phone", "N/A"))  # 'N/A'

# Modifying
student["age"] = 26         # Update existing key
student["city"] = "NYC"     # Add new key-value pair
student.update({"grade": "A+", "year": 2024})  # Multiple updates

# Removing
del student["grade"]
age = student.pop("age")    # Remove and return value
last = student.popitem()    # Remove and return last inserted (3.7+)

# View methods
print(student.keys())       # dict_keys(['name', 'city', ...])
print(student.values())     # dict_values(['Alice', 'NYC', ...])
print(student.items())      # dict_items([('name', 'Alice'), ...])

# Iteration
for key, value in student.items():
    print(f"{key}: {value}")
```

## Internal working

### Lists — Dynamic Arrays

Python lists are **dynamic arrays** (not linked lists). They allocate extra space for future elements:

```
List: [1, 2, 3, _, _]
       0  1  2  3  4
       ↑ allocated capacity
```

- **Append** to end: O(1) amortized (occasional resize)
- **Insert** at beginning: O(n) (shift all elements)
- **Index** access: O(1)
- **Pop** from end: O(1)
- **Pop** from front: O(n)

When the underlying array is full and you append, Python allocates a new larger array (typically ~1.125x growth) and copies elements over.

### Sets and Dicts — Hash Tables

Both sets and dictionaries use **hash tables**:

1. When you add an element/key, Python computes `hash(value)` to get an integer
2. It uses that hash to determine the bucket (slot) where the element is stored
3. For lookup: compute hash, go to bucket, check equality (for collision resolution)

```
Hash table (simplified):
┌─────┐
│  _  │  bucket 0
├─────┤
│  _  │  bucket 1
├─────┤
│ v1 │  bucket 2 (hash("apple") % table_size = 2)
├─────┤
│ v2 │  bucket 3 (hash("banana") % table_size = 3)
└─────┘
```

- **Average lookup/insert/delete**: O(1)
- **Worst case** (many collisions): O(n)
- Keys must be **hashable** (immutable): strings, numbers, tuples of immutables

### Tuples — Fixed Arrays

Tuples are like lists but allocated as fixed-size arrays. They use less memory and are faster for read-only access. Python can also optimize tuple usage because the size is known at creation.

## Real-world analogy

| Collection | Analogy |
|------------|---------|
| **List** | A shopping list (ordered, can add/remove, can have duplicates) |
| **Tuple** | A printed calendar (ordered, fixed, cannot change) |
| **Set** | A bag of unique marbles (no order, no duplicates, can add/remove) |
| **Dict** | A phone book (look up name → number, names are unique) |

## Examples (with code)

### List as stack (LIFO)

```python
stack = []
stack.append(1)    # Push
stack.append(2)
stack.append(3)
print(stack.pop()) # 3
print(stack.pop()) # 2
```

### List as queue (FIFO)

```python
from collections import deque
queue = deque(["Alice", "Bob", "Charlie"])
queue.append("Diana")       # Enqueue
print(queue.popleft())      # Alice (dequeue)
print(queue.popleft())      # Bob
```

### Dictionary patterns

```python
# Counting with dict
text = "hello world"
count = {}
for char in text:
    count[char] = count.get(char, 0) + 1
print(count)
# {'h': 1, 'e': 1, 'l': 3, 'o': 2, ' ': 1, 'w': 1, 'r': 1, 'd': 1}

# Grouping
students = [
    ("Alice", "A"),
    ("Bob", "B"),
    ("Charlie", "A"),
    ("Diana", "C"),
]
by_grade = {}
for name, grade in students:
    by_grade.setdefault(grade, []).append(name)
print(by_grade)  # {'A': ['Alice', 'Charlie'], 'B': ['Bob'], 'C': ['Diana']}
```

### Set for unique tracking

```python
# Remove duplicates from a list
nums = [1, 2, 2, 3, 3, 3, 4]
unique = list(set(nums))
print(unique)  # [1, 2, 3, 4] (order not guaranteed)

# Find common items in two lists
a = [1, 2, 3, 4]
b = [3, 4, 5, 6]
common = list(set(a) & set(b))
print(common)  # [3, 4]
```

## Mini Project: Contact Book

```python
# contact_book.py
print("=== Contact Book ===\n")

contacts = {}  # {name: {"phone": "...", "email": "..."}}

def add_contact():
    name = input("Enter name: ").strip().lower()
    if name in contacts:
        print("Contact already exists!")
        return
    phone = input("Enter phone: ")
    email = input("Enter email: ")
    contacts[name] = {"phone": phone, "email": email}
    print(f"{name.title()} added!\n")

def view_contacts():
    if not contacts:
        print("No contacts saved.\n")
        return
    print(f"\n{'Name':<15} {'Phone':<15} {'Email':<25}")
    print("-" * 55)
    for name, info in sorted(contacts.items()):
        print(f"{name.title():<15} {info['phone']:<15} {info['email']:<25}")
    print()

def search_contact():
    name = input("Enter name to search: ").strip().lower()
    if name in contacts:
        info = contacts[name]
        print(f"\nName:  {name.title()}")
        print(f"Phone: {info['phone']}")
        print(f"Email: {info['email']}\n")
    else:
        print("Contact not found.\n")

def delete_contact():
    name = input("Enter name to delete: ").strip().lower()
    if name in contacts:
        del contacts[name]
        print(f"{name.title()} deleted.\n")
    else:
        print("Contact not found.\n")

def list_all_names():
    if not contacts:
        print("No contacts.\n")
        return
    print("Saved contacts:", ", ".join(sorted(contacts.keys())))

while True:
    print("1. Add Contact")
    print("2. View All Contacts")
    print("3. Search Contact")
    print("4. Delete Contact")
    print("5. List Names")
    print("6. Exit")
    
    choice = input("\nChoose (1-6): ")
    
    if choice == "1":
        add_contact()
    elif choice == "2":
        view_contacts()
    elif choice == "3":
        search_contact()
    elif choice == "4":
        delete_contact()
    elif choice == "5":
        list_all_names()
    elif choice == "6":
        print("Goodbye!")
        break
    else:
        print("Invalid choice.\n")
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Using `{}` for empty set | `empty = {}` (creates dict) | `empty = set()` |
| Forgetting comma in single tuple | `t = (1)` (just int 1) | `t = (1,)` |
| Modifying set while iterating | `for x in s: s.remove(x)` | Iterate over copy: `for x in s.copy():` |
| Using mutable dict key | `{[1,2]: "value"}` | Use tuple: `{(1,2): "value"}` |
| Expecting ordered set | `{3,1,2}` might print in any order | Sets are unordered |
| `append` vs `extend` | `list.append([4,5])` → nested | `list.extend([4,5])` → flat |
| Shallow copy confusion | `b = a` (same list) | `b = a.copy()` or `b = list(a)` |
| Dict key not found | `d["missing"]` → KeyError | Use `d.get("missing", default)` |

## Interview questions + answers

**Q1: What is the difference between a list and a tuple?**
A1: Lists are mutable (can be changed), tuples are immutable. Lists use more memory. Tuples can be used as dictionary keys (if they contain hashable elements).

**Q2: How is a set different from a list?**
A2: Sets are unordered, unique (no duplicates), mutable but only contain hashable elements. Sets have O(1) average lookup vs O(n) for lists.

**Q3: What is the time complexity of `list.append()`?**
A3: O(1) amortized. Python occasionally resizes the underlying array, so most appends are O(1) with rare O(n) resize operations.

**Q4: How do dictionaries work internally?**
A4: Dictionaries use hash tables. The key is hashed using `hash()`, the hash determines the bucket, and the key-value pair is stored. Average O(1) lookup.

**Q5: What is the difference between `.remove()` and `.pop()` on a list?**
A5: `.remove(value)` removes the first occurrence of a value. `.pop(index)` removes and returns the item at the given index (default: last element).

**Q6: Can you use a list as a dictionary key?**
A6: No — lists are mutable and not hashable. Use a tuple instead if you need a sequence as a key.

**Q7: What is the `defaultdict`?**
A7: A subclass of `dict` from `collections` that provides a default value for missing keys: `d = defaultdict(list); d["x"].append(1)` works without checking.

**Q8: What is the difference between `dict.items()` and `dict.keys()`?**
A8: `.items()` returns key-value pairs as tuples. `.keys()` returns just the keys. Both return view objects that reflect changes to the dict.

**Q9: How do you merge two dictionaries?**
A9: Python 3.9+: `d1 | d2` or `d1.update(d2)`. Older: `{**d1, **d2}`.

**Q10: What is the advantage of using `deque` over a list for a queue?**
A10: `deque.appendleft()` and `deque.popleft()` are O(1), while `list.insert(0, ...)` and `list.pop(0)` are O(n).

## Revision notes

- **Lists**: ordered, mutable, allow duplicates, O(1) index/append
- **Tuples**: ordered, immutable, pack/unpack, can be dict keys
- **Sets**: unordered, unique, O(1) lookup, `| & - ^` operations
- **Dicts**: key-value mappings, O(1) lookup, keys must be hashable
- List methods: `append`, `extend`, `insert`, `remove`, `pop`, `sort`, `reverse`
- Set operations: `union(|)`, `intersection(&)`, `difference(-)`, `symmetric_difference(^)`
- Dict methods: `keys()`, `values()`, `items()`, `get()`, `update()`, `pop()`
- Lists = dynamic arrays (amortized O(1) append)
- Dicts/Set = hash tables (O(1) average operations)
- Tuples = fixed arrays (immutable, memory efficient)
- Use `deque` from `collections` for efficient queue operations
