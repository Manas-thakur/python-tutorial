# Files

## Definition

**File I/O** (Input/Output) refers to reading data from and writing data to files on the filesystem. Python provides built-in functions and modules to work with text files, binary files, CSV files, and JSON files.

```python
with open("filename.txt", "r") as file:
    content = file.read()
```

## Why it exists

Programs need to persist data beyond a single execution:
- Save user data and configuration
- Process large datasets that don't fit in memory
- Exchange data with other programs (CSV, JSON)
- Create logs and reports
- Read input files for batch processing

Without file I/O, all data would be lost when a program ends.

## How it works

### 1. Opening and Closing Files

```python
file = open("data.txt", "r")   # Open file
content = file.read()           # Read content
file.close()                    # Always close!
```

**File Modes:**

| Mode | Description | File pointer | If file exists | If file doesn't exist |
|------|-------------|-------------|----------------|----------------------|
| `"r"` | Read (text) | Beginning | Read | Error |
| `"w"` | Write (text) | Beginning | Overwrite | Create |
| `"a"` | Append (text) | End | Append | Create |
| `"r+"` | Read + Write | Beginning | Read/Write | Error |
| `"w+"` | Read + Write (overwrite) | Beginning | Overwrite | Create |
| `"a+"` | Read + Append | End | Read/Append | Create |
| `"b"` | Binary mode (add to above) | Varies | Varies | Varies |

### The `with` Statement (Context Manager)

The `with` statement automatically closes the file, even if an error occurs:

```python
# Recommended way — never forgets to close
with open("data.txt", "r") as file:
    content = file.read()
# File is automatically closed here
```

### 2. Reading Files

```python
with open("data.txt", "r") as f:
    # Read entire file as one string
    content = f.read()
    
    # Read one line at a time
    line = f.readline()      # "First line\n"
    
    # Read all lines into a list
    lines = f.readlines()    # ["line1\n", "line2\n", ...]
    
    # Iterate line by line (memory efficient)
    for line in f:
        print(line.strip())
```

### 3. Writing Files

```python
with open("output.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("Second line\n")
    
    # Write multiple lines
    lines = ["line1\n", "line2\n", "line3\n"]
    f.writelines(lines)
```

### 4. CSV Files

CSV (Comma-Separated Values) is a common format for tabular data.

```python
import csv

# Reading CSV
with open("data.csv", "r") as f:
    reader = csv.reader(f)
    header = next(reader)     # Skip header row
    for row in reader:
        print(row)            # Each row is a list of strings

# Reading with DictReader
with open("data.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"], row["age"])  # Access by column name

# Writing CSV
with open("output.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["Name", "Age", "City"])      # Header
    writer.writerow(["Alice", 30, "NYC"])
    writer.writerow(["Bob", 25, "LA"])
    # Write multiple rows
    writer.writerows([["Charlie", 35, "SF"], ["Diana", 28, "CHI"]])

# Writing with DictWriter
with open("output.csv", "w", newline="") as f:
    fieldnames = ["Name", "Age", "City"]
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerow({"Name": "Alice", "Age": 30, "City": "NYC"})
```

### 5. JSON Files

JSON (JavaScript Object Notation) is the most common data interchange format.

```python
import json

# Python → JSON
data = {
    "name": "Alice",
    "age": 30,
    "city": "NYC",
    "hobbies": ["reading", "hiking"],
    "is_student": False
}

# Convert Python object to JSON string
json_string = json.dumps(data, indent=2)
print(json_string)

# Write JSON to file
with open("data.json", "w") as f:
    json.dump(data, f, indent=2)

# Read JSON from file
with open("data.json", "r") as f:
    loaded_data = json.load(f)
    print(loaded_data["name"])  # Alice

# Convert JSON string to Python object
json_str = '{"name": "Bob", "age": 25}'
parsed = json.loads(json_str)
print(parsed["name"])  # Bob
```

**JSON ↔ Python Type Mapping:**

| JSON | Python |
|------|--------|
| `string` | `str` |
| `number` | `int` / `float` |
| `boolean` | `bool` |
| `null` | `None` |
| `array` | `list` |
| `object` | `dict` |

## Internal working

### File Descriptors

When you open a file, the operating system assigns a **file descriptor** (an integer). Python wraps this in a file object:

```python
with open("test.txt", "w") as f:
    print(f.fileno())  # 3 (file descriptor number)
    print(f.mode)      # 'w'
    print(f.name)      # 'test.txt'
```

### Buffering

File I/O is **buffered** by default. Data is accumulated in memory and written to disk in larger chunks for efficiency:

- **Text mode**: line-buffered (flushed on newline) or block-buffered
- **Binary mode**: block-buffered (larger chunks)
- Use `flush=True` or `f.flush()` to force write to disk

### Reading and Writing Internally

`f.read(size)` reads `size` bytes (or characters in text mode). `f.readline()` reads until `\n`. Python's file objects implement the iterator protocol for efficient line-by-line reading.

### Context Manager Protocol

The `with` statement calls `f.__enter__()` on entry and `f.__exit__()` on exit (which calls `f.close()`). This guarantees the file is closed even if an exception occurs.

## Examples (with code)

### Count words in a file

```python
def count_words(filename):
    """Count word occurrences in a text file."""
    word_count = {}
    with open(filename, "r") as f:
        for line in f:
            words = line.strip().lower().split()
            for word in words:
                word_count[word] = word_count.get(word, 0) + 1
    return word_count

# Sort by frequency
word_counts = count_words("sample.txt")
top_words = sorted(word_counts.items(), key=lambda x: x[1], reverse=True)[:10]
for word, count in top_words:
    print(f"{word}: {count}")
```

### Copy a file

```python
with open("source.txt", "r") as src, open("dest.txt", "w") as dst:
    for line in src:
        dst.write(line)
```

### Append to a log file

```python
from datetime import datetime

def log_event(message):
    with open("app.log", "a") as f:
        timestamp = datetime.now().isoformat()
        f.write(f"[{timestamp}] {message}\n")

log_event("Application started")
log_event("User logged in")
```

### Merge CSV files

```python
import csv

def merge_csv(file1, file2, output):
    with open(output, "w", newline="") as out:
        writer = csv.writer(out)
        header_written = False
        
        for fname in [file1, file2]:
            with open(fname, "r") as f:
                reader = csv.reader(f)
                header = next(reader)
                if not header_written:
                    writer.writerow(header)
                    header_written = True
                writer.writerows(reader)
```

## Mini Project: Notes App

```python
# notes_app.py
import json
from datetime import datetime

print("=== Notes App ===\n")

NOTES_FILE = "notes.json"

def load_notes():
    try:
        with open(NOTES_FILE, "r") as f:
            return json.load(f)
    except (FileNotFoundError, json.JSONDecodeError):
        return []

def save_notes(notes):
    with open(NOTES_FILE, "w") as f:
        json.dump(notes, f, indent=2)

def add_note():
    title = input("Note title: ")
    content = input("Note content: ")
    
    note = {
        "title": title,
        "content": content,
        "created": datetime.now().isoformat(),
        "id": len(load_notes()) + 1
    }
    
    notes = load_notes()
    notes.append(note)
    save_notes(notes)
    print(f"Note '{title}' saved!\n")

def view_notes():
    notes = load_notes()
    if not notes:
        print("No notes found.\n")
        return
    
    for note in notes:
        print(f"\n[{note['id']}] {note['title']}")
        print(f"    {note['created']}")
        print(f"    {note['content'][:50]}...")
    print()

def delete_note():
    notes = load_notes()
    if not notes:
        print("No notes to delete.\n")
        return
    
    note_id = int(input("Enter note ID to delete: "))
    notes = [n for n in notes if n["id"] != note_id]
    save_notes(notes)
    print("Note deleted.\n")

def search_notes():
    query = input("Search: ").lower()
    notes = load_notes()
    results = [n for n in notes 
               if query in n["title"].lower() or query in n["content"].lower()]
    
    if results:
        print(f"\nFound {len(results)} note(s):")
        for note in results:
            print(f"  [{note['id']}] {note['title']}")
    else:
        print("No matches.\n")

while True:
    print("1. Add Note")
    print("2. View All Notes")
    print("3. Delete Note")
    print("4. Search Notes")
    print("5. Exit")
    
    choice = input("\nChoose (1-5): ")
    
    if choice == "1": add_note()
    elif choice == "2": view_notes()
    elif choice == "3": delete_note()
    elif choice == "4": search_notes()
    elif choice == "5":
        print("Goodbye!")
        break
    else:
        print("Invalid choice.\n")
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Forgetting to close file | `f = open("f.txt"); content = f.read()` (no close) | Use `with open(...) as f:` |
| Writing to file opened as `"r"` | `open("f.txt", "r").write("x")` | Open with `"w"` or `"a"` |
| Reading past end of file | `f.read()` returns `""` — not error | Check if content is `""` |
| Newline in CSV | `csv.writer` adds `\r\n` issues | Use `newline=""` when opening file for CSV |
| Not handling `FileNotFoundError` | `open("missing.txt")` crashes | Use try/except or `os.path.exists()` |
| JSON serialization error | `json.dump(set([1,2]))` — sets not JSON | Convert to list first |
| Writing binary data in text mode | `open("f.jpg", "w")` for image | Use `open("f.jpg", "wb")` |
| Forgetting `encoding=` | `open("f.txt")` (uses system default) | Explicit: `open("f.txt", encoding="utf-8")` |

## Interview questions + answers

**Q1: What is the `with` statement and why is it used for file handling?**
A1: The `with` statement is a context manager that ensures proper acquisition and release of resources. For files, it automatically calls `close()` even if an exception occurs, preventing resource leaks.

**Q2: What is the difference between `read()`, `readline()`, and `readlines()`?**
A2: `read()` reads the entire file as a string. `readline()` reads one line (including `\n`). `readlines()` reads all lines into a list of strings.

**Q3: What are the different file modes in Python?**
A3: `"r"` (read), `"w"` (write, overwrite), `"a"` (append), `"r+"` (read+write), `"w+"` (read+write, overwrite), `"a+"` (read+append). Add `"b"` for binary: `"rb"`, `"wb"`.

**Q4: How do you handle JSON in Python?**
A4: `json.dump(obj, file)` to write JSON to a file, `json.load(file)` to read. `json.dumps(obj)` to convert to string, `json.loads(string)` to parse.

**Q5: What is the difference between `csv.reader` and `csv.DictReader`?**
A5: `csv.reader` returns each row as a list. `csv.DictReader` uses the first row as keys and returns each row as a dictionary keyed by column name.

**Q6: How do you read a large file line by line without loading it entirely into memory?**
A6: Use `for line in file:` — this iterates line-by-line, reading from disk as needed. The file object behaves as a generator of lines.

**Q7: What does `newline=""` do when opening a file for CSV writing?**
A7: It prevents the csv module from doubling newlines. The csv module handles newlines internally; without `newline=""`, extra `\r` characters may appear.

**Q8: What encoding does Python use by default for text files?**
A8: Platform-dependent (typically UTF-8 on modern systems). It's best practice to specify explicitly: `open("f.txt", encoding="utf-8")`.

**Q9: What is the difference between `"w"` and `"a"` modes?**
A9: `"w"` truncates the file (overwrites from beginning). `"a"` appends to the end (preserves existing content).

**Q10: How do you check if a file exists before opening it?**
A10: `import os; os.path.exists("filename")` or `from pathlib import Path; Path("filename").exists()`.

## Revision notes

- `open(filename, mode)` → file object
- `with open(...) as f:` → auto-close (recommended)
- Modes: `r` (read), `w` (write), `a` (append), `r+`/`w+`/`a+` (read+write), add `b` for binary
- Reading: `f.read()`, `f.readline()`, `f.readlines()`, `for line in f`
- Writing: `f.write(string)`, `f.writelines(list)`
- **CSV**: `csv.reader`, `csv.writer`, `csv.DictReader`, `csv.DictWriter` (use `newline=""`)
- **JSON**: `json.load(f)` (file→obj), `json.dump(obj, f)` (obj→file), `json.loads(s)`, `json.dumps(obj)`
- Always specify encoding: `open("f.txt", encoding="utf-8")`
- `FileNotFoundError` for missing files in `"r"` mode
- File objects are iterable (line-by-line reading)
- `flush()` forces write to disk
- `os.path.exists()` / `Path.exists()` to check if file exists
