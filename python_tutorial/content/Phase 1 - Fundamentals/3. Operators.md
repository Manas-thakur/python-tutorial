# Operators

## Definition

**Operators** are special symbols (or keywords) that perform operations on values and variables. Python has several categories of operators: arithmetic, comparison (relational), logical, and assignment operators.

```python
result = 10 + 5   # '+' is an operator, '10' and '5' are operands
```

## Why it exists

Operators exist because programs need to:
- Perform mathematical calculations (arithmetic operators)
- Compare values to make decisions (comparison operators)
- Combine multiple conditions (logical operators)
- Store and update values efficiently (assignment operators)

Without operators, every computation and decision would require verbose function calls.

## How it works

### 1. Arithmetic Operators

Used for mathematical operations:

| Operator | Name | Example | Result |
|----------|------|---------|--------|
| `+` | Addition | `10 + 3` | `13` |
| `-` | Subtraction | `10 - 3` | `7` |
| `*` | Multiplication | `10 * 3` | `30` |
| `/` | Division (float) | `10 / 3` | `3.333...` |
| `//` | Floor Division | `10 // 3` | `3` |
| `%` | Modulo (remainder) | `10 % 3` | `1` |
| `**` | Exponentiation | `2 ** 4` | `16` |

```python
print(10 + 3)    # 13
print(10 - 3)    # 7
print(10 * 3)    # 30
print(10 / 3)    # 3.3333333333333335
print(10 // 3)   # 3 (floor division  -  rounds down)
print(10 % 3)    # 1 (remainder)
print(2 ** 4)    # 16 (2 to the power of 4)
```

**Special behaviors:**

```python
# String + and *
print("Hello" + " " + "World")   # "Hello World" (concatenation)
print("Ha" * 3)                  # "HaHaHa" (repetition)
```

### 2. Comparison Operators

Compare values and return `True` or `False`:

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `==` | Equal to | `5 == 5` | `True` |
| `!=` | Not equal to | `5 != 3` | `True` |
| `<` | Less than | `3 < 5` | `True` |
| `>` | Greater than | `3 > 5` | `False` |
| `<=` | Less than or equal | `5 <= 5` | `True` |
| `>=` | Greater than or equal | `5 >= 3` | `True` |

```python
print(10 == 10)   # True
print(10 != 5)    # True
print(10 < 5)     # False
print(10 > 5)     # True
print(10 <= 10)   # True
print(10 >= 11)   # False
```

**Chained comparisons:**

```python
x = 5
print(1 < x < 10)    # True (equivalent to 1 < x and x < 10)
print(10 > x > 1)    # True
```

### 3. Logical Operators

Combine conditional statements:

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `and` | True if both are True | `True and False` | `False` |
| `or` | True if at least one is True | `True or False` | `True` |
| `not` | Inverts the boolean | `not True` | `False` |

```python
age = 25
has_license = True

print(age >= 18 and has_license)   # True (can drive)
print(age >= 18 or has_license)    # True
print(not has_license)             # False
```

**Short-circuit evaluation:**

- `and` stops evaluating at the first `False`
- `or` stops evaluating at the first `True`

```python
def get_expensive_value():
    print("Computing...")
    return 42

# Short-circuit: the second function is NEVER called
result = False and get_expensive_value()  # Nothing printed!
result = True or get_expensive_value()    # Nothing printed!
```

**Truthy/Falsy values:**

Values that behave like `False` in boolean contexts:
- `False`, `None`, `0`, `0.0`, `""`, `[]`, `{}`, `()`, `set()`

Everything else is truthy.

```python
print(bool(0))          # False
print(bool(""))         # False
print(bool([]))         # False
print(bool("hello"))    # True
print(bool([1, 2]))     # True
```

### 4. Assignment Operators

Store and update values:

| Operator | Example | Equivalent to |
|----------|---------|---------------|
| `=` | `x = 5` | `x = 5` |
| `+=` | `x += 3` | `x = x + 3` |
| `-=` | `x -= 3` | `x = x - 3` |
| `*=` | `x *= 3` | `x = x * 3` |
| `/=` | `x /= 3` | `x = x / 3` |
| `//=` | `x //= 3` | `x = x // 3` |
| `%=` | `x %= 3` | `x = x % 3` |
| `**=` | `x **= 3` | `x = x ** 3` |

```python
x = 10
x += 5   # x = 15
x -= 3   # x = 12
x *= 2   # x = 24
x /= 4   # x = 6.0
x //= 2  # x = 3.0
print(x) # 3.0
```

### Operator Precedence (PEMDAS)

The order in which operators are evaluated (highest to lowest):

| Level | Operators | Notes |
|-------|-----------|-------|
| 1 | `**` | Exponentiation (right-to-left) |
| 2 | `+x`, `-x`, `~x` | Unary operators |
| 3 | `*`, `/`, `//`, `%` | Multiplication, division |
| 4 | `+`, `-` | Addition, subtraction |
| 5 | `<<`, `>>` | Bitwise shifts |
| 6 | `&` | Bitwise AND |
| 7 | `^` | Bitwise XOR |
| 8 | `|` | Bitwise OR |
| 9 | `==`, `!=`, `<`, `>`, `<=`, `>=` | Comparisons |
| 10 | `not` | Logical NOT |
| 11 | `and` | Logical AND |
| 12 | `or` | Logical OR |
| 13 | `=`, `+=`, etc. | Assignment |

```python
result = 2 + 3 * 4 ** 2
# 4 ** 2 = 16 (exponent first)
# 3 * 16 = 48 (multiplication next)
# 2 + 48 = 50 (addition last)
print(result)  # 50

# Use parentheses to override
result = (2 + 3) * 4 ** 2
print(result)  # 5 * 16 = 80
```

## Internal working

### Operator Overloading

Operators in Python are implemented via **dunder (double underscore) methods** on objects. When you write `a + b`, Python calls `a.__add__(b)` under the hood.

```python
# What happens internally
x = 10 + 3   # Python calls: x = (10).__add__(3)
y = 10 - 3   # Python calls: y = (10).__sub__(3)
```

This is why different types behave differently with the same operator:
- `int.__add__` does numeric addition
- `str.__add__` does string concatenation

### Integer and Float Operations

- If either operand is a `float`, Python converts the result to `float` (wider type)
- `//` performs floor division: result is the largest integer ≤ the true division result

### Short-Circuit Internals

Logical operators `and`/`or` return the **actual value** of one operand, not necessarily `True`/`False`:

```python
print(0 and 42)    # 0 (0 is falsy, short-circuit returns 0)
print(3 and 42)    # 42 (3 is truthy, returns second operand)
print(0 or 42)     # 42 (0 is falsy, returns second operand)
print(3 or 42)     # 3 (3 is truthy, short-circuit returns 3)
```

## Real-world analogy

Think of operators as **kitchen tools**:

- **Arithmetic operators** = measuring cups and scales (combine ingredients)
- **Comparison operators** = a scale that compares weights (which is heavier?)
- **Logical operators** = a recipe requiring both eggs AND flour, or milk OR water
- **Assignment operators** = updating your shopping list (add 2 more eggs)

upper = 24.9 * (height ** 2)

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Using `=` instead of `==` | `if x = 5:` | `if x == 5:` |
| Integer division confusion | `7 / 2 = 3.5` (correct) | But `7 // 2 = 3` |
| Modulo with negative numbers | `-7 % 3 = ?` | `-7 % 3 = 2` (Python: result has sign of divisor) |
| Operator precedence | `2 + 3 * 4 = 20` (wrong) | `2 + 3 * 4 = 14` (correct: `*` first) |
| Assignment in expression | `if x = 5:` (SyntaxError) | `if x == 5:` |
| Forgetting parentheses for `not` | `not x > 5 and y < 10` | `not (x > 5 and y < 10)` |
| Dividing integers expecting float | `5 / 2` gives `2.5` (correct) | But `5 // 2` gives `2` |

## Interview questions + answers

**Q1: What is the difference between `/` and `//`?**
A1: `/` performs regular division and always returns a float. `//` performs floor division, rounding down to the nearest integer.

**Q2: What is the `%` operator and when would you use it?**
A2: The modulo operator returns the remainder of division. Common uses: checking even/odd (`n % 2 == 0`), cycling through indices, extracting digits.

**Q3: What is short-circuit evaluation?**
A3: `and` and `or` stop evaluating as soon as the result is determined. `False and anything` → short-circuits at False. `True or anything` → short-circuits at True.

**Q4: What does `3 * "ab" + "c"` evaluate to?**
A4: `"abababc"`. `*` repeats the string, `+` concatenates.

**Q5: What is operator overloading?**
A5: Giving different meanings to the same operator for different types. E.g., `+` adds numbers but concatenates strings. Implemented via dunder methods like `__add__`.

**Q6: What is the result of `2 ** 3 ** 2`?**
A6: `512`. `**` is right-associative, so it's `2 ** (3 ** 2) = 2 ** 9 = 512`.

**Q7: What is the difference between `=` and `==`?**
A7: `=` is assignment (store a value). `==` is comparison (check if two values are equal).

**Q8: How does Python handle `and`/`or` differently from other languages?**
A8: They return the actual value of the last evaluated expression, not necessarily `True`/`False`. E.g., `0 and 42` returns `0`, not `False`.

**Q9: What is the result of `10 % -3`?**
A9: `-2`. In Python, `%` returns a result with the sign of the divisor.

**Q10: What is operator precedence?**
A10: Rules that determine the order in which operators are evaluated. PEMDAS: Parentheses, Exponentiation, Multiplication/Division, Addition/Subtraction. Use parentheses for clarity.

## Revision notes

- Arithmetic: `+ - * / // % **`
- Comparison: `== != < > <= >=` (return bool)
- Logical: `and or not` (short-circuit evaluation)
- Assignment: `= += -= *= /= //= %= **=`
- `/` → float division, `//` → floor division, `%` → modulo
- `**` is right-associative: `2**3**2 = 2**9 = 512`
- Precedence: parentheses > exponent > mult/div > add/sub > comparison > not > and > or
- `and` returns first falsy or last truthy value
- `or` returns first truthy or last falsy value
- Chained comparisons: `1 < x < 10` works in Python
- String: `+` concatenates, `*` repeats
- Operators implemented via dunder methods (`__add__`, `__eq__`, etc.)
- Use parentheses for clarity, not just correctness
