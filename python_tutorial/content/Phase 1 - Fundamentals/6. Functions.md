# Functions

## Definition

A **function** is a reusable block of code that performs a specific task. Functions take **inputs** (parameters/arguments), process them, and optionally return a **result**. Functions are defined using the `def` keyword.

```python
def greet(name):
    return f"Hello, {name}!"

message = greet("Alice")
print(message)  # Hello, Alice!
```

## Why it exists

Functions exist to:
- **Reuse code**  -  write once, use many times
- **Organize**  -  break complex programs into manageable pieces
- **Abstract**  -  hide implementation details behind a clean interface
- **Test**  -  isolated units are easier to test and debug
- **Avoid repetition**  -  follow the DRY (Don't Repeat Yourself) principle

Without functions, every program would be a single monolithic block of repeated code.

## How it works

### Defining a Function

```python
def function_name(parameter1, parameter2, ...):
    """Docstring: describes what the function does"""
    # Function body
    return result  # Optional
```

### Parameters vs Arguments

- **Parameter**: variable in the function definition (placeholder)
- **Argument**: actual value passed when calling the function

```python
def multiply(x, y):   # x, y are parameters
    return x * y

result = multiply(3, 4)  # 3, 4 are arguments
```

### Types of Parameters

#### 1. Positional Arguments

Order matters:

```python
def describe_pet(name, animal_type):
    return f"{name} is a {animal_type}"

print(describe_pet("Max", "dog"))  # Max is a dog
print(describe_pet("dog", "Max"))  # dog is a Max (wrong order!)
```

#### 2. Keyword Arguments

Order doesn't matter  -  specified by name:

```python
def describe_pet(name, animal_type):
    return f"{name} is a {animal_type}"

print(describe_pet(animal_type="dog", name="Max"))  # Max is a dog
```

#### 3. Default Parameters

```python
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))           # Hello, Alice!
print(greet("Bob", "Hi"))       # Hi, Bob!
```

**Important:** Default parameters are evaluated once at function definition time.

#### 4. `*args`  -  Variable Positional Arguments

```python
def sum_all(*numbers):
    return sum(numbers)

print(sum_all(1, 2, 3))       # 6
print(sum_all(1, 2, 3, 4, 5)) # 15

# numbers is a tuple inside the function
```

#### 5. `**kwargs`  -  Variable Keyword Arguments

```python
def print_info(**info):
    for key, value in info.items():
        print(f"{key}: {value}")

print_info(name="Alice", age=25, city="NYC")
# name: Alice
# age: 25
# city: NYC

# info is a dict inside the function
```

### Return Values

```python
def add(a, b):
    return a + b   # Returns the result

def nothing():
    pass           # Returns None

def multiple_returns(x):
    if x > 0:
        return "positive"
    elif x < 0:
        return "negative"
    return "zero"  # Implicit else

# Multiple return values (actually a tuple)
def get_stats(numbers):
    return min(numbers), max(numbers), sum(numbers) / len(numbers)

minimum, maximum, avg = get_stats([1, 2, 3, 4, 5])
print(minimum, maximum, avg)  # 1 5 3.0
```

### Scope  -  LEGB Rule

Python resolves variable names using the **LEGB** rule:

1. **L**ocal  -  Inside the current function
2. **E**nclosing  -  Any enclosing functions (nested functions)
3. **G**lobal  -  The top-level module
4. **B**uilt-in  -  Python's built-in names (`print`, `len`, `range`, etc.)

```python
x = "global"        # Global scope

def outer():
    x = "outer"     # Enclosing scope (relative to inner)
    
    def inner():
        x = "inner" # Local scope
        print(f"Inner: {x}")
    
    inner()
    print(f"Outer: {x}")

outer()
print(f"Global: {x}")
# Inner: inner
# Outer: outer
# Global: global
```

#### Modifying Global Variables

```python
count = 0

def increment():
    global count  # Declare we're using the global variable
    count += 1

increment()
print(count)  # 1
```

**Best practice:** Avoid `global` when possible. Pass values as parameters and return new values.

## Internal working

### Function Call Stack

When you call a function:
1. Python creates a new **stack frame** (activation record)
2. Local variables are created in this frame
3. Parameters are bound to arguments
4. The function body executes
5. Return value is passed back to the caller
6. The stack frame is destroyed

```python
def outer():
    a = 1
    inner()
    
def inner():
    b = 2
    print("In inner")

outer()
```

```
Stack (LIFO):
┌─────────────────┐
│ inner() frame   │  ← Current execution
│ b = 2           │
├─────────────────┤
│ outer() frame   │
│ a = 1           │
├─────────────────┤
│ global frame    │
│ outer, inner    │
└─────────────────┘
```

### Everything is an Object

In Python, functions are **first-class objects**:

```python
def greet(name):
    return f"Hello, {name}"

# Assign to variable
my_func = greet
print(my_func("Alice"))  # Hello, Alice

# Pass as argument
def apply(func, value):
    return func(value)

print(apply(greet, "Bob"))  # Hello, Bob

# Return from another function
def make_multiplier(n):
    def multiplier(x):
        return x * n
    return multiplier

double = make_multiplier(2)
print(double(5))  # 10
```

### Docstrings

The first statement in a function can be a string literal  -  this is the **docstring**:

```python
def calculate_bmi(weight, height):
    """
    Calculate BMI given weight (kg) and height (m).
    
    Args:
        weight: Weight in kilograms
        height: Height in meters
    
    Returns:
        BMI as a float
    """
    return weight / (height ** 2)

help(calculate_bmi)
```

## Real-world analogy

A function is like a **vending machine**:

- **Function definition** = building the machine (takes time, done once)
- **Parameters** = what you select (inputs: item code, payment)
- **Function body** = the internal mechanism that processes your selection
- **Return value** = the item you receive (output)
- **Side effects** = the machine making noise (printing, writing files)
- **Docstring** = instructions on the machine (what it does, how to use it)
- **Scope** = only the machine knows its internal workings (you can't see them from outside)

## Examples (with code)

### Basic functions

```python
def is_even(number):
    """Return True if number is even."""
    return number % 2 == 0

def factorial(n):
    """Calculate factorial recursively."""
    if n <= 1:
        return 1
    return n * factorial(n - 1)

def celsius_to_fahrenheit(c):
    """Convert Celsius to Fahrenheit."""
    return (c * 9/5) + 32

print(is_even(4))        # True
print(factorial(5))      # 120
print(celsius_to_fahrenheit(0))  # 32.0
```

### Function as argument

```python
def apply_twice(func, value):
    return func(func(value))

def add_one(x):
    return x + 1

print(apply_twice(add_one, 5))  # 7
```

### Validation with functions

```python
def get_positive_number(prompt):
    """Keep asking until user enters a valid positive number."""
    while True:
        try:
            value = float(input(prompt))
            if value > 0:
                return value
            print("Please enter a positive number.")
        except ValueError:
            print("Invalid input. Enter a number.")

# Usage
age = get_positive_number("Enter your age: ")
```

## Mini Project: Unit Converter

```python
# unit_converter.py
print("=== Unit Converter ===\n")

def celsius_to_fahrenheit(c):
    return (c * 9/5) + 32

def fahrenheit_to_celsius(f):
    return (f - 32) * 5/9

def km_to_miles(km):
    return km * 0.621371

def miles_to_km(miles):
    return miles / 0.621371

def kg_to_pounds(kg):
    return kg * 2.20462

def pounds_to_kg(lbs):
    return lbs / 2.20462

def liters_to_gallons(l):
    return l * 0.264172

def gallons_to_liters(gal):
    return gal / 0.264172

def show_menu():
    print("Select conversion type:")
    print("1. Celsius ↔ Fahrenheit")
    print("2. Kilometers ↔ Miles")
    print("3. Kilograms ↔ Pounds")
    print("4. Liters ↔ Gallons")
    print("5. Exit")

def run_converter():
    converters = {
        "1": ("Celsius", "Fahrenheit", celsius_to_fahrenheit, fahrenheit_to_celsius),
        "2": ("Kilometers", "Miles", km_to_miles, miles_to_km),
        "3": ("Kilograms", "Pounds", kg_to_pounds, pounds_to_kg),
        "4": ("Liters", "Gallons", liters_to_gallons, gallons_to_liters),
    }
    
    while True:
        show_menu()
        choice = input("\nChoose (1-5): ")
        
        if choice == "5":
            print("Goodbye!")
            break
        
        if choice not in converters:
            print("Invalid choice. Try again.\n")
            continue
        
        unit_a, unit_b, to_b, to_a = converters[choice]
        direction = input(f"1: {unit_a} → {unit_b}\n2: {unit_b} → {unit_a}\nChoose (1/2): ")
        
        try:
            value = float(input(f"Enter value: "))
            if direction == "1":
                result = to_b(value)
                print(f"\n{value} {unit_a} = {result:.2f} {unit_b}\n")
            elif direction == "2":
                result = to_a(value)
                print(f"\n{value} {unit_b} = {result:.2f} {unit_a}\n")
            else:
                print("Invalid direction.\n")
        except ValueError:
            print("Invalid number.\n")

if __name__ == "__main__":
    run_converter()
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Forgetting `return` | Function returns `None` | Add `return result` |
| Mutable default args | `def f(x=[])` (list persists across calls!) | `def f(x=None)` then `x = x or []` |
| Modifying global without `global` | `x += 1` in function (if x is global) | `global x; x += 1` (or avoid globals) |
| Confusing `return` and `print` | `print(result)` in function instead of returning | `return result` and print outside |
| Wrong argument order | `f(5, y=2)` vs `f(x=2, 5)` (positional after keyword) | Keyword args must follow positional |
| Missing `self` in methods | `def method():` inside class | `def method(self):` |
| Reassigning parameter name | `def f(list):` (shadows built-in) | `def f(items):` |

## Interview questions + answers

**Q1: What is the difference between a parameter and an argument?**
A1: A **parameter** is the variable in the function definition (placeholder). An **argument** is the actual value passed when calling the function.

**Q2: Explain `*args` and `**kwargs`.**
A2: `*args` captures any number of positional arguments as a tuple. `**kwargs` captures any number of keyword arguments as a dictionary.

**Q3: What is the LEGB rule?**
A3: Python's scope resolution order: Local → Enclosing → Global → Built-in. Python searches these scopes in order when looking up a variable name.

**Q4: What happens if a function doesn't have a `return` statement?**
A4: It returns `None` by default.

**Q5: Why are mutable default arguments dangerous?**
A5: Default arguments are evaluated once at function definition time. If the default is mutable (like a list), changes persist across calls: `def f(x=[])`  -  all calls share the same list.

**Q6: What is a docstring?**
A6: A string literal as the first statement in a function/module/class that documents what it does. Accessible via `help()` and `.__doc__`.

**Q7: Can a function return multiple values?**
A7: Yes, as a tuple: `return a, b, c`. The caller can unpack: `x, y, z = func()`.

**Q8: What are first-class functions?**
A8: Functions are objects in Python  -  they can be assigned to variables, passed as arguments, returned from other functions, and stored in data structures.

**Q9: What is the difference between positional and keyword arguments?**
A9: Positional arguments are matched by order. Keyword arguments are matched by parameter name. Keyword args must come after positional args.

**Q10: What does `pass` do in a function?**
A10: It's a no-op statement that does nothing. Used as a placeholder for future code.

## Revision notes

- Function = reusable block of code defined with `def`
- Parameters: positional, keyword, default, `*args`, `**kwargs`
- `return` sends value back; no `return` → returns `None`
- LEGB: Local → Enclosing → Global → Built-in scope resolution
- Avoid `global`  -  pass parameters and return values instead

- Docstrings: `"""description"""` as first line of function
- Mutable default args are evaluated once  -  use `None` pattern
- `*args` → tuple of positional args, `**kwargs` → dict of keyword args
- Keyword arguments must follow positional arguments
- `pass` = do nothing (placeholder)
