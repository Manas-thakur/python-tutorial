# Input and Output

## Definition

**Input** is data a program receives from the outside world (keyboard, files, network). **Output** is data a program sends to the outside world (screen, files, network). In Python, `input()` reads user input as a string, and `print()` writes output to the console with flexible formatting options.

```python
name = input("Enter your name: ")    # Input
print(f"Hello, {name}!")             # Output
```

## Why it exists

Programs need to interact with users and other systems:
- **Input** allows programs to be dynamic rather than hard-coded — the same program can behave differently based on user data
- **Output** allows programs to communicate results back to the user
- **String formatting** makes output readable, well-structured, and professional

Without input/output, programs would be completely isolated and useless for real-world tasks.

## How it works

### 1. `print()` Function

`print()` sends data to the console (standard output). It can take multiple arguments and has two key parameters:

```python
print("Hello", "World")               # Hello World (space separator by default)
print("Hello", "World", sep="-")      # Hello-World
print("Hello", end="")                # No newline at end
print("World")                        # HelloWorld (continues on same line)
print("Line 1", "Line 2", sep="\n")   # Line 1\nLine 2
```

**Parameters:**
- `sep` — separator between multiple values (default: space)
- `end` — string appended after the last value (default: newline `\n`)

```python
# Printing different types
print(42)               # 42
print(3.14)             # 3.14
print([1, 2, 3])        # [1, 2, 3]
print(True)             # True
```

### 2. `input()` Function

`input()` reads a line from the user (standard input) and always returns a **string**:

```python
name = input("What's your name? ")   # Prompt is optional
print(f"Hi {name}!")
```

**Critical:** `input()` returns a string. If you need numbers, you must convert:

```python
age = input("Enter age: ")       # "25" (string)
age_num = int(age)               # 25 (integer)

# Or in one step:
age = int(input("Enter age: "))
```

### 3. String Formatting

Python has three main approaches to string formatting:

#### a) `%` Formatting (old style — legacy)

```python
name = "Alice"
age = 25
print("Name: %s, Age: %d" % (name, age))    # Name: Alice, Age: 25
print("PI: %.2f" % 3.14159)                  # PI: 3.14

# %s = string, %d = integer, %f = float, %.2f = float with 2 decimals
```

#### b) `.format()` method (Python 3+)

```python
name = "Alice"
age = 25
print("Name: {}, Age: {}".format(name, age))            # Positional
print("Name: {1}, Age: {0}".format(age, name))          # Indexed
print("Name: {n}, Age: {a}".format(n=name, a=age))      # Keyword

# Format specifiers
print("PI: {:.2f}".format(3.14159))                     # PI: 3.14
print("{:<10} | {:>10}".format("Left", "Right"))        # Alignment
print("{:^10}".format("Center"))                        # Center align
```

#### c) f-strings (Python 3.6+ — recommended)

f-strings are the most readable and performant approach:

```python
name = "Alice"
age = 25
print(f"Name: {name}, Age: {age}")                     # Basic
print(f"Next year: {age + 1}")                         # Expressions
print(f"PI: {3.14159:.2f}")                            # Format specifiers

# Expressions and function calls
print(f"Uppercase: {name.upper()}")
print(f"Sum: {sum([1, 2, 3])}")

# Alignment
print(f"{'left':<10}|{'right':>10}")
print(f"{'center':^20}")

# Numbers
price = 49.95
print(f"Price: ${price:.2f}")                          # Price: $49.95
print(f"{1000000:,}")                                   # 1,000,000
print(f"{0.25:.1%}")                                    # 25.0%
```

## Internal working

### How `print()` works

`print()` internally:
1. Converts each argument to string via `str()`
2. Joins them with `sep`
3. Appends `end`
4. Writes to `sys.stdout`

```python
# What print() roughly does internally
import sys

def my_print(*args, sep=" ", end="\n"):
    output = sep.join(str(arg) for arg in args)
    sys.stdout.write(output + end)
```

### How `input()` works

`input()` internally:
1. Writes the prompt to `sys.stdout` (no newline)
2. Reads a line from `sys.stdin`
3. Strips the trailing newline
4. Returns the string (or raises `EOFError` if input stream ends)

## Real-world analogy

Think of `input()` and `print()` like a **restaurant order**:

- **`print()`** = The waiter announcing your order to the kitchen (output)
- **`input()`** = The waiter asking for your choice and listening for your reply (input)
- **f-strings** = Presenting the bill with clear formatting (items, quantities, prices)
- **`sep`/`end`** = How the waiter arranges items on the tray or ends the sentence

## Examples (with code)

### Basic I/O

```python
# Getting user details
name = input("Enter your name: ")
age = int(input("Enter your age: "))
city = input("Enter your city: ")

print("=" * 30)
print(f"Name: {name}")
print(f"Age: {age}")
print(f"City: {city}")
print(f"{name} is {age} years old and lives in {city}.")
```

### Using `sep` and `end`

```python
# Building a progress indicator
import time

for i in range(5):
    print(".", end="", flush=True)
    time.sleep(0.5)
print(" Done!")

# Output: ..... Done!
```

### Formatting variations

```python
name = "Bob"
score = 87.5
total = 100

# % formatting
print("Student: %s, Score: %.1f/%d" % (name, score, total))

# .format() method
print("Student: {}, Score: {:.1f}/{}".format(name, score, total))

# f-string (best)
print(f"Student: {name}, Score: {score:.1f}/{total}")
```

### Table formatting

```python
print(f"{'Item':<10} {'Price':>8} {'Qty':>4}")
print("-" * 24)
print(f"{'Apples':<10} {1.50:>8.2f} {3:>4}")
print(f"{'Bananas':<10} {0.75:>8.2f} {5:>4}")
print(f"{'Oranges':<10} {2.00:>8.2f} {2:>4}")
```

## Mini Project: Personalized Greeting App

```python
# greeting_app.py
print("=== Personalized Greeting App ===\n")

# Get user info
name = input("What's your name? ")
age = int(input("How old are you? "))
city = input("Where do you live? ")
hobby = input("What's your favorite hobby? ")

# Current year
from datetime import datetime
current_year = datetime.now().year
birth_year = current_year - age

# Personalized messages
print("\n" + "=" * 50)
print(f"Hello {name}! 👋")
print("=" * 50)

print(f"""
Here's your personalized profile:
────────────────────────────────────
  Name    : {name}
  Age     : {age}
  Born in : {birth_year}
  City    : {city}
  Hobby   : {hobby}
────────────────────────────────────
""")

# Time-based greeting
hour = datetime.now().hour
if hour < 12:
    greeting = "Good morning"
elif hour < 18:
    greeting = "Good afternoon"
else:
    greeting = "Good evening"

print(f"{greeting}, {name}! Hope you enjoy {hobby} today in {city}!")

# Age-based message
if age < 13:
    print("You've got so many adventures ahead of you!")
elif age < 20:
    print("These are some of the best years — enjoy them!")
elif age < 65:
    print("Keep chasing your dreams!")
else:
    print("You're wise and wonderful!")
```

## Common mistakes

| Mistake | Wrong | Right |
|---------|-------|-------|
| Forgetting `int()` with input | `age = input("Age: ")` then `age + 1` | `age = int(input("Age: "))` |
| Missing parentheses | `print "hello"` (Python 2 syntax) | `print("hello")` |
| f-string without `f` prefix | `"Hello {name}"` | `f"Hello {name}"` |
| Wrong quotes in f-string | `f'Value: {x:.2f}'` (fine) | But can't use same quote inside |
| Using `input()` for password | Password visible while typing | Use `getpass.getpass()` |
| Forgetting `end=""` | `print(".", end="")` forget to set | Need `flush=True` sometimes |
| Mismatched format placeholders | `"%s %d" % ("hi")` (missing arg) | Match count of placeholders to args |

## Interview questions + answers

**Q1: What does `input()` always return?**
A1: A string. You must explicitly convert to int/float if you need a number.

**Q2: What is the difference between `print()` in Python 2 and Python 3?**
A2: Python 2: `print "hello"` (statement). Python 3: `print("hello")` (function). In Python 3, you can use `sep`, `end`, `file` parameters.

**Q3: What do the `sep` and `end` parameters of `print()` do?**
A3: `sep` is the separator between multiple arguments (default: space). `end` is appended after the last value (default: newline).

**Q4: What are f-strings and why are they preferred?**
A4: f-strings (Python 3.6+) embed expressions directly in string literals using `{expr}`. They're more readable, faster, and less error-prone than `%` formatting or `.format()`.

**Q5: How do you format a float to 2 decimal places using f-strings?**
A5: `f"{value:.2f}"`

**Q6: What happens if `input()` reaches end-of-file (EOF)?**
A6: It raises `EOFError`. Handle with try/except if reading from files/pipes.

**Q7: How can you read a password without showing it on screen?**
A7: Use `from getpass import getpass; password = getpass("Password: ")`.

**Q8: What is `sys.stdout.write()` and how is it different from `print()`?**
A8: `sys.stdout.write()` writes raw text without adding a newline. `print()` uses it internally but adds formatting like `sep` and `end`.

**Q9: How do you print without a newline in Python?**
A9: Use `print("text", end="")`.

**Q10: What does `print("Hello", "World", sep=", ", end="!")` output?**
A10: `Hello, World!` (comma and space between, exclamation at end, no newline).

## Revision notes

- `input()` → always returns a string; convert with `int()`, `float()` etc.
- `print()` → flexible output with `sep` and `end` parameters
- **Three formatting styles:**
  - `%` formatting: `"Value: %.2f" % x` (legacy)
  - `.format()`: `"Value: {:.2f}".format(x)` (verbose)
  - **f-strings**: `f"Value: {x:.2f}"` (preferred, Python 3.6+)
- f-string features: expressions, format specifiers, alignment, commas
- `end=""` → suppress newline; `flush=True` → force output immediately
- `sys.stdout.write()` → lower-level output
- `getpass.getpass()` → hidden password input
- Format specifiers: `.2f` (2 decimals), `,` (thousands), `<`/`>`/`^` (alignment), `%` (percentage), `d`/`b`/`o`/`x` (integer bases)
