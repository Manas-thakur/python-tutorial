# 3. Async Programming

## Definition

**Async programming** (asynchronous programming) is a concurrency model where a single thread of execution can handle multiple tasks concurrently by **suspending** a task when it waits (e.g., for I/O) and switching to another task. It is not parallelism  -  it is concurrency within one thread.

Key keywords:
- `async def`  -  defines a **coroutine** (a function that can be paused and resumed)
- `await`  -  suspends the current coroutine until the awaited awaitable completes
- **Event loop**  -  the orchestrator that schedules and runs coroutines
- **asyncio**  -  Python's standard library for async programming

```python
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)   # Non-blocking sleep
    print("World")

asyncio.run(say_hello())
```

## Why it exists

Traditional synchronous code blocks on I/O operations (network requests, file reads, database queries). This wastes CPU cycles while waiting. Threading can help but has overhead (memory per thread, GIL contention).

Async programming solves this by:
- **Maximizing I/O throughput**: A single thread can handle thousands of concurrent connections
- **Lower overhead**: Coroutines are much lighter than threads (a few KB vs MB per thread)
- **Explicit concurrency**: `await` makes it clear where control can switch  -  no mysterious race conditions from preemptive threading

It's ideal for: web servers, API clients, web scrapers, chat applications, any I/O-bound workload.

## How it works

1. Define a coroutine with `async def`.
2. Inside it, use `await` on other coroutines or awaitables.
3. Run the top-level coroutine with `asyncio.run(main())`.
4. The event loop picks up the coroutine, runs it until it hits an `await`, suspends it, and resumes it when the awaited operation completes.

```python
import asyncio

async def fetch_data(url):
    print(f"Fetching {url}")
    await asyncio.sleep(2)       # Simulate network I/O
    print(f"Finished {url}")
    return f"Data from {url}"

async def main():
    # Sequential (bad)
    result1 = await fetch_data("url1")
    result2 = await fetch_data("url2")

    # Concurrent (good)
    task1 = asyncio.create_task(fetch_data("url1"))
    task2 = asyncio.create_task(fetch_data("url2"))
    results = await asyncio.gather(task1, task2)

asyncio.run(main())
```

**Key difference**: `await coro()` starts the coroutine and waits for it. `asyncio.create_task(coro())` schedules it to run **concurrently** in the background.

## Internal working

### Event Loop

The event loop is the heart of asyncio. It maintains:
- A **task queue** of coroutines ready to run
- A **callback registry** for I/O events (via select/epoll/kqueue)
- A **timer heap** for delayed operations (`asyncio.sleep`)

Pseudo-execution:

```
Loop iteration:
  1. Run all ready tasks (coroutines that are not awaiting)
  2. Check timers → move expired ones to ready queue
  3. Check I/O events → move done ones to ready queue
  4. If nothing ready, wait for next I/O event or timer
```

When a coroutine does `await something`, it yields control back to the event loop, which can then run another coroutine.

### Coroutines vs Generators

Internally, coroutines build on generators. An `async def` function returns a **coroutine object**. When awaited, it's wrapped into a **Task** which is a Future. The event loop drives the coroutine using `send()` (like generators), suspending on `await` and resuming when the result is available.

```python
# A generator in disguise (simplified):
# coro.send(None)  starts it
# coro.throw(exc)  injects exception
# StopIteration.value  contains the return value
```

### `asyncio.gather` vs `asyncio.create_task`

- `create_task(coro)` schedules the coroutine on the event loop, returns a `Task` (subclass of `Future`). It starts running immediately.
- `gather(*tasks)` runs multiple awaitables concurrently and waits for ALL to complete. It returns a list of results in order.
- `asyncio.wait()` also runs concurrently but allows finer control (FIRST_COMPLETED, FIRST_EXCEPTION, ALL_COMPLETED).

## Real-world analogy

**A coffee shop with one barista.**

- **Sync**: The barista takes an order, makes the coffee, serves it. Only then takes the next order. Customers wait.
- **Threading**: Hire many baristas (threads), each serving one customer. Wastes space and coordination overhead.
- **Async**: One barista takes an order, starts the coffee machine (non-blocking I/O), takes the next order while machine runs, checks machine, pours, serves. The barista **switches between tasks** while waiting for machines.

The barista (event loop) never stands idle.

## Examples (with code)

### Basic async/await

```python
import asyncio

async def greet(name, delay):
    await asyncio.sleep(delay)
    print(f"Hello, {name}!")

async def main():
    await greet("Alice", 1)
    await greet("Bob", 1)   # Sequential - takes 2s total

asyncio.run(main())
```

### Concurrent execution with `create_task`

```python
async def main():
    task1 = asyncio.create_task(greet("Alice", 2))
    task2 = asyncio.create_task(greet("Bob", 1))

    print("Tasks started, waiting...")
    await task1  # Wait for Alice (Bob finishes earlier)
    await task2  # Already done

# Total time: ~2s instead of 3s
asyncio.run(main())
```

### `asyncio.gather()`

```python
async def fetch(id):
    await asyncio.sleep(id)
    return f"Result {id}"

async def main():
    results = await asyncio.gather(
        fetch(3),
        fetch(1),
        fetch(2),
    )
    print(results)  # ['Result 3', 'Result 1', 'Result 2'] - order preserved

asyncio.run(main())
```

### `asyncio.sleep()` (non-blocking)

```python
import asyncio
import time

async def main():
    # Non-blocking: other tasks can run during sleep
    start = time.perf_counter()
    async def waiter(n):
        await asyncio.sleep(n)
        return n

    tasks = [waiter(i) for i in range(1, 4)]
    await asyncio.gather(*tasks)
    print(f"Took {time.perf_counter() - start:.2f}s")  # ~3s, not 6s
```

### `async with` and `async for`

```python
import asyncio

# async with
class AsyncResource:
    async def __aenter__(self):
        print("Acquiring resource")
        await asyncio.sleep(0.5)
        return self

    async def __aexit__(self, *args):
        print("Releasing resource")
        await asyncio.sleep(0.2)

async def use_resource():
    async with AsyncResource() as res:
        print("Using resource")

# async for
class AsyncCounter:
    def __init__(self, limit):
        self.limit = limit
        self.current = 0

    def __aiter__(self):
        return self

    async def __anext__(self):
        if self.current >= self.limit:
            raise StopAsyncIteration
        await asyncio.sleep(0.1)
        self.current += 1
        return self.current

async def count_up():
    async for num in AsyncCounter(5):
        print(num)
```

### Async HTTP requests with aiohttp

```python
import asyncio
import aiohttp

async def fetch_url(session, url):
    async with session.get(url) as response:
        return await response.text()

async def main():
    urls = [
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/2",
        "https://httpbin.org/delay/3",
    ]
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_url(session, url) for url in urls]
        pages = await asyncio.gather(*tasks)
        print(f"Fetched {len(pages)} pages")

asyncio.run(main())
```

### Rate limiter with semaphore

```python
import asyncio

class RateLimiter:
    def __init__(self, max_concurrent):
        self.semaphore = asyncio.Semaphore(max_concurrent)

    async def acquire(self):
        await self.semaphore.acquire()

    def release(self):
        self.semaphore.release()

    async def __aenter__(self):
        await self.acquire()
        return self

    async def __aexit__(self, *args):
        self.release()

# Usage:
async def limited_task(limiter, id):
    async with limiter:
        print(f"Running task {id}")
        await asyncio.sleep(1)
```

### Async web page fetcher (mini project)

```python
import asyncio
import aiohttp
import time

async def fetch_page(session, url):
    start = time.perf_counter()
    async with session.get(url) as resp:
        text = await resp.text()
        elapsed = time.perf_counter() - start
        print(f"Fetched {url} ({len(text)} bytes) in {elapsed:.2f}s")
        return text

async def main():
    urls = [
        "https://httpbin.org/delay/1",
        "https://httpbin.org/delay/2",
        "https://httpbin.org/delay/3",
        "https://httpbin.org/delay/1.5",
        "https://httpbin.org/delay/2.5",
    ]
    async with aiohttp.ClientSession() as session:
        tasks = [fetch_page(session, url) for url in urls]
        results = await asyncio.gather(*tasks, return_exceptions=True)
        successes = sum(1 for r in results if not isinstance(r, Exception))
        print(f"Completed {successes}/{len(urls)} pages")

asyncio.run(main())
```

## Common mistakes

| Mistake | Why it's wrong | Fix |
|---|---|---|
| Blocking calls (e.g., `time.sleep()`) in async code | Blocks the entire event loop; no other tasks run | Use `await asyncio.sleep()` |
| Forgetting `await` | Returns a coroutine object instead of result; no error until comparison | Add `await` |
| Running `async` functions without an event loop | `RuntimeError: no running event loop` | Use `asyncio.run()` |
| Mixing sync and async libraries in hot paths | Sync call blocks event loop | Use async libraries (aiohttp, asyncpg) |
| Not handling exceptions in `gather` | One exception cancels all tasks | Use `return_exceptions=True` |
| Creating tasks but never awaiting them | Tasks cancelled when event loop closes | Always `await` or `gather` tasks |
| Using `asyncio.run()` inside a running event loop | RuntimeError (e.g., in Jupyter) | Use `await` directly or `nest_asyncio` |

## Interview questions + answers

**Q1: What is the difference between concurrency and parallelism?**
A: Concurrency is about dealing with many things at once (interleaved execution on one CPU). Parallelism is about doing many things at once (multiple CPUs). Async provides concurrency within a single thread.

**Q2: What is a coroutine?**
A: A coroutine is a function defined with `async def`. It returns a coroutine object that can be `await`ed. It can suspend execution at `await` points and yield control back to the event loop.

**Q3: What is the Global Interpreter Lock (GIL) and does async bypass it?**
A: The GIL prevents multiple threads from executing Python bytecode in parallel. Async runs on a single thread, so it doesn't face GIL issues but also doesn't give CPU parallelism.

**Q4: What is `asyncio.gather()` vs `asyncio.create_task()`?**
A: `create_task` schedules a single coroutine and returns a Task. `gather` runs multiple awaitables concurrently and waits for all to complete, returning results in order.

**Q5: How do you handle timeouts in asyncio?**
A: Use `asyncio.wait_for(coro, timeout=5.0)`. Raises `asyncio.TimeoutError` if the coroutine doesn't complete in time.

**Q6: What is `asyncio.run()` doing internally?**
A: It creates a new event loop, runs the given coroutine until complete, then closes the loop.

**Q7: Can you mix async and sync code?**
A: Yes. Use `asyncio.to_thread()` to run a sync function in a thread pool without blocking the event loop. Or use `loop.run_in_executor()`.

**Q8: What is uvloop?**
A: A drop-in replacement for the default asyncio event loop, built on libuv (Node.js's event loop). It's typically 2-4x faster for I/O operations.

## Knowledge check (5-10 questions with answers)

1. **What keyword defines a coroutine?**
   → `async def`

2. **What does `await` do?**
   → It suspends the current coroutine until the awaited awaitable completes, yielding control to the event loop.

3. **What is the main difference between `time.sleep(1)` and `await asyncio.sleep(1)`?**
   → `time.sleep(1)` blocks the entire thread (and event loop). `asyncio.sleep(1)` yields control, allowing other tasks to run.

4. **How do you run a coroutine from a sync context?**
   → Use `asyncio.run(coro())`.

5. **True or False: `asyncio.gather` returns results in the order tasks finish.**
   → False. It returns results in the order of the awaitables passed in, regardless of completion order.

6. **What is a Task?**
   → A Task wraps a coroutine and schedules it on the event loop. Created with `asyncio.create_task()`.

7. **What happens when you forget `await` on a coroutine?**
   → You get a coroutine object instead of the result. The coroutine may not execute at all (it will be garbage collected with a warning).

8. **How do you make a class work with `async with`?**
   → Implement `__aenter__` (async) and `__aexit__` (async) methods.

## Revision notes (bullet points summary)

- `async def` → coroutine, `await` → suspend and yield control
- Event loop: orchestrates coroutines, handles I/O, runs timers
- `asyncio.run(main())`  -  entry point
- `create_task()`  -  schedule concurrent work
- `asyncio.gather()`  -  run many tasks concurrently, collect results
- `asyncio.sleep()`  -  non-blocking sleep
- `async with` / `async for`  -  async context managers and iterators
- aiohttp  -  async HTTP library
- Never use blocking calls (`time.sleep`, `requests.get`) in async code
- Great for I/O-bound workloads; useless for CPU-bound
