# Dataclasses

## Definition
A **dataclass** is a class decorated with `@dataclass` that automatically generates special methods like `__init__`, `__repr__`, `__eq__`, and `__hash__` based on class-annotated fields. It's defined in the `dataclasses` module (Python 3.7+).

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

p1 = Point(3.0, 4.0)
p2 = Point(3.0, 4.0)
print(p1)           # Point(x=3.0, y=4.0)
print(p1 == p2)     # True (auto-generated __eq__)
print(p1.x)         # 3.0
```

## Why it exists
- **Reduce boilerplate**: No more manually writing `__init__`, `__repr__`, `__eq__`, `__hash__` for simple data containers
- **Immutability**: `frozen=True` creates immutable objects (like namedtuples but with richer typing)
- **Comparison support**: `order=True` generates `__lt__`, `__le__`, `__gt__`, `__ge__`
- **Type hints enforced**: Fields are defined with type annotations (not enforced at runtime, but available for mypy)
- **Default values**: Simple field defaults and `default_factory` for mutable defaults
- **Validation**: `__post_init__` hook runs after `__init__` for additional setup/validation
- **Inheritance**: Dataclasses support inheritance cleanly
- **Serialize-friendly**: Easy to convert to/from dicts and JSON (via `.asdict()`)

## How it works
- `@dataclass` decorator inspects the class's type annotations to find fields
- It generates `__init__` with parameters in field declaration order
- Fields with defaults must come after fields without defaults
- The decorator can generate `__repr__`, `__eq__`, `__hash__`, `__lt__`, `__le__`, `__gt__`, `__ge__`
- `field()` function provides fine-grained control over individual fields
- `__post_init__` is called automatically at the end of `__init__`

```python
from dataclasses import dataclass, field

@dataclass
class Student:
    name: str
    age: int
    grades: list[int] = field(default_factory=list)  # Mutable default!
    active: bool = True  # Immutable default

s = Student("Alice", 25)
print(s)  # Student(name='Alice', age=25, grades=[], active=True)
```

## Internal working
- `@dataclass` is implemented using the `inspect` module — it reads the class's `__annotations__` to find fields
- It generates the `__init__` method as a string and compiles it using `exec()` or uses pre-built code templates
- The generated methods are set on the class using `setattr`
- Fields are stored in `__dataclass_fields__` as a dict mapping names to `Field` objects
- `Field` objects contain metadata: type, default, default_factory, init, repr, compare, hash, etc.
- `asdict()` and `astuple()` use `deepcopy` internally to return copies
- The decorator supports multiple modes: `init=True`, `repr=True`, `eq=True`, `order=False`, `frozen=False`, `unsafe_hash=False`

```python
# Inspecting dataclass internals
@dataclass
class Person:
    name: str
    age: int = 30

print(Person.__dataclass_fields__)
# {'name': Field(type=str, default=MISSING, ...), 'age': Field(type=int, default=30, ...)}
```

## Real-world analogy
A dataclass is like a standardized form that you fill out. The form has fields (name, age, address) and standard processing:
- **`__init__`**: The form itself — fill in the blanks
- **`__repr__`**: A printed summary of what's on the form
- **`__eq__`**: Checking if two forms have identical information
- **`__hash__`**: A unique filing number for the form
- **`__post_init__`**: A clerk who reviews the form after you fill it out, checking for errors

Without dataclasses, you'd have to hand-write each of these processes for every new form type.

## Examples (with code)

### Basic dataclass
```python
from dataclasses import dataclass

@dataclass
class Book:
    title: str
    author: str
    year: int
    pages: int

book = Book("1984", "George Orwell", 1949, 328)
print(book)               # Book(title='1984', author='George Orwell', year=1949, pages=328)
print(book.title)         # 1984
print(repr(book))         # Book(title='1984', author='George Orwell', year=1949, pages=328)
```

### Default values
```python
from dataclasses import dataclass, field
from typing import List

@dataclass
class Config:
    host: str = "localhost"
    port: int = 8080
    debug: bool = False
    tags: list[str] = field(default_factory=list)  # Mutable default!

c1 = Config()
c2 = Config(host="example.com", port=443, tags=["production"])
```

### frozen=True (immutable)
```python
from dataclasses import dataclass

@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(1.0, 2.0)
# p.x = 3.0  # FrozenInstanceError: cannot assign to field 'x'

# Frozen dataclasses are hashable (can be used in sets/dicts)
point_set = {Point(0, 0), Point(1, 1)}
print(point_set)
```

### order=True (comparison methods)
```python
from dataclasses import dataclass

@dataclass(order=True)
class Employee:
    name: str
    salary: int
    department: str

e1 = Employee("Alice", 70000, "Engineering")
e2 = Employee("Bob", 90000, "Management")
e3 = Employee("Charlie", 70000, "Engineering")

print(e1 < e2)   # True (70000 < 90000)
print(e1 == e3)  # False (different names)

# Sort works automatically
employees = [e1, e2, e3]
for e in sorted(employees):
    print(f"{e.name}: ${e.salary}")
```

### __post_init__ for validation
```python
from dataclasses import dataclass
from typing import List

@dataclass
class Student:
    name: str
    age: int
    grades: list[float]

    def __post_init__(self):
        if self.age < 0:
            raise ValueError("Age cannot be negative")
        if self.age < 5:
            raise ValueError("Student too young")
        for grade in self.grades:
            if not (0 <= grade <= 100):
                raise ValueError(f"Invalid grade: {grade}")

# Valid
s1 = Student("Alice", 15, [85.5, 92.0, 78.5])

# Invalid
try:
    s2 = Student("Bob", -1, [90])
except ValueError as e:
    print(e)  # Age cannot be negative
```

### field() for fine control
```python
from dataclasses import dataclass, field
from typing import List

@dataclass
class InventoryItem:
    name: str
    price: float
    quantity: int = 0
    tags: List[str] = field(default_factory=list)
    sku: str = field(init=False, repr=False)  # Computed, not in __init__, not in __repr__
    internal_id: int = field(repr=False, compare=False)  # Hidden from repr/comparison

    def __post_init__(self):
        self.sku = f"{self.name.upper()}-{id(self):x}"

item = InventoryItem("Widget", 9.99, 10, ["electronic"], internal_id=12345)
print(item)  # InventoryItem(name='Widget', price=9.99, quantity=10, tags=['electronic'])
print(item.sku)  # WIDGET-...
```

### Inheritance
```python
from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int

@dataclass
class Employee(Person):
    employee_id: str
    department: str

emp = Employee("Alice", 30, "E123", "Engineering")
print(emp)  # Employee(name='Alice', age=30, employee_id='E123', department='Engineering')
```

### Converting to/from dict
```python
from dataclasses import dataclass, asdict, astuple

@dataclass
class Product:
    name: str
    price: float
    quantity: int

p = Product("Laptop", 999.99, 5)
print(asdict(p))   # {'name': 'Laptop', 'price': 999.99, 'quantity': 5}
print(astuple(p))  # ('Laptop', 999.99, 5)

# Convert dict back to dataclass
data = {'name': 'Phone', 'price': 499.99, 'quantity': 10}
p2 = Product(**data)
print(p2)  # Product(name='Phone', price=499.99, quantity=10)
```

## Common mistakes

| Mistake | Wrong | Correct |
|---------|-------|---------|
| Mutable default values | `items: list = []` | `items: list = field(default_factory=list)` |
| Forgetting parentheses on `@dataclass` | `@dataclass` (works but no options) | `@dataclass()` or `@dataclass` both OK |
| Order of fields with defaults | `name: str = "default"; age: int` (error) | Fields without defaults must come before fields with defaults |
| Modifying frozen dataclass | `p.x = 5` | Create a new one: `Point(p.x + 1, p.y)` |
| Expecting type enforcement | Can assign wrong type at runtime | Use `mypy` separately |
| Forgetting `__post_init__` namespace | Can't call other methods that require set fields | `__post_init__` can call any method |
| Using `field()` wrong | `items: list = field([])` | Use `field(default_factory=list)` |

## Interview questions + answers

**Q1: What does `@dataclass` auto-generate?**
A: `__init__`, `__repr__`, `__eq__` (and optionally `__hash__`, `__lt__`, `__le__`, `__gt__`, `__ge__`, `__slots__` with Python 3.10+).

**Q2: How do you create a mutable default like an empty list in a dataclass?**
A: Use `field(default_factory=list)`. Never use `[]` directly — it would be shared across instances.

**Q3: What's the difference between `dataclass` and `namedtuple`?**
A: Dataclasses are mutable by default, support inheritance, have `__post_init__`, and integrate with type hints. Namedtuples are immutable tuples with named fields.

**Q4: How do you make a dataclass immutable?**
A: Use `@dataclass(frozen=True)`.

**Q5: What is `__post_init__` used for?**
A: For validation, computed fields, or any initialization logic that runs after `__init__`.

**Q6: How do you exclude a field from `__repr__` or `__init__`?**
A: Use `field(repr=False, init=False)`.

**Q7: Can dataclasses inherit from each other?**
A: Yes. Fields from parent classes are inherited, and `__init__` includes all parent fields too.

**Q8: How do you compare dataclass instances by a specific field?**
A: Use `order=True` for field-by-field comparison (in declaration order), or implement custom comparison methods.

## Knowledge check

**Q1:** What module provides the `dataclass` decorator?
A: `dataclasses`

**Q2:** What attribute stores the field definitions of a dataclass?
A: `__dataclass_fields__`

**Q3:** How do you specify a mutable default value (e.g., an empty list)?
A: `field(default_factory=list)`

**Q4:** True or False: Dataclasses are always immutable.
A: False. They are mutable by default. Use `frozen=True` for immutability.

**Q5:** What method runs automatically after `__init__` in a dataclass?
A: `__post_init__`

**Q6:** What parameter makes a dataclass comparable with `<`, `>`, `<=`, `>=`?
A: `order=True`

**Q7:** How do you convert a dataclass to a dictionary?
A: `dataclasses.asdict(instance)`

**Q8:** Can a field be excluded from the `__init__` method?
A: Yes, with `field(init=False)`.

**Q9:** What happens if you use a mutable default without `default_factory`?
A: All instances share the same mutable object — a bug.

**Q10:** Give an example of a frozen dataclass with a `__post_init__` validation.
A:
```python
@dataclass(frozen=True)
class Temperature:
    celsius: float

    def __post_init__(self):
        if self.celsius < -273.15:
            raise ValueError("Below absolute zero!")
```

## Revision notes

- `@dataclass` auto-generates `__init__`, `__repr__`, `__eq__` from annotated fields
- Fields defined with type annotations: `name: str`
- Mutable defaults require `field(default_factory=...)` — never `= []` or `= {}`
- `frozen=True` makes instances immutable (and hashable)
- `order=True` generates comparison methods (`<`, `<=`, `>`, `>=`)
- `__post_init__` runs after `__init__`, useful for validation/computed fields
- `field()` controls per-field behavior: `init`, `repr`, `compare`, `hash`, `default`, `default_factory`
- `asdict()` and `astuple()` convert to dictionaries/tuples
- Inheritance works naturally with dataclasses
- Type hints are not enforced at runtime — use `mypy` for that
- Stored in `__dataclass_fields__` dict
