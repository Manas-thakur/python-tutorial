# Enums

## Definition
An **enum** (enumeration) is a set of symbolic names bound to unique, constant values. Python's `Enum` class from the `enum` module provides a way to create enumerated constants with type safety and structured member access.

```python
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

print(Color.RED)          # Color.RED
print(type(Color.RED))    # <enum 'Color'>
print(Color.RED.name)     # 'RED'
print(Color.RED.value)    # 1
```

## Why it exists
- **Named constants**: Replace magic numbers/strings with readable, self-documenting names
- **Type safety**: Only valid enum members can be used  -  prevents invalid values
- **Grouping**: Related constants are organized within a class namespace
- **Iteration**: Easy to loop over all possible values
- **Comparisons**: Identity (`is`) and equality (`==`) work intuitively
- **Mapping**: Bidirectional lookup by name and by value
- **Prevent typos**: `Color.REDD` raises `AttributeError`  -  catches mistakes early
- **Maintainability**: Add/remove constants in one place; changes propagate automatically

## How it works
- Define a class that inherits from `Enum`
- Class attributes become enum members  -  each is an instance of the enum class
- Members are **single-ton** objects  -  only one instance of each member exists
- `name` attribute stores the member name (string); `value` stores the assigned value
- Members can be accessed by name (`Color.RED`), by value (`Color(1)`), or by name string (`Color['RED']`)
- Enums are final by default  -  you cannot create new instances outside the defined members

```python
from enum import Enum

class Status(Enum):
    PENDING = "pending"
    ACTIVE = "active"
    INACTIVE = "inactive"

# Three ways to access members
print(Status.PENDING)             # Status.PENDING
print(Status("pending"))          # Status.PENDING  (by value)
print(Status['PENDING'])          # Status.PENDING  (by name)

# Members are singletons
s1 = Status.PENDING
s2 = Status.PENDING
print(s1 is s2)  # True
```

## Internal working
- `Enum` uses Python's metaclass machinery (`EnumMeta`) to create enum classes
- When the class body executes, each attribute is collected as an enum member
- `EnumMeta.__new__` converts each attribute into an instance of the enum class
- Members are stored as class attributes AND in `_member_map_` (dict) and `_value2member_map_` (dict for reverse lookup)
- `__call__` on the enum class (e.g., `Color(1)`) does a value lookup in `_value2member_map_`
- `__getitem__` (e.g., `Color['RED']`) does a name lookup in `_member_map_`
- `Enum` guarantees uniqueness of values by default (raises `ValueError` for duplicates unless using `@unique`)
- Members have `__members__` attribute that provides an ordered mapping of name → member

```python
from enum import Enum

class Direction(Enum):
    NORTH = 1
    SOUTH = 2
    EAST = 3
    WEST = 4

# Internal data structures
print(Direction._member_map_)
# {'NORTH': <Direction.NORTH: 1>, 'SOUTH': <Direction.SOUTH: 2>, ...}

print(Direction._value2member_map_)
# {1: <Direction.NORTH: 1>, 2: <Direction.SOUTH: 2>, ...}

print(Direction.__members__)
# OrderedDict([('NORTH', <Direction.NORTH: 1>), ...])
```

## Real-world analogy
An enum is like a restaurant menu. The menu defines fixed options (appetizers, mains, desserts), each with a name and a price. You can choose only what's on the menu  -  no custom items. The dish names must be spelled correctly to order. Similarly, enums define a fixed set of allowed values, and you must use them exactly as defined. You can look up a dish by name (enum name), by menu number (enum value), or iterate the entire menu.

### Without enum (bad):
```python
# Magic strings  -  easy to typo
order_status = "pending"  # or "pendng"? "pendin"? No type safety
```

### With enum (good):
```python
order_status = Status.PENDING  # IDE autocompletion, type check safety
```

## Examples (with code)

### Basic enum
```python
from enum import Enum

class Season(Enum):
    SPRING = 1
    SUMMER = 2
    FALL = 3
    WINTER = 4

# Access
print(Season.SPRING)          # Season.SPRING
print(Season.SPRING.name)     # 'SPRING'
print(Season.SPRING.value)    # 1

# Comparison
print(Season.SPRING == Season.SPRING)  # True
print(Season.SPRING is Season.SPRING)  # True
print(Season.SPRING == 1)             # False (different types!)
print(Season.SPRING.value == 1)       # True

# Iteration
for s in Season:
    print(f"{s.name}: {s.value}")
# SPRING: 1, SUMMER: 2, FALL: 3, WINTER: 4

# Lookup
print(Season(1))      # Season.SPRING  (by value)
print(Season['WINTER'])  # Season.WINTER  (by name)
```

### Using string values
```python
from enum import Enum

class HttpMethod(Enum):

print(HttpMethod.GET)           # HttpMethod.GET
```

### IntEnum  -  enums that behave like integers
```python
from enum import IntEnum

class Priority(IntEnum):
    LOW = 1
    MEDIUM = 2
    HIGH = 3

# IntEnum members are also integers!
print(Priority.HIGH)           # Priority.HIGH
print(Priority.HIGH > 2)       # True  (int comparison works!)
print(Priority.HIGH == 3)      # True
print(Priority.MEDIUM + 1)     # 3
print(Priority.LOW < Priority.HIGH)  # True
```

### StrEnum (Python 3.11+)
```python
from enum import StrEnum

class Language(StrEnum):
    PYTHON = "python"
    JAVA = "java"
    RUST = "rust"

# StrEnum members are also strings
print(Language.PYTHON.upper())  # 'PYTHON'
print(f"Language: {Language.RUST}")  # 'Language: rust'
```

### auto()  -  automatic values
```python
from enum import Enum, auto

class Color(Enum):
    RED = auto()
    GREEN = auto()
    BLUE = auto()

print(list(Color))
# [<Color.RED: 1>, <Color.GREEN: 2>, <Color.BLUE: 3>]

class Status(Enum):
    PENDING = auto()
    ACTIVE = auto()
    INACTIVE = auto()

for s in Status:
    print(f"{s.name}: {s.value}")
# PENDING: 1, ACTIVE: 2, INACTIVE: 3
```

### @unique decorator
```python
from enum import Enum, unique

@unique
class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3
    # CRIMSON = 1  # ValueError: duplicate value found!

# Without @unique, duplicate values create aliases
class ColorAliased(Enum):
    RED = 1
    CRIMSON = 1  # This is allowed  -  CRIMSON is an alias for RED

print(ColorAliased(1))  # ColorAliased.RED (first one wins)
```

### Enum with methods
```python
from enum import Enum

class Planet(Enum):
    MERCURY = (0.38, 4879)
    VENUS = (0.91, 12104)
    EARTH = (1.00, 12756)
    MARS = (0.38, 6792)

    def __init__(self, gravity_ratio, diameter_km):
        self.gravity_ratio = gravity_ratio
        self.diameter_km = diameter_km
    
    def weight_on(self, earth_weight):
        """Calculate weight on this planet."""
        return earth_weight * self.gravity_ratio

earth_weight = 70  # kg
for planet in Planet:
    print(f"On {planet.name}: {planet.weight_on(earth_weight):.1f} kg")
```

### Enum for HTTP status codes (mini project concept)
```python
from enum import IntEnum

class HttpStatus(IntEnum):
    # 1xx Informational
    CONTINUE = 100
    SWITCHING_PROTOCOLS = 101
    
    # 2xx Success
    OK = 200
    CREATED = 201
    ACCEPTED = 202
    NO_CONTENT = 204
    
    # 3xx Redirection
    MOVED_PERMANENTLY = 301
    FOUND = 302
    NOT_MODIFIED = 304
    
    # 4xx Client Error
    BAD_REQUEST = 400
    UNAUTHORIZED = 401
    FORBIDDEN = 403
    NOT_FOUND = 404
    METHOD_NOT_ALLOWED = 405
    CONFLICT = 409
    TEAPOT = 418  # I'm a teapot
    
    # 5xx Server Error
    INTERNAL_SERVER_ERROR = 500
    NOT_IMPLEMENTED = 501
    BAD_GATEWAY = 502
    SERVICE_UNAVAILABLE = 503

    @property
    def category(self):
        if 100 <= self.value < 200:
            return "Informational"
        elif 200 <= self.value < 300:
            return "Success"
        elif 300 <= self.value < 400:
            return "Redirection"
        elif 400 <= self.value < 500:
            return "Client Error"
        else:
            return "Server Error"

print(HttpStatus.NOT_FOUND)          # HttpStatus.NOT_FOUND
print(HttpStatus.NOT_FOUND.value)    # 404
print(HttpStatus(200))               # HttpStatus.OK
print(HttpStatus.OK.category)        # Success
```

### Enum for days of week
```python
from enum import Enum, auto

class Weekday(Enum):
    MONDAY = auto()
    TUESDAY = auto()
    WEDNESDAY = auto()
    THURSDAY = auto()
    FRIDAY = auto()
    SATURDAY = auto()
    SUNDAY = auto()

    def is_weekend(self):
        return self in (Weekday.SATURDAY, Weekday.SUNDAY)
    
    def next_day(self):
        members = list(Weekday)
        idx = members.index(self)
        return members[(idx + 1) % len(members)]

today = Weekday.WEDNESDAY
print(f"{today.name}, weekend? {today.is_weekend()}")
print(f"Next day: {today.next_day().name}")
```

## Common mistakes

| Mistake | Wrong | Correct |
|---------|-------|---------|
| Comparing enum to raw value | `Color.RED == 1` (False) | `Color.RED.value == 1` or use `IntEnum` |
| Treating enum as iterable class | `for x in Color:`  -  works (iterates members) | Fine, but remember `Color` is the class, not a collection |
| Creating new instances | `Color(5)` with value 5 not defined | Raises `ValueError` |
| Duplicate values (without `@unique`) | Two members with same value  -  second is alias | Use `@unique` to prevent |
| Modifying enum class at runtime | `Color.ORANGE = 5` | Not allowed (enum metaclass prevents this) |
| Using `Enum` for simple string constants | `class S(Enum): A = "a"` | Can be overkill for simple constants |

## Interview questions + answers

**Q1: What's the difference between `Enum` and `IntEnum`?**
A: `IntEnum` members are also `int` subclasses, so they can be used anywhere integers are expected. Regular `Enum` members are not integers (even if values are ints).

**Q2: How do you ensure no two enum members have the same value?**
A: Use the `@unique` decorator from `enum`.

**Q3: How do you iterate over all enum members?**
A: `for member in MyEnum: print(member)` or `for name, member in MyEnum.__members__.items():`

**Q4: What is `auto()` in enums?**
A: It automatically assigns incrementing integer values starting from 1.

**Q5: Can an enum have methods?**
A: Yes. Enums can have methods, properties, and even `__init__` for custom initialization.

**Q6: Are enum members singletons?**
A: Yes. Only one instance of each member exists. Use `is` for comparison: `Color.RED is Color.RED`.

**Q7: How do you create an enum with string values that behave like strings?**
A: Use `StrEnum` (Python 3.11+) or subclass both `str` and `Enum`: `class S(str, Enum): A = "a"`.

**Q8: When should you use an enum vs a set of constants?**
A: Use enums when you want type safety, iteration, grouping, and name→value lookups. Use simple constants for very simple cases or when C-level performance matters.

## Knowledge check

**Q1:** What module provides enum support?
A: `enum`

**Q2:** How do you get the name of an enum member?
A: `.name` attribute (e.g., `Color.RED.name`)

**Q3:** How do you get the value of an enum member?
A: `.value` attribute (e.g., `Color.RED.value`)

**Q4:** What does `Color(2)` do if `Color` has `GREEN = 2`?
A: Returns `Color.GREEN`

**Q5:** True or False: `Color.RED == 1` returns True.
A: False for regular `Enum`, True for `IntEnum`.

**Q6:** What decorator prevents duplicate enum values?
A: `@unique`

**Q7:** What does `auto()` return by default?
A: `1`, `2`, `3`, ... (incrementing integers)

**Q8:** Can you add a new member to an enum at runtime?
A: No. Enum classes are immutable after creation.

**Q9:** What's the output of `for m in Weekday: print(m.name)`?
A: All weekday names in declaration order.

**Q10:** How do you look up an enum member by its name string?
A: `Color['RED']`  -  uses `__getitem__` on the enum class.

## Revision notes

- `class MyEnum(Enum):` defines an enumeration; attributes become constant members
- Members have `.name` (string) and `.value` (assigned value)
- Access: `MyEnum.MEMBER`, `MyEnum(value)`, `MyEnum['MEMBER']`
- Members are **singletons**  -  use `is` for identity comparison
- `IntEnum` members behave as integers (comparable, mathematical operations)
- `StrEnum` members behave as strings (Python 3.11+)
- `auto()` auto-assigns incrementing integer values
- `@unique` prevents duplicate values
- Enums are iterable: `for member in MyEnum:`
- Enums can have methods, `__init__`, properties
- Internal storage: `_member_map_` (name→member), `_value2member_map_` (value→member)
- Prevents invalid values  -  type-safe alternative to magic strings/numbers
