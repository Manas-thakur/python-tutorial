# Iterators

## Definition
An **iterator** is an object that represents a stream of data. It implements the **iterator protocol** consisting of two methods: `__iter__()` (returns itself) and `__next__()` (returns the next element or raises `StopIteration`). An **iterable** is any object that can return an iterator via `iter()`.

```python
# An iterator in action
nums = [1, 2, 3]          # list is iterable
it = iter(nums)            # get iterator
print(next(it))            # 1
print(next(it))            # 2
print(next(it))            # 3
print(next(it))            # StopIteration!
```

## Why it exists
- **Memory efficiency**: Iterators produce values one at a time, not all at once (lazy evaluation)
- **Unified interface**: The `for` loop works the same way for lists, files, databases, infinite sequences
- **Composability**: Iterators can be chained, filtered, and transformed without materializing intermediate results
- **Infinite sequences**: Possible to represent endless data streams (e.g., sensor readings, counters)
- **Abstraction**: Separate the "how to iterate" from "what to do with each item"

## How it works
- An **iterable** has `__iter__()` that returns an iterator
- An **iterator** has `__iter__()` (returns `self`) and `__next__()` (returns next value or raises `StopIteration`)
- The `for` loop calls `iter()` on the iterable, then repeatedly calls `next()` until `StopIteration`
- Built-in functions `iter()` and `next()` provide the public API

```python
# How a for loop works internally
iterable = [1, 2, 3]
iterator = iter(iterable)   # calls __iter__()
while True:
    try:
        item = next(iterator)  # calls __next__()
        print(item)
    except StopIteration:
        break
# Output: 1 2 3
```

## Internal working
- CPython implements `iter()` by calling `PyObject_GetIter()`, which checks for `__iter__()` method
- If `__iter__()` is not found, it falls back to `__getitem__()` (old-style iteration with integer indices)
- `next()` is implemented via `PyIter_Next()` which calls the C-level `tp_iternext` slot
- `StopIteration` is a `BaseException` subclass — it signals the end of iteration without being treated as an error
- The `FOR_ITER` opcode handles iteration inside loops at the bytecode level
- Custom iterators in pure Python are slower than built-in ones due to the interpreter overhead of each `__next__()` call

```python
# Old-style iterable via __getitem__ (fallback protocol)
class OldStyleIterable:
    def __init__(self, data):
        self.data = data
    
    def __getitem__(self, index):
        if index >= len(self.data):
            raise IndexError
        return self.data[index]

# Works with for loop even without __iter__!
for item in OldStyleIterable([10, 20, 30]):
    print(item)
```

## Real-world analogy
An iterator is like a bookmark in a book. The book itself is **iterable** (you can get a bookmark for it). The bookmark is the **iterator** — it remembers your position, can tell you what page you're on, and can move to the next page. When you reach the last page, the bookmark says "no more pages" (StopIteration). You can have multiple bookmarks (iterators) in the same book (iterable), each at different positions.

## Examples (with code)

### Iterable vs Iterator
```python
# Most collections are iterable but NOT iterators
my_list = [1, 2, 3]
print(hasattr(my_list, '__iter__'))   # True
print(hasattr(my_list, '__next__'))   # False — not an iterator

# Get an iterator from an iterable
it = iter(my_list)
print(hasattr(it, '__iter__'))   # True
print(hasattr(it, '__next__'))   # True — is an iterator

# Exhausting an iterator
print(list(it))  # [1, 2, 3]
print(list(it))  # [] — iterator is exhausted!
```

### Creating a custom iterator
```python
class RangeIterator:
    """Custom iterator that behaves like range()."""
    
    def __init__(self, start, stop, step=1):
        self.current = start
        self.stop = stop
        self.step = step
    
    def __iter__(self):
        return self  # iterator returns itself
    
    def __next__(self):
        if self.current >= self.stop:
            raise StopIteration
        value = self.current
        self.current += self.step
        return value

# Usage
for num in RangeIterator(0, 10, 2):
    print(num)  # 0, 2, 4, 6, 8

# Or manually
it = RangeIterator(0, 3)
print(next(it))  # 0
print(next(it))  # 1
print(next(it))  # 2
# print(next(it))  # StopIteration
```

### Custom iterable (separate iterator class)
```python
class Playlist:
    """Iterable collection of songs."""
    
    def __init__(self, name):
        self.name = name
        self.songs = []
    
    def add_song(self, song):
        self.songs.append(song)
    
    def __iter__(self):
        return PlaylistIterator(self)

class PlaylistIterator:
    """Iterator for Playlist."""
    
    def __init__(self, playlist):
        self._playlist = playlist
        self._index = 0
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self._index >= len(self._playlist.songs):
            raise StopIteration
        song = self._playlist.songs[self._index]
        self._index += 1
        return song

# Usage
my_playlist = Playlist("Favorites")
my_playlist.add_song("Bohemian Rhapsody")
my_playlist.add_song("Stairway to Heaven")
my_playlist.add_song("Hotel California")

for song in my_playlist:
    print(f"Playing: {song}")
```

### Using iter() with sentinel
```python
# iter(callable, sentinel) — calls callable until sentinel is returned
import random

def coin_flip():
    return random.choice(["H", "T"])

# Keep flipping until we get heads
results = list(iter(coin_flip, "H"))
print(results)  # e.g., ['T', 'T', 'T'] (no 'H' because sentinel stops before it)
```

### Reading files with iterators
```python
# Files are iterable (and their own iterators)
with open("data.txt", "w") as f:
    f.write("line1\nline2\nline3\n")

with open("data.txt") as f:
    for line in f:  # iterator reads one line at a time — memory efficient
        print(repr(line))

# Manual iteration
with open("data.txt") as f:
    it = iter(f)
    print(next(it))  # 'line1\n'
    print(next(it))  # 'line2\n'
```

### Multiple independent iterators
```python
nums = [1, 2, 3, 4, 5]

it1 = iter(nums)
it2 = iter(nums)

print(next(it1))  # 1
print(next(it1))  # 2
print(next(it2))  # 1 — independent! separate bookmark
print(next(it1))  # 3
```

### itertools — the iterator toolbox
```python
from itertools import count, cycle, repeat, chain, islice, takewhile

# Infinite counter
for i in count(10, 5):
    if i > 30:
        break
    print(i)  # 10, 15, 20, 25, 30

# Cycle
colors = cycle(["red", "green", "blue"])
for _ in range(6):
    print(next(colors))  # red, green, blue, red, green, blue

# Chain — combine iterators
combined = chain([1, 2, 3], [4, 5, 6])
print(list(combined))  # [1, 2, 3, 4, 5, 6]

# islice — slice an iterator lazily
data = count(1)
sliced = islice(data, 5, 10)
print(list(sliced))  # [5, 6, 7, 8, 9]
```

## Common mistakes

| Mistake | Wrong | Correct |
|---------|-------|---------|
| Reusing an exhausted iterator | `it = iter([1,2]); list(it); list(it)` second gives `[]` | Get a fresh iterator |
| Modifying iterable during iteration | `for x in lst: lst.remove(x)` | Iterate over a copy: `for x in lst[:]` |
| Forgetting `iter()` on iterable | Has `__iter__` but calling `next()` directly | Call `iter()` first to get iterator |
| Treating iterators as sequences | `it[2]` on an iterator | Convert to list first or use `itertools.islice` |
| Assuming iterator has length | `len(iterator)` | Count manually or convert to list |
| Infinite loop with iterator | `for x in infinite_iterator:` without `break` | Use `itertools.islice` or a break condition |
| Not implementing `__iter__` for iterator class | Iterator class missing `__iter__` | Should return `self` from `__iter__` |

## Interview questions + answers

**Q1: What's the difference between an iterable and an iterator?**
A: An iterable has `__iter__()` that returns an iterator. An iterator has both `__iter__()` (returns self) and `__next__()` (returns next item or raises `StopIteration`). All iterators are iterables, but not all iterables are iterators.

**Q2: How does a `for` loop work internally?**
A: It calls `iter()` on the iterable to get an iterator, then repeatedly calls `next()` on it, catching `StopIteration` to exit gracefully.

**Q3: Can you iterate over the same iterator twice?**
A: No. An iterator is a one-time use object. Once exhausted, it's done. Create a new iterator from the iterable.

**Q4: What happens when `next()` is called on an exhausted iterator?**
A: It raises `StopIteration`.

**Q5: What is `iter(callable, sentinel)` used for?**
A: It calls `callable` repeatedly until it returns `sentinel`, creating an iterator from the results.

**Q6: What is lazy evaluation?**
A: Values are computed on-demand, one at a time, rather than all at once. This saves memory and allows infinite sequences.

**Q7: How do you convert an iterator to a list?**
A: `list(iterator)`. But be careful with infinite iterators.

**Q8: What's the difference between `__getitem__` and `__iter__` for iteration?**
A: Python first tries `__iter__`. If absent, it falls back to `__getitem__` with 0-based integer indices, catching `IndexError` to stop.

## Knowledge check

**Q1:** What two methods define the iterator protocol?
A: `__iter__()` and `__next__()`

**Q2:** What exception stops iteration?
A: `StopIteration`

**Q3:** True or False: A list is an iterator.
A: False. A list is iterable, but not an iterator (it has `__iter__` but not `__next__`).

**Q4:** What does `next(it)` return when `it` is exhausted?
A: It raises `StopIteration` (does not return a value).

**Q5:** How do you get an iterator from an iterable?
A: Call `iter(iterable)`.

**Q6:** What are the two forms of the `iter()` built-in function?
A: `iter(iterable)` and `iter(callable, sentinel)`.

**Q7:** What's wrong with `for x in open("file"): content = open("file").read()`?
A: Nothing technically wrong, but it reopens the file inefficiently instead of using the open file handle.

**Q8:** Can two iterators from the same list have independent positions?
A: Yes. Each iterator maintains its own state/position.

**Q9:** What does `list(iter([1, 2, 3]))` return?
A: `[1, 2, 3]`

**Q10:** How do you make an iterator that produces values forever?
A: Have `__next__()` never raise `StopIteration`.

## Revision notes

- **Iterable**: Has `__iter__()` → returns an iterator (e.g., list, tuple, dict, str, file)
- **Iterator**: Has `__iter__()` → returns self, and `__next__()` → returns next value or raises `StopIteration`
- `for` loop = `iter()` + `next()` + catch `StopIteration`
- Iterators are **one-time use** and **lazy** (compute on demand)
- `iter(iterable)` gets an iterator; `next(iterator)` gets next value
- Custom iterators implement `__iter__` and `__next__`
- `__getitem__` provides fallback iteration (old-style protocol)
- `itertools` provides powerful iterator tools: `count`, `cycle`, `chain`, `islice`, `takewhile`
- Iterators save memory by not materializing all values at once
- Modifying a collection while iterating over it is dangerous
