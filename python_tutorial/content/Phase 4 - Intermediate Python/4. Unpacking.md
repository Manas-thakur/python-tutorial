# Unpacking

## Definition
Unpacking (also called destructuring) is the process of splitting iterables and dictionaries into individual variables. Python provides powerful unpacking syntax using `*` for iterables and `**` for dictionaries.

```python
# Basic tuple unpacking
a, b = (1, 2)
print(a)  # 1
print(b)  # 2

# Extended unpacking with *
first, *rest = [1, 2, 3, 4]
print(first)  # 1
print(rest)   # [2, 3, 4]
```

## Why it exists
- **Cleaner code**: Extract values from collections without indexing
- **Variable swapping**: Elegant `a, b = b, a` syntax
- **Function argument handling**: `func(*args)` and `func(**kwargs)` calling patterns
- **Data transformation**: Restructure data easily during assignment
- **Return multiple values**: Functions can return tuples that are unpacked at the call site
- **Readability**: Express intent clearly  -  "split this collection into these named parts"

## How it works
- Python matches the number of variables on the left to elements on the right
- `*` packs remaining elements into a **list** (not tuple) in extended unpacking
- `**` unpacks dictionaries into keyword arguments (in calls) or merges dicts (in literals)
- Unpacking works with any iterable (lists, tuples, strings, sets, generators, dicts)
- The `*` expression can appear in any position: `first, *middle, last = items`

```python
# Works with any iterable
a, b, c = "ABC"
print(a, b, c)  # A B C

# * in any position
first, *middle, last = [1, 2, 3, 4, 5]
print(first)    # 1
print(middle)   # [2, 3, 4]
print(last)     # 5

*beginning, final = range(10)
print(beginning)  # [0, 1, 2, 3, 4, 5, 6, 7, 8]
print(final)      # 9
```

## Internal working
- The CPython interpreter uses the `UNPACK_SEQUENCE` opcode for normal unpacking
- For extended unpacking (with `*`), it uses `UNPACK_EX` opcode
- `UNPACK_SEQUENCE` checks that the number of elements matches exactly (raises `ValueError` otherwise)
- `UNPACK_EX` splits the iterable into: elements before `*`, the `*` collection, and elements after `*`
- For dict unpacking with `**` in calls, Python uses `BUILD_MAP_UNPACK_WITH_CALL`
- In Python 3.9+, `**` can merge dicts: `merged = {**dict1, **dict2}`

```python
# What happens internally is roughly:
# a, b = (1, 2) 
# → check length == 2
# → a = iterable[0], b = iterable[1]

# a, *b = [1, 2, 3]
# → split at *: before=[a], star=[b], after=[]
# → a = iterable[0], b = list(iterable[1:])
```

## Real-world analogy
Unpacking is like opening a package that contains several items and placing each item into its own labeled drawer. Extended unpacking (`*`) is like saying "put the first item in drawer A, the last item in drawer C, and everything else in drawer B." The `**` dict unpacking is like merging two filing cabinets  -  you take all folders from both cabinets and put them into one combined cabinet.

## Examples (with code)

### Basic tuple and list unpacking
```python
# Tuple unpacking
point = (3, 7)
x, y = point
print(f"x={x}, y={y}")  # x=3, y=7

# List unpacking
rgb = [255, 128, 0]
red, green, blue = rgb
print(f"RGB({red}, {green}, {blue})")

# Ignoring values with underscore
name, _, score = ("Alice", "Grade 5", 95)
print(f"{name}: {score}")  # Alice: 95
```

### Swapping variables
```python
a = 10
b = 20

a, b = b, a  # Swap in one line
print(a, b)  # 20 10

# How it works: right side is evaluated first (20, 10)
# then unpacked into a and b
```

### Extended unpacking (`*`)
```python
# Head and tail
head, *tail = [1, 2, 3, 4, 5]
print(head)  # 1
print(tail)  # [2, 3, 4, 5]

# * in the middle
start, *middle, end = [10, 20, 30, 40, 50]
print(start)   # 10
print(middle)  # [20, 30, 40]
print(end)     # 50

# Only * (collect all)
*all_items, = [1, 2, 3]
print(all_items)  # [1, 2, 3]

# Multiple * unpacking in nested structures
items = [(1, "a"), (2, "b"), (3, "c")]
for num, letter in items:
    print(f"{num}: {letter}")
```

### String unpacking
```python
name = "Python"
p, y, t, h, o, n = name
print(p, y, t, h, o, n)  # P y t h o n

first, *rest = "Hello!"
print(first)  # H
print(''.join(rest))  # ello!
```

### Dictionary unpacking (`**`)
```python
# Merging dictionaries (Python 3.5+)
d1 = {"a": 1, "b": 2}
d2 = {"c": 3, "d": 4}
merged = {**d1, **d2}
print(merged)  # {'a': 1, 'b': 2, 'c': 3, 'd': 4}

# Override keys (later overrides earlier)
d3 = {"a": 100, "b": 200}
d4 = {"b": 999, "c": 300}
merged2 = {**d3, **d4}
print(merged2)  # {'a': 100, 'b': 999, 'c': 300}

# Adding keys while merging
config = {**d1, "extra": 42}
print(config)  # {'a': 1, 'b': 2, 'extra': 42}
```

### Unpacking in function calls
```python
def divide(a, b):
    return a / b

# Unpack list as positional args
args = [10, 3]
print(divide(*args))  # 3.333...

# Unpack dict as keyword args
kwargs = {"a": 20, "b": 4}
print(divide(**kwargs))  # 5.0

# Combined
def connect(host, port, timeout=30):
    print(f"{host}:{port} (timeout={timeout}s)")

connect(**{"host": "localhost", "port": 8080, "timeout": 60})
```

### Nested unpacking
```python
# Unpack nested structures
data = [1, (2, 3), 4]
a, (b, c), d = data
print(a, b, c, d)  # 1 2 3 4

# Deeply nested
nested = {"point": [3, 7], "label": "A"}
_, (x, y) = nested.items()  # Hmm, tricky
print(x, y)

# Unpacking in for loops
pairs = [(1, 'a'), (2, 'b'), (3, 'c')]
for num, char in pairs:
    print(f"{num}: {char}")

# Unpacking generator expressions
squares = [(x, x**2) for x in range(5)]
for n, sq in squares:
    print(f"{n}² = {sq}")
```

### Using * for remaining items
```python
# Processing log lines
log_line = "ERROR 2024-01-15 Disk full on /dev/sda1"
level, date, *message = log_line.split()
print(f"Level: {level}")     # ERROR
print(f"Date: {date}")       # 2024-01-15
print(f"Message: {message}")  # ['Disk', 'full', 'on', '/dev/sda1']
print(full_msg = ' '.join(message))  # Disk full on /dev/sda1
```

## Common mistakes

| Mistake | Wrong | Correct |
|---------|-------|---------|
| Mismatch in unpacking count | `a, b = [1, 2, 3]` | `a, b, c = [1, 2, 3]` or `a, *b = [1, 2, 3]` |
| Using * with dict (iterates keys) | `a, b = *{"x": 1, "y": 2}` (actually errors) | `a, b = {"x": 1, "y": 2}` gives `"x", "y"` |
| Only one * allowed per level | `*a, *b = [1, 2, 3]` (SyntaxError) | Use slicing instead |
| Forgetting that * produces a list | `*head, last = (1, 2, 3)` then `head` is `[1, 2]` (not tuple) | It's a list  -  documented behavior |
| Trying to unpack generator infinitely | `a, b = itertools.count()` | Works for small generators but will hang on infinite |
| * on left side of tuple without parentheses | `a, *b = 1, 2, 3` is fine | Actually works! Python infers tuple |

## Interview questions + answers

**Q1: What's the output of `a, b, *c = [1, 2]`?**
A: `a=1, b=2, c=[]`  -  `*c` captures an empty list when there are no remaining items.

**Q2: How does `a, b = b, a` swap work internally?**
A: The right side evaluates first, creating a tuple `(b, a)`, then unpacks into `a` and `b`.

**Q3: Can you use multiple `*` in one unpacking?**
A: No. `*a, *b = [1, 2, 3]` raises `SyntaxError`. Only one starred expression per assignment level.

**Q4: What's the difference between `*args` in function def vs function call?**
A: In def, `*args` collects arguments into a tuple. In call, `*iterable` unpacks an iterable into positional args.

**Q5: How do you merge two dictionaries?**
A: `merged = {**dict1, **dict2}` (Python 3.5+) or `merged = dict1 | dict2` (Python 3.9+).

**Q6: What does `first, *_, last = items` do?**
A: Unpacks the first element, ignores the middle (conventionally), and assigns the last element.

**Q7: What type is `rest` in `first, *rest = (1, 2, 3)`?**
A: `rest` is always a `list`, even when unpacking a tuple.

**Q8: How do you unpack a nested list like `[1, [2, 3], 4]`?**
A: `a, (b, c), d = [1, [2, 3], 4]`

## Knowledge check

**Q1:** What does `a, b = 10, 20` assign to `a` and `b`?
A: a=10, b=20

**Q2:** What's the output of `x, y, z = "Hi"`?
A: `ValueError: not enough values to unpack`  -  3 variables but 2 values.

**Q3:** What does `*mid, last = [1, 2, 3, 4]` produce?
A: `mid=[1, 2, 3]`, `last=4`

**Q4:** Use unpacking to swap `x = 5` and `y = 10`.
A: `x, y = y, x`

**Q5:** What does `first, *_, last = "Python"` give?
A: `first='P'`, `last='n'`

**Q6:** How do you merge `d1 = {"a": 1}` and `d2 = {"b": 2}` into one dict?
A: `{**d1, **d2}` or `d1 | d2`

**Q7:** What does `{**{"x": 1}, "x": 2}` produce?
A: `{'x': 2}`  -  later values override earlier ones.

**Q8:** True or False: `*a, *b = [1, 2, 3]` is valid Python.
A: False. Only one starred expression per unpacking level.

**Q9:** What type does `*_` capture as?
A: list

**Q10:** Unpack `(1, 2, 3, 4, 5)` to get first, last, and middle elements.
A: `first, *middle, last = (1, 2, 3, 4, 5)`

## Revision notes

- Basic unpacking: `a, b = (1, 2)`  -  left side matches right side element count
- Extended unpacking with `*`: `first, *rest = iterable`  -  `rest` is always a **list**
- `*` can appear in any position: `first, *mid, last`
- Swapping: `a, b = b, a`  -  a Pythonic idiom
- Dict unpacking with `**`: `{**d1, **d2}` merges dicts; later keys override
- In function calls: `func(*iterable)` and `func(**dict)` 
- Nested unpacking: `a, (b, c) = [1, (2, 3)]`
- Only one `*` per level in assignment
- Works with any iterable (strings, tuples, lists, generators)
- `*_` ignores middle items by convention
