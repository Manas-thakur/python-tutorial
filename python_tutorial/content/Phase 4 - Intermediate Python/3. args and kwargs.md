# *args and **kwargs

## Definition
`*args` and `**kwargs` allow functions to accept a variable number of arguments. `*args` captures extra **positional** arguments as a **tuple**, while `**kwargs` captures extra **keyword** arguments as a **dictionary**.

```python
def func(*args, **kwargs):
    print(args)   # tuple of positional args
    print(kwargs) # dict of keyword args

func(1, 2, 3, name="Alice", age=25)
# (1, 2, 3)
# {'name': 'Alice', 'age': 25}
```

## Why it exists
- **Flexibility**: Create APIs that handle varying numbers of arguments gracefully
- **Wrapper functions**: Pass through arguments to another function without knowing them in advance (used extensively in decorators)
- **Unpacking**: Split iterables and dictionaries into function arguments
- **Polymorphism**: Handle different call signatures with the same function
- **Subclassing**: Override methods while accepting the same arguments as the parent
- **Logging and configuration**: Accept arbitrary metadata with `**kwargs`

## How it works
- `*args` collects all **extra positional** arguments after the regular parameters into a tuple
- `**kwargs` collects all **extra keyword** arguments that don't match parameter names into a dict
- The names `args` and `kwargs` are conventions  -  only the `*` and `**` matter syntactically
- `*args` must come before `**kwargs` in the function signature
- Regular parameters, `*args`, `**kwargs` can be combined, but order matters

```python
def example(a, b, *args, **kwargs):
    print(f"a={a}, b={b}")
    print(f"args={args}")
    print(f"kwargs={kwargs}")

example(1, 2, 3, 4, 5, x=10, y=20)
# a=1, b=2
# args=(3, 4, 5)
# kwargs={'x': 10, 'y': 20}
```

## Internal working
- When Python calls a function with `*args` in the signature, the interpreter:
  1. Binds regular parameter names to passed positional arguments
  2. Packs any remaining positional arguments into a tuple assigned to the `args` variable
  3. Binds keyword parameters
  4. Packs remaining keyword arguments into a dict assigned to the `kwargs` variable
- For `**kwargs`, Python uses the `PyDict` type to store key-value pairs
- The actual mechanism involves the `LOAD_BUILD_CLASS`, `CALL_FUNCTION_EX` opcodes in CPython
- Unpacking (`func(*list, **dict)` uses `BUILD_TUPLE_UNPACK_WITH_CALL` and `BUILD_MAP_UNPACK_WITH_CALL` opcodes)

```python
# Unpacking in function calls
def add(a, b, c):
    return a + b + c

numbers = [1, 2, 3]
print(add(*numbers))  # 6  -  unpacks list elements

data = {"a": 10, "b": 20, "c": 30}
print(add(**data))    # 60  -  unpacks dict as keyword args
```

## Real-world analogy
Think of `*args` like a "miscellaneous items" bin at an airport security checkpoint. Regular luggage (parameters) goes through normally, but anything extra (additional items) gets tossed into the bin (`*args`). `**kwargs` is like a form with a "additional notes" section  -  you've filled in all the standard fields, but there's a catch-all field for anything else you want to specify.

## Examples (with code)

### Basic *args
```python
def sum_all(*args):
    total = 0
    for num in args:
        total += num
    return total

print(sum_all(1, 2, 3))       # 6
print(sum_all(10, 20, 30, 40))  # 100
print(sum_all())               # 0

def concatenate(*args):
    return " ".join(args)

print(concatenate("Hello", "World", "!"))  # Hello World !
```

### Basic **kwargs
```python
def create_profile(**kwargs):
    for key, value in kwargs.items():
        print(f"{key}: {value}")

create_profile(name="Alice", age=25, city="New York")
# name: Alice
# age: 25
# city: New York
```

### Combined with regular parameters
```python
def introduce(greeting, *names, **details):
    print(f"{greeting}, {' and '.join(names)}!")
    for key, value in details.items():
        print(f"  {key}: {value}")

introduce("Hello", "Alice", "Bob", location="party", time="8pm")
# Hello, Alice and Bob!
#   location: party
#   time: 8pm
```

### Unpacking in function calls
```python
def math_op(a, b, c):
    return a * b + c

# Unpack list
nums = [3, 4, 5]
print(math_op(*nums))  # 17

# Unpack tuple
coords = (2, 3, 4)
print(math_op(*coords))  # 10

# Unpack dict
params = {"a": 5, "b": 6, "c": 7}
print(math_op(**params))  # 37

# Mixed
print(math_op(1, *[2, 3]))  # 5
```

### Wrapper/decorator pattern
```python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__} with args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)
        print(f"{func.__name__} returned {result}")
        return result
    return wrapper

@logger
def add(a, b):
    return a + b

@logger
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

add(3, 4)
greet("Alice", greeting="Hi")
```

```python
def send_request(url, method="GET", *args, **kwargs):

```

### Using * for keyword-only arguments (Python 3+)
```python
def connect(host, port, *, timeout=30, ssl=True):
    """* forces everything after to be keyword-only."""
    print(f"Connecting to {host}:{port} (timeout={timeout}, ssl={ssl})")

connect("localhost", 8080, timeout=60, ssl=True)  # OK
# connect("localhost", 8080, 60, True)  # ERROR  -  must be keyword args
```

## Common mistakes

| Mistake | Wrong | Correct |
|---------|-------|---------|
| Forgetting that `args` is a tuple | `*args[0]` (invalid syntax) | `args[0]` (no star when accessing) |
| Using `**kwargs` when you need a specific dict | `def f(**kwargs): name = kwargs["name"]` | Better to use explicit parameter `def f(name, **kwargs)` |
| Ordering parameters wrong | `def f(**kwargs, *args)` | `def f(*args, **kwargs)`  -  `*args` before `**kwargs` |
| Thinking names `args`/`kwargs` are required | `def f(*stuff, **things)` | Works fine  -  only `*` and `**` are syntax |
| Unpacking dict with single `*` | `dict(**d)` or `func(*d)` | Use `**` for dicts: `func(**d)` |
| Mutating `**kwargs` of type `dict` silently | `kwargs["key"] = value` inside function | Works but can be confusing  -  side effects |
| Using `**kwargs` for everything | `def f(**kwargs): a = kwargs["a"]` | Better to be explicit with parameter names |

## Interview questions + answers

**Q1: What's the difference between `*args` and `**kwargs`?**
A: `*args` captures extra positional arguments as a tuple; `**kwargs` captures extra keyword arguments as a dictionary.

**Q2: Can you have both `*args` and `**kwargs` in the same function? What order?**
A: Yes. Order: regular params → `*args` → regular params with defaults → `**kwargs`. `*args` must come before `**kwargs`.

**Q3: What does `*` mean as a standalone parameter in `def func(a, b, *, c):`?**
A: It's a "keyword-only argument separator." Everything after `*` must be passed as a keyword argument.

**Q4: How do you unpack a list into function arguments?**
A: `func(*my_list)`  -  the `*` unpacks the list elements as positional arguments.

**Q5: What is the output of `def f(a, b, c): return a+b+c` called as `f(**{"b": 2, "c": 3, "a": 1})`?**
A: 6. `**` unpacks the dict into keyword arguments.

**Q6: Can you use `**kwargs` with `*args` in a subclass?**
A: Yes, this is common: `def __init__(self, *args, **kwargs): super().__init__(*args, **kwargs)`

**Q7: What's wrong with `def f(*args, x):`?**
A: Nothing  -  `x` is keyword-only because it comes after `*args`.

**Q8: What is the naming convention for `*args` and `**kwargs`?**
A: The names `args` and `kwargs` are conventions (PEP 8). Only `*` and `**` are syntactically required.

## Knowledge check

**Q1:** What does `def total(*nums): return sum(nums)` return for `total(1, 2, 3, 4, 5)`?
A: 15

**Q2:** What is the type of `args` inside a function defined with `*args`?
A: tuple

**Q3:** How do you make a parameter keyword-only?
A: Use `*` as a separator: `def f(a, b, *, c):`

**Q4:** What does `func(**{"name": "Alice"})` do?
A: Calls `func(name="Alice")`  -  unpacks dict as keyword arguments.

**Q5:** True or False: You must name the parameters `args` and `kwargs`.
A: False. Only `*` and `**` are syntax.

**Q6:** What's the output? `def f(a, b, *c): print(a, b, c)` then `f(1, 2, 3, 4, 5)`
A: `1 2 (3, 4, 5)`

**Q7:** Can `*args` be placed after `**kwargs` in a function definition?
A: No. `*args` must come before `**kwargs`.

**Q8:** How would you define a function that accepts at least one required argument and then any number of extra positional arguments?
A: `def func(required, *args):`

**Q9:** What's wrong with `def func(*args, **kwargs): func(*args, **kwargs)`? (recursive)
A: It will recurse infinitely (stack overflow) since it calls itself with no base case.

**Q10:** What does `*` do in the call `print(*[1, 2, 3])`?
A: Unpacks the list: `print(1, 2, 3)`

## Revision notes

- `*args` collects extra positional args into a **tuple**
- `**kwargs` collects extra keyword args into a **dict**
- Order in function def: `regular, *args, default, **kwargs`
- `*` alone creates keyword-only parameters: `def f(a, *, b)`  -  `b` is keyword-only
- Unpacking: `func(*list)` and `func(**dict)` in function calls
- Essential for decorators: `def wrapper(*args, **kwargs):`
- Names `args`/`kwargs` are convention, not syntax
- `*args` before `**kwargs` is required
- `args` is a tuple (immutable); `kwargs` is a dict (mutable)
- Avoid overusing `**kwargs`  -  explicit parameters are better for clarity
