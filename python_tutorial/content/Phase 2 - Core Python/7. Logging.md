# 7. Logging

## Definition
Logging is the practice of recording events, errors, and informational messages from a program as it runs. Python's built-in `logging` module provides a flexible framework for generating log messages with different severity levels, formatting, and output destinations (console, files, network, etc.).

## Why it exists
- **Debugging**: Track program flow and variable values without cluttering code with `print()` statements
- **Troubleshooting production issues**: When something goes wrong in production, logs are the first place to look
- **Auditing**: Record who did what and when for compliance and security
- **Monitoring**: Track system health, performance metrics, and error rates
- **Separation of concerns**: `print()` is for user output; logging is for operational data
- **Configurable verbosity**: Change log level at runtime (DEBUG in dev, WARNING in production) without code changes
- **Structured output**: Timestamps, module names, line numbers, thread IDs — automatically included

## How it works

### Basic logging
```python
import logging

# Simple configuration
logging.basicConfig(level=logging.INFO)
logging.debug("This won't show")       # Below INFO level
logging.info("Application started")    # Will show
logging.warning("Low disk space")      # Will show
logging.error("File not found")        # Will show
logging.critical("System shutting down")  # Will show
```

### Log levels (ordered by severity)
| Level | Numeric Value | When to Use |
|-------|---------------|-------------|
| `DEBUG` | 10 | Detailed diagnostic info (development only) |
| `INFO` | 20 | General operational messages (startup, shutdown) |
| `WARNING` | 30 | Something unexpected but not an error |
| `ERROR` | 40 | A problem that prevented an operation |
| `CRITICAL` | 50 | A serious failure — program may abort |

### Logger hierarchy
Loggers are arranged in a hierarchy based on their names (dots separate levels):
- Root logger: `""` (get with `logging.getLogger()`)
- `"myapp"` is child of root
- `"myapp.module"` is child of `"myapp"`
- By default, log messages propagate up to parent loggers

## Internal working

### The logging pipeline
```
LogRecord → Logger → Filter → Handlers → Formatter → Output
                        ↕
                    Level Check
```

1. **LogRecord** is created when you call `logger.info("message")`
2. The **Logger** checks if the message level is >= the logger's threshold. If not, it's dropped.
3. If passed, the message goes through any **Filters** attached to the logger
4. The message is sent to each **Handler** attached to the logger
5. Each handler checks its own level threshold
6. If passed, the **Handler** formats the message using its **Formatter**
7. The **Formatter** converts the log record into a string using the specified format
8. The handler writes to its destination (console, file, etc.)
9. By default, the message **propagates** to parent loggers

### BasicConfig
`logging.basicConfig()` is a convenience function that:
- Creates a `StreamHandler` writing to stderr (if no handler is attached to the root logger)
- Sets the root logger's level
- Sets the handler's formatter
- **Must be called before any logging calls** — it does nothing if called after logging has started

### Logger creation pattern (`__name__`)
```python
# In module_a.py
import logging
logger = logging.getLogger(__name__)  # Gets "myapp.module_a" if package is "myapp"
logger.info("Module A loaded")         # Logger name = module_a
```

Using `__name__` gives you the fully qualified module name, which makes it easy to identify where a log message came from.

## Real-world analogy
A **ship's logbook**:
- **DEBUG** = The captain's private notes about engine checks and minor adjustments (not shared with the crew)
- **INFO** = Standard log entries: "Departed port at 0800", "Changed course at 1400"
- **WARNING** = "Fuel gauge fluctuating" — something to watch but not an emergency
- **ERROR** = "Engine Room flooding" — a serious problem that needs immediate action
- **CRITICAL** = "Abandon ship!" — the ship is going down
- **Handlers** = Different distribution: one copy goes to the bridge (console), one to the archive (file), one to headquarters (network)
- **Formatters** = The standard format every entry must follow: "Date | Time | Importance | Message"

## Examples (with code)

### Basic configuration
```python
import logging

logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S'
)

logging.debug("This is a debug message")
logging.info("This is an info message")
logging.warning("This is a warning message")
logging.error("This is an error message")
logging.critical("This is a critical message")
```

2024-01-15 10:30:00 - root - ERROR - This is an error message

### Logging to a file
```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(levelname)s - %(message)s',
    filename='app.log',      # Write to file instead of console
    filemode='a'             # 'a' = append, 'w' = overwrite
)

logging.info("Application started")
logging.error("Something went wrong")
```

### Module-level loggers
```python
# --- main.py ---
import logging
import mymodule

logging.basicConfig(
    level=logging.DEBUG,
    format='%(name)s - %(levelname)s - %(message)s'
)

logger = logging.getLogger(__name__)  # "__main__"
logger.info("Main application starting")

mymodule.do_something()

# --- mymodule.py ---
import logging
logger = logging.getLogger(__name__)  # "mymodule"

def do_something():
    logger.debug("Doing something...")
    logger.info("Something done")
```

### Handlers (multiple destinations)
```python
import logging

# Create logger
logger = logging.getLogger("myapp")
logger.setLevel(logging.DEBUG)

# Console handler (stderr)
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.INFO)
console_format = logging.Formatter('%(levelname)s: %(message)s')
console_handler.setFormatter(console_format)

# File handler (all levels)
file_handler = logging.FileHandler('app.log')
file_handler.setLevel(logging.DEBUG)
file_format = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
file_handler.setFormatter(file_format)

# Add handlers to logger
logger.addHandler(console_handler)
logger.addHandler(file_handler)

# Now use the logger
logger.debug("Only in file")     # DEBUG: only file handler
logger.info("In both")           # INFO: both handlers
logger.error("Error message")    # INFO: both handlers
```

### RotatingFileHandler
```python
import logging
from logging.handlers import RotatingFileHandler

logger = logging.getLogger("myapp")
handler = RotatingFileHandler(
    'app.log',
    maxBytes=1024 * 1024,      # 1 MB per file
    backupCount=5               # Keep 5 backup files
)
handler.setFormatter(logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
))
logger.addHandler(handler)
logger.setLevel(logging.DEBUG)

# When app.log reaches 1MB, it's renamed app.log.1, app.log becomes new, etc.
```

### TimedRotatingFileHandler
```python
from logging.handlers import TimedRotatingFileHandler

handler = TimedRotatingFileHandler(
    'app.log',
    when='midnight',      # Rotate at midnight
    interval=1,
    backupCount=30         # Keep 30 days of logs
)
```

### Structured logging with extra data
```python
import logging

logger = logging.getLogger("payments")

# Log with extra context
logger.info(
    "Payment processed",
    extra={
        'user_id': 12345,
        'amount': 99.99,
        'currency': 'USD'
    }
)

# In the formatter, you can access extra fields:
formatter = logging.Formatter(
    '%(asctime)s - %(user_id)s - %(amount)s - %(message)s'
)
```

### Exception logging
```python
import logging

logger = logging.getLogger(__name__)

try:
    x = 1 / 0
except ZeroDivisionError:
    # Log the exception with traceback
    logger.exception("An error occurred while dividing")
    # Or with specific level:
    logger.error("Failed to divide", exc_info=True)
```

### Configuration via dictionary
```python
import logging.config

LOGGING_CONFIG = {
    'version': 1,
    'formatters': {
        'detailed': {
            'format': '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
        },
        'simple': {
            'format': '%(levelname)s: %(message)s'
        }
    },
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
            'level': 'INFO',
            'formatter': 'simple',
            'stream': 'ext://sys.stdout'
        },
        'file': {
            'class': 'logging.handlers.RotatingFileHandler',
            'level': 'DEBUG',
            'formatter': 'detailed',
            'filename': 'app.log',
            'maxBytes': 1048576,
            'backupCount': 5
        }
    },
    'loggers': {
        'myapp': {
            'handlers': ['console', 'file'],
            'level': 'DEBUG',
        }
    },
    'root': {
        'level': 'WARNING',
        'handlers': ['console']
    }
}

logging.config.dictConfig(LOGGING_CONFIG)
logger = logging.getLogger("myapp")
logger.info("Configured via dictionary")
```

## Common mistakes

1. **Calling `logging.basicConfig()` after logging has started** — It does nothing if any logging call has been made. Configure at the very start of the application.

2. **Using `print()` for logging** — Hard to filter, no levels, no timestamps, can't redirect without modifying code.

3. **Not using `__name__` for logger names** — Hard-coded strings like `"mylogger"` miss out on automatic module hierarchy.

4. **Forgetting to set `level` on the logger AND handlers** — Both the logger and handler have level thresholds. A message must pass both.

5. **Logging sensitive information** — Never log passwords, API keys, credit card numbers, or personal data.

6. **Too much logging at INFO level in production** — Can overwhelm storage and make it hard to find real issues. Use DEBUG for verbose output.

7. **Too little logging** — When something breaks in production, you'll wish you had more information. Log entry/exit of important operations.

8. **Not using `logger.exception()` in except blocks** — `logger.exception()` automatically includes the full traceback.

9. **String formatting in logging calls** — `logger.info(f"User {id} logged in")` always formats the string, even if the message is filtered by level. Use lazy formatting: `logger.info("User %s logged in", id)`.

10. **Not rotating log files** — Without rotation, a single log file grows indefinitely, consuming disk space.

## Interview questions + answers

**Q1: What are the five log levels in Python logging?**
A: DEBUG (10), INFO (20), WARNING (30), ERROR (40), CRITICAL (50).

**Q2: What's the difference between `logging.error()` and `logging.exception()`?**
A: Both log at ERROR level, but `logging.exception()` also includes the full traceback. It should only be called inside an `except` block.

**Q3: What does `basicConfig()` configure?**
A: The root logger: its level, a handler (console by default), and a formatter. It must be called before any logging calls.

**Q4: What is a `RotatingFileHandler`?**
A: A handler that writes to a file and automatically rotates (creates new file) when the file reaches a certain size, keeping a configurable number of backups.

**Q5: Why use `getLogger(__name__)` instead of a custom string?**
A: `__name__` automatically reflects the module's fully qualified name (e.g., `package.module`), which helps identify where log messages come from and enables hierarchical logging configuration.

**Q6: How do you log to both console and file?**
A: Create multiple handlers (`StreamHandler` for console, `FileHandler` for file) and add them to the logger with `logger.addHandler()`.

**Q7: What is log propagation?**
A: By default, log messages propagate from child loggers to parent loggers (up to the root). You can disable this with `logger.propagate = False`.

**Q8: How do you change log level at runtime?**
A: `logger.setLevel(logging.DEBUG)` or `logging.getLogger().setLevel(logging.INFO)`.

## Knowledge check (5-10 questions with answers)

**Q1: What's the default logging level?**
A: WARNING. Messages at DEBUG and INFO are suppressed by default.

**Q2: What function adds a timestamp to log messages?**
A: Including `%(asctime)s` in the format string.

**Q3: How do you log only to a file (not console)?**
A: Pass `filename='app.log'` to `basicConfig()` or use a `FileHandler`.

**Q4: What happens if you call `basicConfig()` after a logging call?**
A: Nothing — it's silently ignored. Configure logging before any other code.

**Q5: Which handler rotates logs by time?**
A: `TimedRotatingFileHandler` from `logging.handlers`.

**Q6: What does `logger.exception()` include that `logger.error()` doesn't?**
A: The full traceback of the current exception.

**Q7: How do you stop a child logger from sending messages to the root logger?**
A: Set `logger.propagate = False`.

**Q8: What's the numeric value of logging.WARNING?**
A: 30

**Q9: How do you include the module name in log format?**
A: `%(name)s` in the format string.

**Q10: What module provides `RotatingFileHandler`?**
A: `logging.handlers`

## Revision notes (bullet points summary)

- Five levels: DEBUG (10), INFO (20), WARNING (30), ERROR (40), CRITICAL (50)
- Default level is WARNING — DEBUG/INFO don't show
- `logging.basicConfig()` configures the root logger (call once at startup)
- Format with `%(asctime)s`, `%(name)s`, `%(levelname)s`, `%(message)s`, `%(lineno)d`
- Use `getLogger(__name__)` for hierarchical logger names
- Handlers control output destination: `StreamHandler` (console), `FileHandler` (file), `RotatingFileHandler` (rotated file)
- Formatters control output format with `logging.Formatter()`
- `logger.exception()` logs with traceback (use in `except` blocks)
- Use lazy formatting: `logger.info("User %s", name)` not `f"User {name}"`
- Configure at application start; never after logging has begun
- Add `exc_info=True` to log traceback at any level
- Use `DictConfig` for complex logging setups
- Never log sensitive data (passwords, keys, PII)
- Always log important events: startup/shutdown, errors, significant state changes

### Creative Mini Project: File Downloader with Logging

```python
"""
File Downloader with Comprehensive Logging
Downloads files from URLs with progress tracking and full logging.
"""

import logging
import logging.handlers
import sys
from pathlib import Path
import requests
import time
import os

# ============================================================
# LOGGING SETUP
# ============================================================

def setup_logger(name="downloader", log_file="downloader.log", level=logging.DEBUG):
    """Configure a logger with console and file handlers."""
    logger = logging.getLogger(name)
    logger.setLevel(level)
    logger.handlers.clear()

    # Format
    detailed_format = logging.Formatter(
        '%(asctime)s | %(name)s | %(levelname)-8s | %(funcName)s:%(lineno)d | %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S'
    )
    console_format = logging.Formatter(
        '%(levelname)-8s | %(message)s'
    )

    # Console handler (INFO and above)
    console = logging.StreamHandler(sys.stdout)
    console.setLevel(logging.INFO)
    console.setFormatter(console_format)
    logger.addHandler(console)

    # File handler with rotation (all levels)
    file_handler = logging.handlers.RotatingFileHandler(
        log_file,
        maxBytes=5 * 1024 * 1024,   # 5 MB
        backupCount=3
    )
    file_handler.setLevel(logging.DEBUG)
    file_handler.setFormatter(detailed_format)
    logger.addHandler(file_handler)

    # Error file handler (ERROR and above, separate file)
    error_handler = logging.handlers.RotatingFileHandler(
        "errors.log",
        maxBytes=2 * 1024 * 1024,  # 2 MB
        backupCount=2
    )
    error_handler.setLevel(logging.ERROR)
    error_handler.setFormatter(detailed_format)
    logger.addHandler(error_handler)

    return logger


# ============================================================
# FILE DOWNLOADER
# ============================================================

class FileDownloader:
    """Download files from URLs with progress tracking and comprehensive logging."""

    def __init__(self, download_dir="downloads"):
        self.download_dir = Path(download_dir)
        self.download_dir.mkdir(parents=True, exist_ok=True)
        self.logger = logging.getLogger(__name__)
        self.session = requests.Session()
        self.session.headers.update({
            'User-Agent': 'FileDownloader/1.0'
        })
        self.stats = {
            'successful': 0,
            'failed': 0,
            'skipped': 0,
            'total_bytes': 0,
            'start_time': None
        }

    def download_file(self, url, filename=None, overwrite=False, chunk_size=8192):
        """Download a single file from URL."""
        self.logger.info("Starting download: %s", url)

        if not filename:
            filename = url.split('/')[-1] or f"download_{int(time.time())}"
            if '.' not in filename:
                filename += ".bin"

        filepath = self.download_dir / filename

        # Check if file exists
        if filepath.exists() and not overwrite:
            self.logger.warning("File already exists, skipping: %s", filepath)
            self.stats['skipped'] += 1
            return filepath, "skipped"

        try:
            self.logger.debug("Sending GET request to %s", url)
            response = self.session.get(url, stream=True, timeout=30)
            response.raise_for_status()

            # Get file size from headers
            total_size = int(response.headers.get('content-length', 0))
            self.logger.info("File size: %s bytes (%s)",
                           total_size, self._format_size(total_size))

            # Write file
            downloaded = 0
            start_time = time.time()

            with open(filepath, 'wb') as f:
                for chunk in response.iter_content(chunk_size=chunk_size):
                    if chunk:
                        f.write(chunk)
                        downloaded += len(chunk)
                        self.stats['total_bytes'] += len(chunk)

                        # Log progress every 10%
                        if total_size > 0:
                            progress = (downloaded / total_size) * 100
                            if progress % 10 < 0.001 or progress > 99:
                                elapsed = time.time() - start_time
                                speed = downloaded / (elapsed * 1024) if elapsed > 0 else 0
                                self.logger.debug("Progress: %.0f%% (%s/%s, %.0f KB/s)",
                                                progress, self._format_size(downloaded),
                                                self._format_size(total_size), speed)

            elapsed = time.time() - start_time
            speed = (downloaded / (elapsed * 1024)) if elapsed > 0 else 0
            self.logger.info("Download complete: %s (%s in %.1fs, %.0f KB/s)",
                           filename, self._format_size(downloaded), elapsed, speed)

            self.stats['successful'] += 1
            return filepath, "success"

        except requests.exceptions.Timeout:
            self.logger.error("Timeout downloading: %s", url)
            self.stats['failed'] += 1
            return None, "timeout"

        except requests.exceptions.HTTPError as e:
            self.logger.error("HTTP error %s for %s", e.response.status_code, url)
            self.stats['failed'] += 1
            return None, f"http_{e.response.status_code}"

        except requests.exceptions.ConnectionError as e:
            self.logger.error("Connection error for %s: %s", url, str(e)[:100])
            self.stats['failed'] += 1
            return None, "connection_error"

        except requests.exceptions.RequestException as e:
            self.logger.error("Request failed for %s: %s", url, str(e)[:150])
            self.stats['failed'] += 1
            return None, "request_error"

        except OSError as e:
            self.logger.critical("File system error writing %s: %s", filepath, e)
            self.stats['failed'] += 1
            return None, "io_error"

        except Exception as e:
            self.logger.critical("Unexpected error downloading %s: %s", url, e, exc_info=True)
            self.stats['failed'] += 1
            return None, "unexpected_error"

    def download_multiple(self, urls, overwrite=False):
        """Download multiple files."""
        self.stats['start_time'] = time.time()
        self.logger.info("Starting batch download of %d files", len(urls))

        results = []
        for i, url in enumerate(urls, 1):
            self.logger.info("[%d/%d] Processing: %s", i, len(urls), url)
            result = self.download_file(url, overwrite=overwrite)
            results.append((url, result))

        self._print_summary()
        return results

    def download_from_file(self, url_file, overwrite=False):
        """Read URLs from a text file and download each."""
        url_path = Path(url_file)
        if not url_path.exists():
            self.logger.error("URL file not found: %s", url_file)
            return []

        with open(url_path) as f:
            urls = [line.strip() for line in f if line.strip() and not line.startswith('#')]

        self.logger.info("Loaded %d URLs from %s", len(urls), url_file)
        return self.download_multiple(urls, overwrite=overwrite)

    def _print_summary(self):
        """Print download summary."""
        elapsed = time.time() - self.stats['start_time'] if self.stats['start_time'] else 0
        self.logger.info("=" * 50)
        self.logger.info("DOWNLOAD SUMMARY")
        self.logger.info("=" * 50)
        self.logger.info("  Successful: %d", self.stats['successful'])
        self.logger.info("  Failed:     %d", self.stats['failed'])
        self.logger.info("  Skipped:    %d", self.stats['skipped'])
        self.logger.info("  Total data: %s", self._format_size(self.stats['total_bytes']))
        self.logger.info("  Time:       %.1f seconds", elapsed)
        if elapsed > 0:
            avg_speed = (self.stats['total_bytes'] / (elapsed * 1024))
            self.logger.info("  Avg speed:  %.0f KB/s", avg_speed)
        self.logger.info("=" * 50)

    @staticmethod
    def _format_size(size_bytes):
        """Format bytes to human-readable."""
        for unit in ['B', 'KB', 'MB', 'GB']:
            if size_bytes < 1024:
                return f"{size_bytes:.1f}{unit}" if isinstance(size_bytes, float) else f"{size_bytes}{unit}"
            size_bytes /= 1024
        return f"{size_bytes:.1f}TB"


# ============================================================
# DEMO
# ============================================================

def demo():
    """Demonstrate the file downloader with logging."""
    # Setup
    logger = setup_logger()
    downloader = FileDownloader("demo_downloads")

    # Test URLs
    test_urls = [
        "https://httpbin.org/bytes/1024",        # 1 KB file
        "https://httpbin.org/bytes/10240",       # 10 KB file
        "https://httpbin.org/delay/2",            # Delayed response
        "https://httpbin.org/status/404",         # 404 error
        "https://nonexistent.domain.test/file",   # Connection error
    ]

    # Download
    logger.info("Starting demo downloads...")
    results = downloader.download_multiple(test_urls, overwrite=True)

    # Show results
    logger.info("\nIndividual Results:")
    for url, (path, status) in results:
        logger.info("  %s → %s", url.split('/')[-1].ljust(20), status)

    # Cleanup
    import shutil
    logger.info("\nCleaning up...")
    shutil.rmtree("demo_downloads", ignore_errors=True)
    logger.info("Demo complete!")

    # Show log files created
    for log_file in ["downloader.log", "errors.log"]:
        if Path(log_file).exists():
            size = Path(log_file).stat().st_size
            print(f"Log file created: {log_file} ({size} bytes)")


if __name__ == "__main__":
    demo()
```
