# 5. JSON

## Definition
**JSON** (JavaScript Object Notation) is a lightweight, text-based data interchange format that's easy for humans to read/write and easy for machines to parse/generate. Python's `json` module provides functions to convert between Python objects and JSON strings/files.

## Why it exists
- **Data interchange**: The universal format for APIs — nearly every web service returns JSON
- **Configuration files**: Better than `.ini` or `.cfg` — supports nested structures naturally
- **Data persistence**: Save program state (lists, dicts, etc.) to a file without a database
- **Human-readable**: Easy to inspect, edit, and debug
- **Language-agnostic**: Every language has a JSON parser — Python ↔ JavaScript ↔ Java ↔ etc.
- **Lightweight**: Minimal overhead compared to XML or YAML for simple data

## How it works

### JSON data types
| JSON | Python |
|------|--------|
| `string` | `str` |
| `number` (int) | `int` |
| `number` (float) | `float` |
| `boolean` | `bool` |
| `null` | `None` |
| `array` | `list` |
| `object` | `dict` |

### JSON format example
```json
{
    "name": "Alice",
    "age": 30,
    "is_student": false,
    "grades": [90, 85, 92],
    "address": {
        "city": "New York",
        "zip": "10001"
    },
    "favorite_color": null
}
```

### The four main functions
| Function | Direction | Purpose |
|----------|-----------|---------|
| `json.dumps(obj)` | Python → JSON string | Serialize to string |
| `json.dump(obj, file)` | Python → file | Write JSON to file |
| `json.loads(string)` | JSON string → Python | Parse from string |
| `json.load(file)` | File → Python | Read JSON from file |

## Internal working

### Serialization process (`dumps` / `dump`)
1. Python recursively walks the object tree
2. Each Python type is mapped to its JSON equivalent (dict → object, list → array, str → string, etc.)
3. Special types (datetime, Decimal, custom objects) raise `TypeError` unless you provide a custom encoder
4. The result is a UTF-8 encoded string (or written directly to a file)
5. `ensure_ascii=True` by default: non-ASCII characters are escaped as `\uXXXX`

### Deserialization process (`loads` / `load`)
1. The JSON string is tokenized into lexical tokens (strings, numbers, braces, brackets, etc.)
2. A parser builds the Python object tree following JSON grammar rules
3. JSON objects become Python `dict` (preserving insertion order in Python 3.7+)
4. JSON arrays become Python `list`
5. `true`/`false`/`null` map to `True`/`False`/`None`

### Custom serialization
For objects that aren't natively JSON-serializable (like `datetime` or custom classes), you either:
- Define a method that returns a serializable dict (e.g., `obj.to_dict()`)
- Pass a custom `default` function to `json.dumps()`
- Create a custom `JSONEncoder` subclass

## Real-world analogy
A **postal package with a packing slip**:
- The **JSON object** is like a cardboard box with a packing slip listing what's inside
- **Keys** are item names on the slip ("shirt", "shoes")
- **Values** are the actual items
- **Arrays** are like multiple identical items ("3 shirts")
- **Nested objects** are like a box within a box (e.g., "shoes" box containing "left shoe" and "right shoe")
- `json.dumps()` is like writing the packing slip
- `json.loads()` is like reading the packing slip to know what's in the box
- **Pretty printing** (`indent=2`) is like organizing the packing slip with neat formatting

## Examples (with code)

### Basic dumps and loads
```python
import json

# Python → JSON string
person = {
    "name": "Alice",
    "age": 30,
    "skills": ["Python", "SQL", "JavaScript"],
    "employed": True
}

json_string = json.dumps(person)
print(json_string)
# {"name": "Alice", "age": 30, "skills": ["Python", "SQL", "JavaScript"], "employed": true}

# JSON string → Python
parsed = json.loads(json_string)
print(parsed["name"])      # Alice
print(parsed["skills"][1]) # SQL
```

### Pretty printing
```python
data = {"name": "Alice", "age": 30, "grades": {"math": 95, "science": 88}}

# Pretty print with indentation
print(json.dumps(data, indent=2))
# {
#   "name": "Alice",
#   "age": 30,
#   "grades": {
#     "math": 95,
#     "science": 88
#   }
# }

# Sort keys alphabetically
print(json.dumps(data, indent=2, sort_keys=True))
```

### Reading and writing to files
```python
import json

# Python → file (dump)
data = {
    "users": [
        {"id": 1, "name": "Alice", "email": "alice@example.com"},
        {"id": 2, "name": "Bob", "email": "bob@example.com"}
    ]
}

with open("data.json", "w") as f:
    json.dump(data, f, indent=2)

# File → Python (load)
with open("data.json", "r") as f:
    loaded = json.load(f)

print(loaded["users"][0]["name"])  # Alice
```

### Serialization of custom objects
```python
import json
from datetime import datetime

class Person:
    def __init__(self, name, age, created_at=None):
        self.name = name
        self.age = age
        self.created_at = created_at or datetime.now()

    def to_dict(self):
        return {
            "name": self.name,
            "age": self.age,
            "created_at": self.created_at.isoformat()
        }

# Using a custom default function
def serialize(obj):
    if isinstance(obj, datetime):
        return obj.isoformat()
    if hasattr(obj, "to_dict"):
        return obj.to_dict()
    raise TypeError(f"Object of type {type(obj)} is not JSON serializable")

p = Person("Alice", 30)
json_str = json.dumps(p, default=serialize, indent=2)
print(json_str)
# {
#   "name": "Alice",
#   "age": 30,
#   "created_at": "2024-01-15T10:30:00.123456"
# }

# Custom JSONEncoder
class PersonEncoder(json.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, Person):
            return obj.to_dict()
        if isinstance(obj, datetime):
            return obj.isoformat()
        return super().default(obj)

json_str2 = json.dumps(p, cls=PersonEncoder, indent=2)
```

### Handling errors
```python
import json

invalid_json = '{"name": "Alice", age: 30}'  # Missing quotes around key

try:
    data = json.loads(invalid_json)
except json.JSONDecodeError as e:
    print(f"Invalid JSON: {e}")
    print(f"  Position: line {e.lineno}, column {e.colno}")
    print(f"  Character: {e.pos}")
```

### Advanced options
```python
import json

data = {"name": "Alice", "score": 95.5, "tags": None}

# Compact output (no spaces)
print(json.dumps(data, separators=(',', ':')))
# {"name":"Alice","score":95.5,"tags":null}

# Skip non-ASCII characters
print(json.dumps({"name": "José"}, ensure_ascii=True))
# {"name": "Jos\u00e9"}
print(json.dumps({"name": "José"}, ensure_ascii=False))
# {"name": "José"}
```

## Common mistakes

1. **Trying to serialize non-serializable objects** — `datetime`, `Decimal`, custom classes. Provide a `default` function or convert first.

2. **Forgetting to open file in correct mode** — `'w'` for dump, `'r'` for load. Binary mode (`'wb'`) works too but produces bytes.

3. **Trailing commas in JSON** — Valid in Python dicts/lists but invalid in JSON.
   ```python
   # Invalid JSON:
   # {"name": "Alice", "age": 30,}
   
   json.loads('{"name": "Alice", "age": 30}')  # Correct
   ```

4. **Single quotes** — JSON requires double quotes for strings and keys.
   ```python
   json.loads("{'name': 'Alice'}")  # Error!
   json.loads('{"name": "Alice"}')  # Correct
   ```

5. **Assuming JSON key order** — JSON objects are unordered per spec. Python dicts preserve insertion order (3.7+), but don't rely on it for cross-language data.

6. **Not handling `JSONDecodeError`** — Always wrap `json.loads()` in try/except when parsing untrusted data.

7. **Modifying loaded data expecting it to affect the file** — `json.load()` returns a new Python object; changes don't affect the file. You must `json.dump()` again.

8. **`True`/`False`/`None` vs `true`/`false`/`null`** — Python uses capitalized; JSON uses lowercase. `json.dumps()` handles the conversion automatically.

9. **Loading huge JSON files into memory** — For very large files, use `ijson` (iterative JSON parser) instead of `json.load()`.

10. **Security: using `eval()` instead of `json.loads()`** — Never use `eval()` to parse JSON. `json.loads()` is safe; `eval()` executes arbitrary code.

## Interview questions + answers

**Q1: What's the difference between `json.dumps()` and `json.dump()`?**
A: `dumps()` returns a JSON string. `dump()` writes directly to a file object.

**Q2: What Python types are automatically JSON serializable?**
A: `dict`, `list`, `tuple` (as array), `str`, `int`, `float`, `bool`, `None`. `tuple` becomes a JSON array. `set`, `datetime`, `Decimal`, custom objects are NOT automatically serializable.

**Q3: How do you pretty-print JSON?**
A: Pass `indent=2` to `json.dumps()` or `json.dump()`.

**Q4: What's the difference between `json.loads()` and `json.load()`?**
A: `loads()` parses a JSON string. `load()` reads and parses from a file object.

**Q5: How do you serialize a `datetime` object?**
A: Either convert to string first (e.g., `isoformat()`), or pass a `default` function to `json.dumps()`.

**Q6: What exception does `json.loads()` raise on invalid input?**
A: `json.JSONDecodeError` (subclass of `ValueError`).

**Q7: How do you handle nested JSON data?**
A: Access it like nested dicts: `data["users"][0]["address"]["city"]`.

**Q8: What is `sort_keys` parameter?**
A: When `True`, dictionary keys are sorted alphabetically in the output JSON. Useful for consistent, diffable output.

**Q9: Can JSON represent a Python `tuple`?**
A: Yes, as a JSON array. `json.dumps((1, 2, 3))` → `[1, 2, 3]`. But `json.loads("[1, 2, 3]")` returns a `list`, not a tuple.

**Q10: What's the `ensure_ascii` parameter?**
A: When `True` (default), non-ASCII characters are escaped as `\uXXXX`. When `False`, they're output as-is.

## Knowledge check (5-10 questions with answers)

**Q1: What does `json.dumps({"name": "Alice"})` return?**
A: `'{"name": "Alice"}'` (a string)

**Q2: What does `json.loads('{"a": 1, "b": 2}')` return?**
A: `{'a': 1, 'b': 2}` (a Python dict)

**Q3: What type does JSON `null` map to in Python?**
A: `None`

**Q4: What type does JSON `true` map to in Python?**
A: `True` (Python bool)

**Q5: How do you write JSON to a file with indentation?**
A: `json.dump(data, file, indent=2)`

**Q6: What happens if you try to `json.dumps()` a `datetime` object?**
A: `TypeError` — it's not JSON serializable by default.

**Q7: Can JSON keys be integers?**
A: In JSON, keys are always strings. Python's `json.dumps()` converts integer keys to strings.

**Q8: What function would you use to read JSON from an API response?**
A: `response.json()` (from the `requests` library) or `json.loads(response.text)`.

**Q9: How do you handle duplicate keys in JSON?**
A: JSON technically doesn't allow duplicate keys. `json.loads()` keeps only the last value for duplicate keys.

**Q10: What's the output of `json.dumps([1, 2, 3], separators=(',', ':'))`?**
A: `'[1,2,3]'` (compact, no spaces)

## Revision notes (bullet points summary)

- JSON is a text-based data format with objects, arrays, strings, numbers, booleans, null
- Python ↔ JSON: `dict` ↔ object, `list` ↔ array, `str` ↔ string, `int`/`float` ↔ number, `bool` ↔ boolean, `None` ↔ null
- `json.dumps(obj)` → JSON string; `json.dump(obj, file)` → write to file
- `json.loads(str)` → Python object; `json.load(file)` → read from file
- Pretty print with `indent=2`; sort keys with `sort_keys=True`
- Non-serializable types (datetime, custom classes) need a `default` function or custom encoder
- JSON uses double quotes only; no trailing commas; `true`/`false`/`null` lowercase
- Handle API responses with `response.json()`
- Catch `json.JSONDecodeError` when parsing untrusted JSON
- `ensure_ascii=False` keeps Unicode characters; `separators=(',', ':')` produces compact output

### Creative Mini Project: Movie Database JSON Manager

```python
"""
Movie Database JSON Manager
A complete CRUD application that stores movie data in a JSON file.
"""

import json
import os
from datetime import datetime
from pathlib import Path


class MovieDatabase:
    """A JSON-backed movie database with full CRUD operations."""

    def __init__(self, filename="movies.json"):
        self.filename = filename
        self.data_file = Path(filename)
        self.movies = []
        self.load()

    def load(self):
        """Load movies from JSON file."""
        if self.data_file.exists():
            try:
                with open(self.data_file, "r") as f:
                    self.movies = json.load(f)
                print(f"Loaded {len(self.movies)} movies from {self.filename}")
            except (json.JSONDecodeError, FileNotFoundError) as e:
                print(f"Error loading database: {e}")
                self.movies = []
        else:
            self.movies = []
            self.save()  # Create empty database file

    def save(self):
        """Save movies to JSON file with pretty printing."""
        try:
            with open(self.data_file, "w") as f:
                json.dump(self.movies, f, indent=2, ensure_ascii=False)
            return True
        except (IOError, PermissionError) as e:
            print(f"Error saving database: {e}")
            return False

    # --- CREATE ---
    def add_movie(self, title, year, genre, rating, director=None):
        """Add a new movie to the database."""
        if not title or not title.strip():
            raise ValueError("Title cannot be empty")

        existing = [m for m in self.movies if m["title"].lower() == title.lower()]
        if existing:
            raise ValueError(f"Movie '{title}' already exists")

        movie = {
            "id": max([m["id"] for m in self.movies], default=0) + 1,
            "title": title.strip(),
            "year": year,
            "genre": genre,
            "rating": rating,
            "director": director or "Unknown",
            "created_at": datetime.now().isoformat(),
            "updated_at": datetime.now().isoformat()
        }
        self.movies.append(movie)
        self.save()
        print(f"✅ Added: {title} ({year})")
        return movie

    # --- READ ---
    def get_all(self, sort_by="title"):
        """Return all movies, optionally sorted."""
        if sort_by == "year":
            return sorted(self.movies, key=lambda m: m["year"], reverse=True)
        elif sort_by == "rating":
            return sorted(self.movies, key=lambda m: m["rating"], reverse=True)
        return sorted(self.movies, key=lambda m: m["title"].lower())

    def search(self, **filters):
        """Search movies by any field."""
        results = self.movies
        for key, value in filters.items():
            if key == "title":
                results = [m for m in results if value.lower() in m["title"].lower()]
            elif key == "genre":
                results = [m for m in results if value.lower() in m["genre"].lower()]
            elif key == "year":
                results = [m for m in results if m["year"] == value]
            elif key == "rating_min":
                results = [m for m in results if m["rating"] >= value]
            elif key == "rating_max":
                results = [m for m in results if m["rating"] <= value]
            elif key == "director":
                results = [m for m in results if value.lower() in m["director"].lower()]
        return results

    def get_by_id(self, movie_id):
        """Get a movie by its ID."""
        for m in self.movies:
            if m["id"] == movie_id:
                return m
        return None

    def get_stats(self):
        """Return statistics about the movie collection."""
        if not self.movies:
            return {"total": 0}
        ratings = [m["rating"] for m in self.movies]
        genres = {}
        decades = {}
        for m in self.movies:
            genre = m["genre"]
            genres[genre] = genres.get(genre, 0) + 1
            decade = (m["year"] // 10) * 10
            decades[decade] = decades.get(decade, 0) + 1
        return {
            "total": len(self.movies),
            "avg_rating": round(sum(ratings) / len(ratings), 2),
            "highest_rated": max(self.movies, key=lambda m: m["rating"]),
            "lowest_rated": min(self.movies, key=lambda m: m["rating"]),
            "genres": dict(sorted(genres.items(), key=lambda x: x[1], reverse=True)),
            "decades": dict(sorted(decades.items())),
            "unique_directors": len(set(m["director"] for m in self.movies))
        }

    # --- UPDATE ---
    def update_movie(self, movie_id, **updates):
        """Update fields of an existing movie."""
        movie = self.get_by_id(movie_id)
        if not movie:
            raise ValueError(f"Movie with ID {movie_id} not found")

        allowed_fields = {"title", "year", "genre", "rating", "director"}
        for key, value in updates.items():
            if key in allowed_fields:
                movie[key] = value

        movie["updated_at"] = datetime.now().isoformat()
        self.save()
        print(f"✅ Updated: {movie['title']}")
        return movie

    # --- DELETE ---
    def delete_movie(self, movie_id):
        """Delete a movie by ID."""
        movie = self.get_by_id(movie_id)
        if not movie:
            raise ValueError(f"Movie with ID {movie_id} not found")
        self.movies = [m for m in self.movies if m["id"] != movie_id]
        self.save()
        print(f"✅ Deleted: {movie['title']}")
        return True

    def clear_all(self):
        """Delete all movies (with confirmation)."""
        confirm = input(f"Delete all {len(self.movies)} movies? (yes/no): ")
        if confirm.lower() == "yes":
            self.movies = []
            self.save()
            print("✅ All movies deleted")
        else:
            print("Cancelled.")

    # --- DISPLAY ---
    def display(self, movies=None):
        """Display movies in a formatted table."""
        movies = movies or self.movies
        if not movies:
            print("📂 No movies in database.")
            return
        print(f"\n{'ID':<4} {'Title':<30} {'Year':<6} {'Rating':<7} {'Genre':<15} {'Director':<20}")
        print("-" * 85)
        for m in movies:
            print(f"{m['id']:<4} {m['title']:<30} {m['year']:<6} {m['rating']:<7.1f} {m['genre']:<15} {m['director']:<20}")
        print(f"\nTotal: {len(movies)} movies")

    def export_to_json(self, output_file):
        """Export movies to a separate JSON file."""
        with open(output_file, "w") as f:
            json.dump(self.movies, f, indent=2)
        print(f"✅ Exported to {output_file}")

    def import_from_json(self, input_file):
        """Import movies from another JSON file."""
        try:
            with open(input_file) as f:
                imported = json.load(f)
            count = 0
            for movie in imported:
                if not any(m["title"].lower() == movie["title"].lower() for m in self.movies):
                    movie["id"] = max([m["id"] for m in self.movies], default=0) + 1
                    self.movies.append(movie)
                    count += 1
            self.save()
            print(f"✅ Imported {count} new movies from {input_file}")
        except (FileNotFoundError, json.JSONDecodeError) as e:
            print(f"Error importing: {e}")
```
