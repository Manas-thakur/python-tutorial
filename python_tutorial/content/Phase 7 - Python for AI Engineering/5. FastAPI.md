# Phase 7: Python for AI Engineering  -  5. FastAPI

## Definition

FastAPI is a modern, high‑performance Python web framework for building APIs (RESTful web services). It is built on top of **Starlette** (for async request handling) and **Pydantic** (for data validation). Key features:
- High performance (on par with Node.js and Go)
- Automatic OpenAPI (Swagger) documentation
- Type‑hint driven request/response validation
- Async/await support out of the box

## Why it exists

Traditional Python web frameworks (Flask, Django) are synchronous  -  they block while waiting for I/O (database queries, file reads, external API calls). FastAPI is **async‑first**, meaning a single server can handle thousands of concurrent connections without blocking. Additionally, FastAPI uses Python type hints to automatically validate requests, serialize responses, and generate interactive API docs  -  eliminating boilerplate and manual validation code.

## How it works

1. `pip install fastapi uvicorn`
2. Create `main.py`: `from fastapi import FastAPI; app = FastAPI()`
3. Define routes with decorators: `@app.get("/")`, `@app.post("/items")`
4. Use type hints for path parameters, query parameters, and request bodies
5. Run: `uvicorn main:app --reload`
6. Visit `/docs` for Swagger UI, `/redoc` for ReDoc

## Internal working

- FastAPI is a thin layer on top of **Starlette** (the ASGI framework). Requests come in as ASGI messages.
- **Route decorators** register endpoints with Starlette's routing table.
- **Pydantic models** in type hints trigger automatic validation and serialization:
  - Request body → validated and converted to a Pydantic model instance.
  - Response model → serialized (`.model_dump()`) and returned as JSON.
- **Dependency injection**: `Depends()` marks a callable as a dependency. FastAPI builds a dependency graph and resolves it before calling the endpoint.
- **Async endpoints** (`async def`) use Python's `asyncio` event loop; synchronous endpoints (`def`) are run in a thread pool.
- **OpenAPI schema** is generated automatically by inspecting type hints, docstrings, and route metadata. `/docs` serves Swagger UI which reads this schema.

## Real-world analogy

Think of FastAPI like a **restaurant**:
- **Requests** are customers arriving.
- **Routes** (`@app.get("/menu")`) are menu sections.
- **Path/query parameters** are customizations ("table for 2", "no nuts").
- **Pydantic models** are the order ticket  -  if it's incomplete, the waiter (validation) sends it back.
- **Dependencies** are prep cooks (checking inventory, preheating oven) that run before the main chef.
- **Async** means the chef can start multiple orders and switch between them while waiting for the oven timer.
- **Swagger docs** is the menu in multiple languages  -  any customer can read it.

## Examples (with code)

### Installation

```bash
pip install fastapi uvicorn
```

### Basic app

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello World"}

@app.get("/items/{item_id}")
def read_item(item_id: int, q: str = None):
    return {"item_id": item_id, "q": q}

# Run: uvicorn main:app --reload
```

### Path parameters

```python
@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}

# Path parameter with type validation  -  FastAPI auto-converts to int
# If you pass a string, you get a 422 validation error
```

### Query parameters

```python
@app.get("/search")
def search(q: str = "", limit: int = 10, offset: int = 0):
    return {"query": q, "limit": limit, "offset": offset}

# Optional: parameters with defaults are optional in the query string
# Required: parameters without defaults must be provided
# /search?q=python&limit=5&offset=0
```

### Request body (Pydantic)

```python
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float
    is_offer: bool = False
    description: str = None

@app.post("/items")
def create_item(item: Item):
    # item is already validated by Pydantic
    return {"name": item.name, "price": item.price}
```

### Response model

```python
from pydantic import BaseModel
from datetime import datetime

class UserOut(BaseModel):
    id: int
    name: str
    created_at: datetime

class UserIn(BaseModel):
    name: str
    password: str

@app.post("/users", response_model=UserOut)
def create_user(user: UserIn):
    # `response_model` filters the response to only include fields in UserOut
    # This is perfect for hiding passwords!
    db_user = {"id": 1, "name": user.name, "created_at": datetime.now()}
    return db_user  # Pydantic will serialize this

# Only `id`, `name`, `created_at` will be in the JSON response
# `password` is automatically excluded!
```

### Async endpoints

```python
import asyncio

@app.get("/slow")
async def slow_endpoint():
    await asyncio.sleep(2)  # non-blocking sleep
    return {"status": "done after 2 seconds"}

# Async allows handling other requests while waiting
# Use async for database queries, HTTP calls, file I/O
```

### Dependency injection

```python
from fastapi import Depends, FastAPI

app = FastAPI()

# Dependency function  -  can be async too
def common_params(q: str = None, skip: int = 0, limit: int = 100):
    return {"q": q, "skip": skip, "limit": limit}

@app.get("/items")
def read_items(params: dict = Depends(common_params)):
    return params

# Dependency with class
class CommonQueryParams:
    def __init__(self, q: str = None, skip: int = 0, limit: int = 100):
        self.q = q
        self.skip = skip
        self.limit = limit

@app.get("/items2")
def read_items2(params: CommonQueryParams = Depends()):
    return params
```

### Dependency for auth

```python
from fastapi import Depends, HTTPException, status

def verify_token(token: str = None):
    if not token or token != "secret":
        raise HTTPException(status_code=401, detail="Invalid token")
    return token

@app.get("/protected")
def protected_route(token: str = Depends(verify_token)):
    return {"message": "You have access!", "token": token}
```

### CORS middleware

```python
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],         # allow all origins (dev only!)
    allow_credentials=True,
    allow_methods=["*"],         # allow all HTTP methods
    allow_headers=["*"],         # allow all headers
)
```

### Static files

```python
from fastapi.staticfiles import StaticFiles

app.mount("/static", StaticFiles(directory="static"), name="static")
# Now files in ./static/ are served at /static/filename.ext
```

### Error handling

```python
from fastapi import HTTPException, Request
from fastapi.responses import JSONResponse

# Raising errors
@app.get("/items/{item_id}")
def read_item(item_id: int):
    if item_id < 1:
        raise HTTPException(
            status_code=404,
            detail=f"Item {item_id} not found",
            headers={"X-Error": "Invalid ID"}
        )
    return {"item_id": item_id}

# Custom exception handler
class UnicornException(Exception):
    def __init__(self, name: str):
        self.name = name

@app.exception_handler(UnicornException)
async def unicorn_handler(request: Request, exc: UnicornException):
    return JSONResponse(
        status_code=418,
        content={"message": f"Oops! {exc.name} did something."}
    )

@app.get("/unicorns/{name}")
def read_unicorn(name: str):
    if name == "bad":
        raise UnicornException(name=name)
    return {"unicorn_name": name}
```

### PUT / DELETE

```python
@app.put("/items/{item_id}")
def update_item(item_id: int, item: Item):
    return {"item_id": item_id, **item.model_dump()}

@app.delete("/items/{item_id}")
def delete_item(item_id: int):
    return {"message": f"Item {item_id} deleted"}
```

## Creative mini project: Todo API with FastAPI

```python
"""
Simple Todo API with FastAPI
"""
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List
from uuid import uuid4

app = FastAPI(title="Todo API")

# In-memory "database"
todos = []

class TodoCreate(BaseModel):
    title: str
    description: str = ""

class Todo(BaseModel):
    id: str
    title: str
    description: str
    completed: bool = False

@app.post("/todos", response_model=Todo, status_code=201)
def create_todo(todo: TodoCreate):
    new_todo = Todo(
        id=str(uuid4()),
        title=todo.title,
        description=todo.description,
        completed=False
    )
    todos.append(new_todo)
    return new_todo

@app.get("/todos", response_model=List[Todo])
def list_todos(completed: bool = None):
    if completed is None:
        return todos
    return [t for t in todos if t.completed == completed]

@app.get("/todos/{todo_id}", response_model=Todo)
def get_todo(todo_id: str):
    for t in todos:
        if t.id == todo_id:
            return t
    raise HTTPException(status_code=404, detail="Todo not found")

@app.put("/todos/{todo_id}", response_model=Todo)
def update_todo(todo_id: str, todo: TodoCreate):
    for t in todos:
        if t.id == todo_id:
            t.title = todo.title
            t.description = todo.description
            return t
    raise HTTPException(status_code=404, detail="Todo not found")

@app.delete("/todos/{todo_id}")
def delete_todo(todo_id: str):
    for i, t in enumerate(todos):
        if t.id == todo_id:
            todos.pop(i)
            return {"message": "Todo deleted"}
    raise HTTPException(status_code=404, detail="Todo not found")

@app.patch("/todos/{todo_id}/toggle", response_model=Todo)
def toggle_todo(todo_id: str):
    for t in todos:
        if t.id == todo_id:
            t.completed = not t.completed
            return t
    raise HTTPException(status_code=404, detail="Todo not found")

# Run: uvicorn todo_api:app --reload
# Docs: http://localhost:8000/docs
```

## Common mistakes

| Mistake | Why it's wrong | Correct |
|---------|----------------|---------|
| `@app.get("/items/{id}")` then `@app.get("/items/help")` fails | `/items/help` matches `{id}` first | Define `/items/help` **before** the parameterized route |
| Forgetting `response_model` | Sensitive fields (password) leaked in response | Use `response_model=UserOut` to filter fields |
| Raising `HTTPException` without `from fastapi import HTTPException` | ImportError | Always import `HTTPException` |
| Using `def` for long sync I/O calls | Blocks the whole server | Use `async def` + `await` for I/O |
| Not installing `python-multipart` for file uploads | 500 error on file upload | `pip install python-multipart` |
| Type annotation with `Optional[str]` instead of `str = None` | Works but `Optional` is redundant with default | `q: str = None` (optional = default value) |
| Writing to a file inside an async endpoint | Blocking I/O on event loop | Use sync `def` (runs in thread pool) or `aiofiles` |
| Not setting CORS middleware for frontend integration | Browser blocks cross-origin requests | Add `CORSMiddleware` |
| Using `list` as mutable default in function | Shared across requests | Use `None` and create inside |

## Interview questions + answers

**Q1:** What is FastAPI and how does it differ from Flask?  
**A:** FastAPI is async‑first, uses Pydantic for validation, generates OpenAPI docs automatically, and is ~3× faster than Flask due to Starlette's ASGI foundation.

**Q2:** How does FastAPI perform automatic validation?  
**A:** It reads Python type hints on function parameters. Path/query/body types are validated and converted automatically via Pydantic. Invalid data returns a 422 JSON error.

**Q3:** What is the difference between `def` and `async def` in FastAPI?  
**A:** `def` runs in a thread pool (doesn't block the event loop, but not truly async). `async def` runs on the asyncio event loop  -  you can `await` inside.

**Q4:** How do you handle CORS in FastAPI?  
**A:** Add `CORSMiddleware` with `allow_origins`, `allow_methods`, `allow_headers`.

**Q5:** What is dependency injection in FastAPI?  
**A:** `Depends()` declares that a function parameter is a dependency. FastAPI resolves the dependency, calls it, passes the result to your endpoint.

**Q6:** How does FastAPI generate automatic documentation?  
**A:** It inspects route decorators, type hints, docstrings, and Pydantic models to produce an OpenAPI schema, served via Swagger UI at `/docs` and ReDoc at `/redoc`.

**Q7:** How do you create a route that accepts both query and path parameters?  
**A:** Define both in the function signature; path params come from the URL `{}`, query params are optional function args.

```python
@app.get("/items/{item_id}")
def get(item_id: int, q: str = None):
```

**Q8:** What is the purpose of `response_model`?  
**A:** It tells FastAPI to serialize the response using a Pydantic model, filtering out extra fields and converting types. Useful for hiding passwords or internal fields.

**Q9:** How do you add authentication to FastAPI?  
**A:** Use `Depends()` with a function that checks headers (e.g., OAuth2, JWT, API key). FastAPI also provides `fastapi.security` utilities.

**Q10:** What is Uvicorn and why is it used?  
**A:** Uvicorn is an ASGI server that runs FastAPI apps. It handles HTTP requests and serves the application. `uvicorn main:app --reload` auto-restarts on code changes.

## Knowledge check (5-10 questions with answers)

**Q1:** What command runs a FastAPI app with auto-reload?  
**A:** `uvicorn main:app --reload`

**Q2:** Which decorator handles HTTP GET requests?  
**A:** `@app.get("/path")`

**Q3:** Where can you view the interactive API documentation?  
**A:** `/docs` (Swagger UI) or `/redoc` (ReDoc)

**Q4:** True or False: FastAPI can only be used with async functions.  
**A:** False  -  it supports both `def` and `async def`.

**Q5:** What library does FastAPI use for request validation?  
**A:** Pydantic

**Q6:** What HTTP status code does FastAPI return for validation errors?  
**A:** 422 (Unprocessable Entity)

**Q7:** How do you define an optional query parameter?  
**A:** Give it a default value: `q: str = None`

**Q8:** What does `Depends()` do?  
**A:** Declares a dependency that FastAPI resolves before calling the endpoint.

**Q9:** How do you make a field required in a Pydantic model?  
**A:** Don't give it a default value.

**Q10:** What is the purpose of CORS middleware?  
**A:** Allows browsers to make cross-origin requests to the API.

## Revision notes (bullet points summary)

- **Import**: `from fastapi import FastAPI`; create `app = FastAPI()`.
- **Run**: `uvicorn main:app --reload`.
- **Routes**: `@app.get()`, `.post()`, `.put()`, `.delete()`, `.patch()`.
- **Path params**: `{item_id}` in URL, typed in function signature.
- **Query params**: optional function args with defaults.
- **Request body**: Pydantic `BaseModel` in function signature.
- **Response model**: `response_model=ModelOut`  -  filters and serializes.
- **Async**: `async def` + `await` for non-blocking I/O.
- **Dependencies**: `Depends()` for reusable logic (auth, DB sessions).
- **Error handling**: `HTTPException`; custom exception handlers.
- **CORS**: `CORSMiddleware` for cross-origin requests.
- **Static files**: `app.mount("/static", StaticFiles(directory="static"))`.
- **Automatic docs**: `/docs` (Swagger), `/redoc` (ReDoc).
- **Validation failures** return 422 automatically.
