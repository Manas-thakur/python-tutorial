# Phase 7: Python for AI Engineering  -  6. Pydantic

## Definition

Pydantic is a Python library for **data validation and settings management** using Python type annotations. At its core is `BaseModel`  -  a class that automatically validates, converts, and serializes data based on the type hints on its fields. Pydantic v2 is written in Rust (pydantic-core) for blazing‑fast validation.

## Why it exists

When your program receives data from the outside world (user input, API requests, config files, databases), you must:
1. **Validate**  -  is the email valid? Is age > 0?
2. **Convert**  -  strings to ints, ISO‑date strings to `datetime` objects.
3. **Document**  -  what fields are expected? What types?

Writing this manually is tedious and error‑prone. Pydantic automates all three using Python type hints, producing clear error messages and giving you type‑safe objects to work with.

## How it works

1. Define a model: `class User(BaseModel): name: str; age: int`
2. Create an instance: `user = User(name='Alice', age='30')`  -  Pydantic coerces `'30'` to `30`
3. Access fields: `user.name`, `user.model_dump()`
4. Validation errors raise `ValidationError` with a list of all failures.
5. Use `@validator` or `@field_validator` for custom validation logic.

## Internal working

- Pydantic uses Python's `__init_subclass__` and metaclasses to inspect type hints at class definition time.
- Each field is wrapped in a `FieldInfo` object holding type, default, constraints, and validators.
- At instantiation, input data is parsed by **pydantic-core** (Rust)  -  it converts types and applies constraints (`ge`, `le`, regex, etc.).
- Validators (`@field_validator`) are applied in the order they're defined.
- If parsing succeeds, the data is stored as a validated `__dict__` on the instance.
- `.model_dump()` recursively converts to Python dicts (with type coercion); `.model_dump_json()` serializes to JSON.
- `model_config = ConfigDict(extra='forbid')` controls handling of unknown fields.

## Real-world analogy

Think of Pydantic like a **bouncer** at a club:
- You (the data) try to enter.
- The bouncer checks your **ID** (type validation)  -  name must be string, age must be int ≥ 18.
- If you look 17 but your ID says 22, the bouncer checks further (**coercion**  -  string "22" → int 22 is fine).
- If your ID is fake or missing (**ValidationError**), you're turned away with a reason.
- Once inside, you get a **wristband** (the validated model instance) proving you passed the checks.
- `.model_dump()` is like emptying your pockets  -  showing exactly what's inside.

### Basic model

```python
from pydantic import BaseModel

class User(BaseModel):
    name: str
    age: int
    email: str = None   # optional with default

# Valid creation  -  type coercion occurs
user = User(name='Alice', age='30', email='alice@example.com')
print(user)            # name='Alice' age=30 email='alice@example.com'
print(user.name)       # 'Alice'
print(user.age)        # 30 (int, not str!)
```

### Field types and validation

```python
from pydantic import BaseModel, EmailStr, constr, conint, Field
from typing import Optional

class User(BaseModel):
    # Basic types
    name: str
    age: int

    # Constrained string: min_length, max_length, regex
    username: constr(min_length=3, max_length=20, pattern=r'^[a-zA-Z0-9_]+$')

    # Constrained int
    score: conint(ge=0, le=100)

    # Email validation (requires pydantic[email])
    email: EmailStr

    # Field() for metadata and defaults
    bio: Optional[str] = Field(None, max_length=500, description="User biography")
    role: str = Field(default='user', frozen=True)  # cannot be changed after init

# Try invalid data  -  this will raise ValidationError
try:
    user = User(
        name='Bob',
        age='invalid',
        username='ab',
        score=150,
        email='not-an-email'
    )
except Exception as e:
    print(e)  # multiple validation errors listed
```

### Validators (v1 style  -  `@validator`)

```python
from pydantic import BaseModel, validator

class PasswordModel(BaseModel):
    password: str
    confirm_password: str

    @validator('confirm_password')
    def passwords_match(cls, v, values, **kwargs):
        if 'password' in values and v != values['password']:
            raise ValueError('passwords do not match')
        return v

    @validator('password')
    def password_strength(cls, v):
        if len(v) < 8:
            raise ValueError('password must be at least 8 characters')
        if not any(c.isdigit() for c in v):
            raise ValueError('password must contain a digit')
        return v
```

### Validators (v2 style  -  `@field_validator`)

```python
from pydantic import BaseModel, field_validator

class UserModel(BaseModel):
    name: str
    age: int

    @field_validator('name')
    @classmethod
    def name_must_be_capitalized(cls, v: str) -> str:
        if not v[0].isupper():
            raise ValueError('name must start with uppercase')
        return v

    @field_validator('age')
    @classmethod
    def age_must_be_adult(cls, v: int) -> int:
        if v < 18:
            raise ValueError('must be 18 or older')
        return v

# Model-level validator (access multiple fields)
from pydantic import model_validator

class Order(BaseModel):
    items: int
    price_per_item: float
    total: float = None

    @model_validator(mode='after')
    def compute_total(self):
        self.total = self.items * self.price_per_item
        return self
```

### Serialization

```python
user = User(name='Alice', age=30, email='a@b.com')

# To dict
data = user.model_dump()
print(data)  # {'name': 'Alice', 'age': 30, 'email': 'a@b.com'}

# To JSON string
json_str = user.model_dump_json()
print(json_str)  # {"name": "Alice", "age": 30, "email": "a@b.com"}

# With indentation
print(user.model_dump_json(indent=2))

# Exclude fields
print(user.model_dump(exclude={'email'}))

# Include only certain fields
print(user.model_dump(include={'name', 'age'}))
```

### Parsing (model_validate)

```python
from pydantic import BaseModel

class Product(BaseModel):
    id: int
    name: str
    price: float

# Parse from dict (runtime validation)
data = {'id': '42', 'name': 'Laptop', 'price': '999.99'}
product = Product.model_validate(data)
print(product)  # id=42 name='Laptop' price=999.99

# Parse from JSON string
json_data = '{"id": 1, "name": "Mouse", "price": 25.50}'
product = Product.model_validate_json(json_data)
print(product)

# From raw Python objects that match the model
raw = (3, 'Keyboard', 75.0)
product = Product.model_validate(raw, from_attributes=True)  # if model has same attrs
```

### Nested models

```python
from pydantic import BaseModel
from typing import List

class Address(BaseModel):
    street: str
    city: str
    zip_code: str

class Employee(BaseModel):
    name: str
    address: Address          # nested model
    tags: List[str]           # list of strings

emp = Employee(
    name='Alice',
    address={'street': '123 Main', 'city': 'NYC', 'zip_code': '10001'},
    tags=['developer', 'python']
)
print(emp)
print(emp.address.city)   # 'NYC'
```

### Error handling

```python
from pydantic import BaseModel, ValidationError

class Item(BaseModel):
    name: str
    quantity: int

try:
    item = Item(name='Book', quantity='not_a_number')
except ValidationError as e:
    print(e)  # human-friendly error
    print(e.errors())  # list of error dicts
    # [
    #   {
    #     'type': 'int_parsing',
    #     'loc': ('quantity',),
    #     'msg': 'Input should be a valid integer',
    #     'input': 'not_a_number'
    #   }
    # ]
```

### Config (extra fields, frozen, etc.)

```python
from pydantic import BaseModel, ConfigDict

class StrictModel(BaseModel):
    model_config = ConfigDict(
        extra='forbid',      # reject unknown fields
        frozen=True,         # make instance immutable
        str_strip_whitespace=True,
        validate_default=True,
    )

    name: str
    age: int

# This would fail due to extra='forbid':
# StrictModel(name='A', age=20, extra_field='bad')

# This would fail due to frozen=True:
# m = StrictModel(name='A', age=20)
# m.age = 21  # raises error
```

## Creative mini project: User registration form validation

```python
"""
User registration validation with Pydantic
"""
from pydantic import BaseModel, EmailStr, field_validator, Field, ConfigDict, ValidationError
from typing import Optional
import re

class UserRegistration(BaseModel):
    model_config = ConfigDict(extra='forbid')

    username: str = Field(..., min_length=3, max_length=20, description="Unique username")
    email: EmailStr
    password: str = Field(..., min_length=8, max_length=64)
    confirm_password: str
    age: int = Field(..., ge=13, le=120, description="Must be at least 13")
    referral_code: Optional[str] = None

    @field_validator('username')
    @classmethod
    def username_alphanumeric(cls, v: str) -> str:
        if not re.match(r'^[a-zA-Z0-9_]+$', v):
            raise ValueError('username must be alphanumeric (letters, digits, underscores)')
        return v.lower()

    @field_validator('password')
    @classmethod
    def password_strength(cls, v: str) -> str:
        if not any(c.isupper() for c in v):
            raise ValueError('password must contain an uppercase letter')
        if not any(c.isdigit() for c in v):
            raise ValueError('password must contain a digit')
        if not any(c in '!@#$%^&*' for c in v):
            raise ValueError('password must contain a special character (!@#$%^&*)')
        return v

    @field_validator('confirm_password')
    @classmethod
    def passwords_match(cls, v: str, info) -> str:
        # info.data gives access to other fields
        if 'password' in info.data and v != info.data['password']:
            raise ValueError('passwords do not match')
        return v

    @field_validator('referral_code')
    @classmethod
    def validate_referral(cls, v: Optional[str]) -> Optional[str]:
        if v is not None and not v.startswith('REF'):
            raise ValueError('referral code must start with REF')
        return v

# Test valid data
try:
    user = UserRegistration(
        username='alice_99',
        email='alice@example.com',
        password='StrongP@ss1',
        confirm_password='StrongP@ss1',
        age=25,
        referral_code='REF123'
    )
    print(" Valid user registered!")
    print(user.model_dump_json(indent=2))
except ValidationError as e:
    print(" Validation failed:")
    print(e)

# Test invalid data
try:
    user = UserRegistration(
        username='ab',
        email='not-an-email',
        password='weak',
        confirm_password='different',
        age=10
    )
except ValidationError as e:
    print("\n Validation errors:")
    for err in e.errors():
        print(f"  - {err['loc'][0]}: {err['msg']} (got: {err['input']})")
```

## Common mistakes

| Mistake | Why it's wrong | Correct |
|---------|----------------|---------|
| Forgetting `pip install pydantic[email]` for `EmailStr` | ImportError | Install with email extra |
| Using `@validator` without `cls` as first param | TypeError | `def validate(cls, v)` |
| Returning `None` from a validator for a required field | Silently sets None | Raise `ValueError` to reject |
| Forgetting `@classmethod` with `@field_validator` (v2) | Function receives self incorrectly | Decorate with `@classmethod` |
| Not using `model_config = ConfigDict(extra='forbid')` | Unknown fields silently accepted | Add to config to reject extras |
| Confusing `model_dump()` and `model_dump_json()` | dict vs JSON string | Use `model_dump()` for Python; `model_dump_json()` for API responses |
| Mutating a `frozen=True` model after creation | RuntimeError | Set `frozen=False` (or `ConfigDict(frozen=False)`) |
| Assuming `model_validate` works on JSON strings | It expects dicts, not strings | Use `model_validate_json()` for JSON strings |

## Interview questions + answers

**Q1:** What is Pydantic and why is it used?  
**A:** Pydantic is a data validation library using Python type hints. It validates, coerces, serializes, and documents data automatically. Used heavily in FastAPI, LangChain, and config management.

**Q2:** What is the difference between `BaseModel.model_dump()` and `.model_dump_json()`?  
**A:** `model_dump()` returns a Python dict; `model_dump_json()` returns a JSON string. Use the former for internal use, the latter for API responses.

**Q3:** How do you validate one field based on another field's value?  
**A:** Use `@field_validator` for one field and access other fields via `info.data` (Pydantic v2), or use `@model_validator(mode='after')` for multi-field validation.

**Q4:** What is the `extra` config option in Pydantic?  
**A:** Controls behavior for unknown fields: `'ignore'` (default  -  silently ignore), `'forbid'` (raise error), `'allow'` (include them in the model).

**Q5:** How does Pydantic handle type coercion?  
**A:** It automatically converts compatible types: `'42'` → `42`, `True` → `1`, `'2024-01-01'` → `datetime`. Strict mode (`model_config = ConfigDict(strict=True)`) disables coercion.

**Q6:** What is the difference between `@validator` (v1) and `@field_validator` (v2)?  
**A:** `@field_validator` is the v2+ style, cleaner syntax, requires `@classmethod`, works with `info` argument. `@validator` is the v1 style, deprecated but still supported.

**Q7:** How do you make a field immutable in Pydantic?  
**A:** Set `frozen=True` in `Field(default=..., frozen=True)` or `model_config = ConfigDict(frozen=True)` for the whole model.

**Q8:** How do you parse a JSON string into a Pydantic model?  
**A:** `MyModel.model_validate_json(json_string)`. Or `MyModel(**json.loads(json_string))`.

**Q9:** What is `constr` and `conint` used for?  
**A:** Constrained types: `constr(min_length=3, max_length=50, regex=...)` and `conint(ge=0, le=100)` add validation directly in the type hint.

**Q10:** How does Pydantic handle nested data?  
**A:** Fields can be other `BaseModel` subclasses. When you pass a dict, it's automatically parsed into the nested model. Works recursively.

## Knowledge check (5-10 questions with answers)

**Q1:** What class do you inherit from to create a Pydantic model?  
**A:** `BaseModel`

**Q2:** What method converts a model to a Python dictionary?  
**A:** `.model_dump()`

**Q3:** True or False: Pydantic automatically converts `'123'` to `123` if the field is `int`.  
**A:** True.

**Q4:** Which type validates an email address?  
**A:** `EmailStr` (requires `pydantic[email]`).

**Q5:** What exception is raised when validation fails?  
**A:** `pydantic.ValidationError`

**Q6:** How do you reject unknown fields in a Pydantic model?  
**A:** `model_config = ConfigDict(extra='forbid')`

**Q7:** Which method parses a JSON string into a Pydantic model?  
**A:** `MyModel.model_validate_json(json_string)`

**Q8:** What does `conint(ge=0, le=100)` constrain?  
**A:** Integer must be between 0 and 100 (inclusive).

**Q9:** Where do you write custom validation logic?  
**A:** In a method decorated with `@field_validator` or `@model_validator`.

**Q10:** How do you make a field optional with a default value?  
**A:** `field_name: type = default_value` (e.g., `age: int = 18`).

## Revision notes (bullet points summary)

- **Pydantic**: data validation via Python type hints.
- **`BaseModel`**: define fields with type annotations; validation happens at instantiation.
- **Type coercion**: strings → ints, floats, dates automatically.
- **Field constraints**: `constr`, `conint`, `confloat`, `Field(ge=, le=, max_length=, pattern=)`.
- **Validators**: `@field_validator` (v2) / `@validator` (v1) for custom logic.
- **`EmailStr`**: requires `pydantic[email]` extra.
- **Serialization**: `model_dump()` → dict; `model_dump_json()` → JSON string.
- **Parsing**: `model_validate()` from dict; `model_validate_json()` from JSON string.
- **Nested models**: field can be another `BaseModel`  -  auto‑parsed.
- **Config**: `model_config = ConfigDict(extra='forbid', frozen=True, str_strip_whitespace=True)`.
- **Errors**: `ValidationError` with `.errors()` list.
- **Pydantic v2** is Rust‑powered (pydantic-core), ~5-50× faster than v1.
