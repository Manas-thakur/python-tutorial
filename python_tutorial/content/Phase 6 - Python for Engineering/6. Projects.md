# Projects

## Definition

This file provides **complete project walkthroughs** for three real-world CLI applications. Each project combines concepts from this phase: API requests, environment variables, testing, CLI argument parsing, and proper project structure.

The three projects are:
1. **CLI Todo App**  -  argparse, JSON file persistence, CRUD operations
2. **Weather App**  -  REST API, .env for API key, argparse for city input
3. **GitHub API Client**  -  REST API, pagination, authentication, display repos/issues

These are not toy examples  -  they are production-ready patterns you can extend, reuse, and adapt for your own needs.

## Why it exists

Learning individual topics (argparse, requests, dotenv, testing) is necessary but insufficient. The real skill is **combining them into a working application**. These projects bridge the gap between "I understand the pieces" and "I can build the whole thing."

Each project also demonstrates a common real-world pattern:
- **Todo App** = File-based persistence + CRUD + CLI (the pattern behind countless tools)
- **Weather App** = External API + env-based config + error handling (the pattern behind any API client)
- **GitHub Client** = Authenticated API + pagination + data display (the pattern behind API SDKs)

## Internal working

### Todo App Internal Flow

```
User:  todo add "Buy milk" --priority high
                     │
                     ▼
                  main.py
              argparse parses:
              command="add",
              task="Buy milk",
              priority="high"
                     │
                     ▼
              tasks = storage.load()
              tasks.append({"id": 3, ...})
              storage.save(tasks)
                     │
                     ▼
              Output: "Added task #3"
```

- JSON is the persistence layer (human-readable, simple, no database needed).
- Each task is a dict with `id`, `task`, `priority`, `done`, `created_at`.
- Tasks are stored in `~/.todo/tasks.json` for cross-session persistence.
- CRUD operations map directly to argparse subcommands.

### Weather App Internal Flow

```
User:  weather "Tokyo"
                     │
                     ▼
              main.py: config = Settings.from_env()
                     │
                     ▼
              fetcher.fetch_weather(city="Tokyo")
                     │
                     ▼
              requests.get(OPENWEATHER_URL, params={...})
                     │
                     ▼
              parser.parse_weather_response(raw_json)
                     │
                     ▼
              display.format_weather_output(weather_data)
                     │
                     ▼
              Output: "Tokyo: 22.5°C, clear sky, 60% humidity"
```

- The API key comes from `.env` (never hardcoded).
- The city comes from the CLI argument (positional, required).
- Error handling: network errors, invalid API key, city not found.

### GitHub Client Internal Flow

```
User:  github repos "octocat"
                     │
                     ▼
              main.py: client = GitHubClient(token=...)
                     │
                     ▼
              client.get_user_repos("octocat")
                     │
                     ▼
              For each page of results (pagination):
                requests.get(url, headers=headers)
                results.extend(page)
                url = next_page_url (from Link header)
                     │
                     ▼
              Display: repo name, stars, description, language
```

- Authentication is via Bearer token (stored in `.env`).
- Pagination follows `Link` headers (rel="next").
- Rate limits are checked before making requests.

## Real-world analogy

- **Todo App** = A notebook with a table of contents. You write tasks on numbered pages (CRUD), and the notebook stays on your desk (JSON file). When you close it (exit the app), everything is still there when you open it again (persistence).

- **Weather App** = Calling a weather hotline. You dial the number (API endpoint), tell the automated system your city (CLI arg), it looks up the data (server query), and reads you the forecast (response). You don't need to know how they measure temperature  -  you just get the result.

- **GitHub Client** = A library card catalog. You search by author name (GitHub username), the catalog shows you all books they wrote (repos), with details like genre (language), page count (stars), and whether it's checked out (issues count). If there are more than 30 books, you flip to the next card (pagination).

```python
from dataclasses import dataclass, asdict
from datetime import datetime
from typing import Optional

@dataclass
class Task:
    id: int
    description: str
    priority: str = "medium"          # low, medium, high
    done: bool = False
    created_at: str = ""

    def __post_init__(self):
        if not self.created_at:
            self.created_at = datetime.now().isoformat()

    @classmethod
    def from_dict(cls, data: dict) -> "Task":
        return cls(**data)

    def to_dict(self) -> dict:
        return asdict(self)

    def mark_done(self):
        self.done = True
```

**`src/storage.py`:**
```python
import json
from pathlib import Path
from src.models import Task

DATA_DIR = Path.home() / ".todo"
DATA_FILE = DATA_DIR / "tasks.json"

class Storage:
    @staticmethod
    def _ensure_dir():
        DATA_DIR.mkdir(parents=True, exist_ok=True)

    @staticmethod
    def load() -> list[Task]:
        Storage._ensure_dir()
        if not DATA_FILE.exists():
            return []
        with open(DATA_FILE, "r") as f:
            data = json.load(f)
        return [Task.from_dict(item) for item in data]

    @staticmethod
    def save(tasks: list[Task]):
        Storage._ensure_dir()
        with open(DATA_FILE, "w") as f:
            json.dump([t.to_dict() for t in tasks], f, indent=2)

    @staticmethod
    def get_next_id(tasks: list[Task]) -> int:
        if not tasks:
            return 1
        return max(t.id for t in tasks) + 1
```

**`src/main.py`:**
```python
import argparse
import sys
from src.storage import Storage
from src.models import Task

def cmd_add(args):
    tasks = Storage.load()
    task = Task(
        id=Storage.get_next_id(tasks),
        description=args.task,
        priority=args.priority,
    )
    tasks.append(task)
    Storage.save(tasks)
    print(f"Added task #{task.id}: {task.description} [{task.priority}]")

def cmd_list(args):
    tasks = Storage.load()
    if not tasks:
        print("No tasks yet! Use 'todo add <task>' to create one.")
        return
    
    # Filter
    if args.status == "done":
        tasks = [t for t in tasks if t.done]
    elif args.status == "pending":
        tasks = [t for t in tasks if not t.done]

    if not tasks:
        print("No tasks match the filter.")
        return

    for t in tasks:
        status = "[x]" if t.done else "[ ]"
        print(f"{t.id:3}. {status} {t.description:<40} [{t.priority:<6}]")

def cmd_done(args):
    tasks = Storage.load()
    for t in tasks:
        if t.id == args.task_id:
            t.mark_done()
            Storage.save(tasks)
            print(f"Task #{args.task_id} marked done!")
            return
    print(f"Task #{args.task_id} not found!")

def cmd_delete(args):
    tasks = Storage.load()
    new_tasks = [t for t in tasks if t.id != args.task_id]
    if len(new_tasks) == len(tasks):
        print(f"Task #{args.task_id} not found!")
        return
    Storage.save(new_tasks)
    print(f"Task #{args.task_id} deleted!")

def cmd_clear(args):
    Storage.save([])
    print("All tasks cleared!")

def main():
    parser = argparse.ArgumentParser(
        description="Simple CLI To-Do App",
        epilog="Happy task managing!",
    )
    subparsers = parser.add_subparsers(dest="command", help="Available commands")

    # add
    p_add = subparsers.add_parser("add", help="Add a new task")
    p_add.add_argument("task", help="Task description")
    p_add.add_argument("--priority", "-p", choices=["low", "medium", "high"], default="medium")

    # list
    p_list = subparsers.add_parser("list", help="List tasks")
    p_list.add_argument("--status", "-s", choices=["all", "done", "pending"], default="pending")

    # done
    p_done = subparsers.add_parser("done", help="Mark task as done")
    p_done.add_argument("task_id", type=int, help="Task ID to mark done")

    # delete
    p_delete = subparsers.add_parser("delete", help="Delete a task")
    p_delete.add_argument("task_id", type=int, help="Task ID to delete")

    # clear
    subparsers.add_parser("clear", help="Delete all tasks")

    args = parser.parse_args()

    commands = {
        "add": cmd_add,
        "list": cmd_list,
        "done": cmd_done,
        "delete": cmd_delete,
        "clear": cmd_clear,
    }

    if args.command in commands:
        commands[args.command](args)
    else:
        parser.print_help()
        sys.exit(1)

if __name__ == "__main__":
    main()
```

```python
import requests
from src.config import settings

BASE_URL = "https://api.openweathermap.org/data/2.5/weather"

def fetch_weather(city: str) -> dict:
    if not settings.api_key:
        raise ValueError("WEATHER_API_KEY not set. Copy .env.example to .env and add your key.")

    params = {"q": city, "appid": settings.api_key, "units": settings.units}
    response = requests.get(BASE_URL, params=params, timeout=settings.timeout)
    response.raise_for_status()
    return response.json()
```

**`src/main.py`:**
```python
import argparse
import sys
from src.config import settings
from src.fetcher import fetch_weather
from src.parser import parse_weather_response
from src.display import format_weather_output

def main():
    parser = argparse.ArgumentParser(description="Check the weather for any city")
    parser.add_argument("city", help="City name (e.g., 'London', 'Tokyo,JP')")
    parser.add_argument("--units", "-u", choices=["metric", "imperial"],
                        default=settings.units, help="Temperature units")

    args = parser.parse_args()

    try:
        raw = fetch_weather(args.city)
        data = parse_weather_response(raw)
        output = format_weather_output(data)
        print(output)
    except ValueError as e:
        print(f"Config error: {e}")
        sys.exit(1)
    except requests.exceptions.HTTPError as e:
        if e.response.status_code == 404:
            print(f"City '{args.city}' not found!")
        elif e.response.status_code == 401:
            print("Invalid API key! Check your WEATHER_API_KEY.")
        else:
            print(f"HTTP error: {e}")
        sys.exit(1)
    except requests.exceptions.ConnectionError:
        print("Could not connect to weather service. Check your internet.")
        sys.exit(1)
    except requests.exceptions.Timeout:
        print("Request timed out.")
        sys.exit(1)

if __name__ == "__main__":
    main()
```

**`src/client.py`:**
```python
import requests
import time
from src.config import settings
from src.models import Repository, Issue

BASE_URL = "https://api.github.com"

class GitHubClient:
    def __init__(self):
        self.session = requests.Session()
        self.session.headers.update({
            "Authorization": f"Bearer {settings.token}",
            "Accept": "application/vnd.github.v3+json",
            "User-Agent": "GitHubClient/1.0",
        })
        self.remaining_requests = 60
        self.reset_time = 0

    def _check_rate_limit(self):
        if self.remaining_requests == 0:
            wait = self.reset_time - time.time() + 1
            if wait > 0:
                print(f"Rate limited. Waiting {wait:.0f}s...")
                time.sleep(wait)

    def _handle_pagination(self, url: str) -> list[dict]:
        """Fetch all pages and return combined results."""
        all_items = []
        while url:
            self._check_rate_limit()
            response = self.session.get(url, timeout=10)
            response.raise_for_status()

            # Track rate limits
            self.remaining_requests = int(response.headers.get("X-RateLimit-Remaining", 0))
            self.reset_time = int(response.headers.get("X-RateLimit-Reset", 0))

            all_items.extend(response.json())

            # Check Link header for next page
            link_header = response.headers.get("Link", "")
            url = None
            if 'rel="next"' in link_header:
                for part in link_header.split(","):
                    if 'rel="next"' in part:
                        url = part.split(";")[0].strip("<> ").strip()
                        break

        return all_items

    def get_user(self, username: str) -> dict:
        url = f"{BASE_URL}/users/{username}"
        response = self.session.get(url, timeout=10)
        response.raise_for_status()
        self.remaining_requests = int(response.headers.get("X-RateLimit-Remaining", 0))
        return response.json()

    def get_user_repos(self, username: str) -> list[Repository]:
        url = f"{BASE_URL}/users/{username}/repos?per_page={settings.per_page}"
        raw_repos = self._handle_pagination(url)
        return [Repository.from_api(r) for r in raw_repos]

    def get_repo_issues(self, owner: str, repo: str, state: str = "open") -> list[Issue]:
        url = f"{BASE_URL}/repos/{owner}/{repo}/issues?state={state}&per_page={settings.per_page}"
        raw_issues = self._handle_pagination(url)
        return [Issue.from_api(i) for i in raw_issues if "pull_request" not in i]

    def get_rate_limit_status(self) -> dict:
        response = self.session.get(f"{BASE_URL}/rate_limit", timeout=10)
        response.raise_for_status()
        return response.json()["resources"]["core"]
```

```python
        print("No repositories found.")
        return
    print(f"\n Repositories ({len(repos)}):")
    print(f"   {'=' * 60}")
    for repo in repos:
        stars = f"⭐ {repo.stars}" if repo.stars else "   "
        print(f"   {repo.name:<30} {stars:<6} {repo.language:<12}")
        print(f"   {'':>4}{repo.description:<56}")
        print(f"   {'':>4} {repo.url}")
        print()

def display_issues(issues: list[Issue]):
    if not issues:
        print("No issues found.")
        return
    print(f"\n Issues ({len(issues)}):")
    print(f"   {'=' * 60}")
    for issue in issues:
        labels = ", ".join(issue.labels) if issue.labels else "none"
        print(f"   #{issue.number:<5} [{issue.state:<5}] {issue.title}")
        print(f"   {'':>4}by {issue.author:<20} {issue.created_at}")
        print(f"   {'':>4}labels: {labels}")
        print(f"   {'':>4} {issue.comments} comments")
        print()
```

```python
import argparse
import sys
from src.config import settings
from src.client import GitHubClient
from src.display import display_user_info, display_repos, display_issues
from requests.exceptions import HTTPError

def main():
    parser = argparse.ArgumentParser(
        description="GitHub API Client  -  explore users, repos, and issues"
    )
    subparsers = parser.add_subparsers(dest="command", help="Commands")

    # user command
    p_user = subparsers.add_parser("user", help="Get user info")
    p_user.add_argument("username", help="GitHub username")

    # repos command
    p_repos = subparsers.add_parser("repos", help="List user's repositories")
    p_repos.add_argument("username", help="GitHub username")

    # issues command
    p_issues = subparsers.add_parser("issues", help="List repository issues")
    p_issues.add_argument("repo", help="Repository (e.g., 'octocat/Hello-World')")

    args = parser.parse_args()

    if not settings.token:
        print("ERROR: GITHUB_TOKEN not set. Create a .env file with your GitHub token.")
        print("Get a token at: https://github.com/settings/tokens")
        sys.exit(1)

    client = GitHubClient()

    try:
        if args.command == "user":
            user = client.get_user(args.username)
            display_user_info(user)

        elif args.command == "repos":
            repos = client.get_user_repos(args.username)
            display_repos(repos)

        elif args.command == "issues":
            if "/" not in args.repo:
                print("ERROR: Repository must be in 'owner/name' format (e.g., 'octocat/Hello-World')")
                sys.exit(1)
            owner, repo_name = args.repo.split("/", 1)
            issues = client.get_repo_issues(owner, repo_name)
            display_issues(issues)

        else:
            parser.print_help()

    except HTTPError as e:
        if e.response.status_code == 404:
            print(f"Not found: {args.command} '{getattr(args, 'username', args.repo, '?')}'")
        elif e.response.status_code == 401:
            print("Authentication failed. Check your GITHUB_TOKEN.")
        elif e.response.status_code == 403:
            print("Forbidden. Your token may lack permissions or you're rate limited.")
        else:
            print(f"HTTP {e.response.status_code}: {e}")
        sys.exit(1)

if __name__ == "__main__":
    main()
```

## Common mistakes

- **Not testing edge cases**: Empty lists, missing data, network failures, invalid input. Test all of them.
- **Hardcoding API keys**: Always use `.env` files + `.env.example` (with dummy values) committed to git.
- **Ignoring error handling**: Network errors, 404s, 401s, timeouts  -  your app WILL encounter them. Handle gracefully.
- **No pagination**: GitHub returns 30 items per page by default. Your client will silently miss data if you don't paginate.
- **One giant file**: Break the project into modules (models, client, display, config). Your future self will thank you.
- **Forgetting `.gitignore`**: Never commit `.env`, `__pycache__/`, `*.pyc`, `.pytest_cache/`.
- **No `.env.example`**: Team members need to know what environment variables to set.
- **Not using `pip install -e .`**: Running `python src/main.py` works, but entry points won't be available.
- **Not following rate limits**: GitHub allows 60 unauthenticated requests/hour. Always authenticate with a token (5000/hour).
- **Hardcoding file paths**: Use `Path.home() / ".appname"` for data files, not relative paths.

## Interview questions + answers

**Q1: How would you handle API pagination in a CLI tool?**
A: Check the `Link` header for `rel="next"`. Parse the URL from the header, fetch the next page, and concatenate results. Loop until no `next` link. This handles any API that follows the standard Link header convention.

**Q2: How do you persist data in a CLI app without a database?**
A: Use JSON files stored in `~/.appname/`. JSON is human-readable, easy to debug, and Python's `json` module handles serialization/deserialization trivially. For larger datasets, consider SQLite.

**Q3: How do you test code that calls an external API?**
A: Mock the network layer. Use `unittest.mock.patch` to replace `requests.get` (or your API client) with a mock that returns pre-defined test data. This makes tests fast, deterministic, and offline-capable.

**Q4: Design a CLI todo app. What commands would it have?**
A: `add <task>` (with `--priority`), `list` (with `--status` filter), `done <id>`, `delete <id>`, `clear`. Data stored in JSON. Uses subparsers for clean command routing.

**Q5: How do you handle errors gracefully in a CLI app?**
A: Catch specific exceptions (HTTPError, ConnectionError, Timeout). Print user-friendly messages (not tracebacks unless `--debug`). Return appropriate exit codes (`sys.exit(0)` for success, `sys.exit(1)` for errors).

**Q6: Why use dataclasses for models instead of plain dicts?**
A: Dataclasses provide: (1) type hints for IDE support, (2) `__init__`, `__repr__`, `__eq__` auto-generated, (3) methods for behaviour (e.g., `to_dict()`), (4) immutability option with `frozen=True`.

**Q7: How would you make a CLI tool installable via pip?**
A: Create `pyproject.toml` with `[project.scripts]` entry point (`myapp = "src.main:main"`). Run `pip install -e .` for development, or package and upload to PyPI.

**Q8: What's the difference between `@patch` and `Mock`?**
A: `@patch("module.object")` temporarily replaces the real object with a `MagicMock` during the test. `Mock()` creates a standalone mock object you can configure with `return_value` and `side_effect`. Use `@patch` to intercept imports, `Mock` for dependency injection.
