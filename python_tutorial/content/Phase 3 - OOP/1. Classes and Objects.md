# 1. Classes and Objects

## Definition

A **class** is a blueprint or template for creating objects. It defines a set of attributes (data) and methods (functions) that the objects of that class will have. An **object** is an instance of a class — a concrete entity created from the class blueprint.

```python
class Dog:
    pass

my_dog = Dog()  # my_dog is an object (instance) of the Dog class
```

## Why it exists

Classes and objects exist to model real-world entities in code. Without them, you'd manage data and behavior separately using scattered variables and functions. OOP bundles related data and behavior together, making code:
- **Organized** — related things stay together
- **Reusable** — one class can create many objects
- **Maintainable** — changes to a class affect all objects uniformly

## How it works

You define a class with the `class` keyword, then instantiate objects by calling the class like a function.

```python
class Dog:
    species = "Canis familiaris"  # class attribute

    def __init__(self, name, age):
        self.name = name          # instance attribute
        self.age = age

    def bark(self):               # instance method
        return f"{self.name} says Woof!"

# Creating objects
dog1 = Dog("Rex", 3)
dog2 = Dog("Bella", 5)

print(dog1.bark())  # Rex says Woof!
print(dog2.bark())  # Bella says Woof!
```

## Internal working

When you write `my_obj = MyClass()`, Python:
1. Calls `MyClass.__new__()` to allocate memory (creates a new empty object)
2. Calls `MyClass.__init__(self, ...)` to initialize the object
3. Returns the object reference

Each object has a `__dict__` attribute — a dictionary storing all instance attributes:

```python
print(dog1.__dict__)  # {'name': 'Rex', 'age': 3}
```

Class attributes are stored in the class's `__dict__`, not the instance's. When you access `dog1.species`, Python first looks in `dog1.__dict__`, then in `Dog.__dict__`.

The `dir()` function lists all attributes and methods of an object:

```python
print(dir(dog1))  # ['__class__', '__dict__', ..., 'age', 'bark', 'name', 'species']
```

### Class attribute vs instance attribute lookup

```python
dog1.species       # Found in Dog.__dict__ → "Canis familiaris"
dog1.name          # Found in dog1.__dict__ → "Rex"
dog1.color         # Not found anywhere → AttributeError
```

**Instance attributes shadow class attributes.** If you set `dog1.species = "Wolf"`, Python creates an entry in `dog1.__dict__`, and that instance no longer sees the class attribute.

## Real-world analogy

A **class** is like a blueprint for a house. The blueprint defines that every house will have doors, windows, and a roof — but doesn't specify their exact measurements. Each **object** is an actual house built from that blueprint. One house might have a red door, another a blue door — they share the structure but differ in specific values.

## Examples (with code)

### Example 1: Basic class with attributes and methods

```python
class Car:
    wheels = 4  # class attribute

    def __init__(self, brand, model, year):
        self.brand = brand    # instance attributes
        self.model = model
        self.year = year
        self.odometer = 0

    def drive(self, km):
        self.odometer += km
        return f"Drove {km} km. Total: {self.odometer} km"

    def info(self):
        return f"{self.year} {self.brand} {self.model}"

car1 = Car("Toyota", "Corolla", 2020)
car2 = Car("Honda", "Civic", 2022)

print(car1.info())         # 2020 Toyota Corolla
print(car1.drive(100))     # Drove 100 km. Total: 100 km
print(car1.wheels)         # 4 (class attribute)
print(car2.wheels)         # 4 (class attribute)
```

### Example 2: Instance vs class vs static methods

```python
class MathUtils:
    pi = 3.14159  # class attribute

    def __init__(self, name):
        self.name = name  # instance attribute

    # Instance method — needs self, can access instance & class data
    def greet(self):
        return f"Hello, I'm {self.name}"

    # Class method — needs cls, can access class data but not instance data
    @classmethod
    def circle_area(cls, radius):
        return cls.pi * radius ** 2

    # Static method — needs neither self nor cls, independent utility
    @staticmethod
    def add(a, b):
        return a + b

m = MathUtils("Algo")
print(m.greet())               # Hello, I'm Algo
print(MathUtils.circle_area(5))  # 78.53975
print(MathUtils.add(10, 20))     # 30
```

### Example 3: The `self` parameter

`self` refers to the current instance. It's automatically passed by Python — you never pass it manually.

```python
class Person:
    def __init__(self, name):
        self.name = name

    def introduce(self):
        return f"My name is {self.name}"

p = Person("Alice")
print(p.introduce())  # Python does: Person.introduce(p)
```

### Example 4: `__dict__` and `dir()` in action

```python
class Book:
    genre = "Fiction"

    def __init__(self, title, author):
        self.title = title
        self.author = author

    def summary(self):
        return f"{self.title} by {self.author}"

b = Book("1984", "George Orwell")
print(b.__dict__)     # {'title': '1984', 'author': 'George Orwell'}
print(Book.__dict__)  # Includes 'genre', 'summary', '__init__', ...

b.pages = 328         # Adding a new attribute dynamically
print(b.__dict__)     # {'title': '1984', 'author': 'George Orwell', 'pages': 328}
```

## Common mistakes

1. **Forgetting `self` as the first parameter** in instance methods → causes `TypeError: takes 0 positional arguments but 1 was given`
2. **Using class attributes when instance attributes are needed** — changes to a mutable class attribute (e.g., a list) affect all instances:
   ```python
   class Bad:
       items = []  # Shared across all instances!
   a = Bad(); b = Bad()
   a.items.append("X")
   print(b.items)  # ['X'] — oops!
   ```
3. **Naming classes in lowercase** — always use PascalCase for class names: `MyClass`, not `myClass`
4. **Confusing `__init__` with a constructor** — `__init__` initializes; `__new__` constructs
5. **Modifying a class attribute on an instance** — this creates an instance attribute that shadows the class attribute instead of modifying it

## Interview questions + answers

**Q1: What is the difference between a class and an object?**
A: A class is a blueprint (template), while an object is an instance created from that blueprint. The class defines attributes and methods; the object has concrete values.

**Q2: What is `self` in Python?**
A: `self` is a reference to the current instance of the class. It's automatically passed as the first argument to instance methods. You can name it anything (though convention demands `self`), and Python always passes the instance as the first argument.

**Q3: Explain the difference between instance, class, and static methods.**
A: Instance methods receive `self` and can access/modify instance and class state. Class methods receive `cls` and can only access/modify class state. Static methods receive neither and are just utility functions namespaced inside the class.

**Q4: What is `__init__`? Is it a constructor?**
A: `__init__` is the initializer, not the constructor. The actual constructor is `__new__`, which allocates memory. `__init__` is called after `__new__` to initialize the newly created object.

**Q5: How does attribute lookup work in Python classes?**
A: Python first checks the instance's `__dict__`. If not found, it checks the class's `__dict__`, then parent classes following the MRO (Method Resolution Order). If still not found, it raises `AttributeError`.

<details><summary>Answer</summary>

## Revision notes (bullet points summary)

- `class ClassName:` defines a class; use PascalCase
- `obj = ClassName()` creates an instance
- `self` is the first parameter of every instance method; references the current instance
- `__init__` initializes an object after `__new__` creates it
- Instance attributes are unique per object; class attributes are shared
- `@classmethod` receives `cls`; `@staticmethod` receives neither
- `__dict__` stores all instance/class attributes in a dictionary
- `dir(obj)` lists all attributes and methods of an object
- Instance attributes shadow class attributes of the same name
- Attribute lookup: instance → class → parent classes → AttributeError
