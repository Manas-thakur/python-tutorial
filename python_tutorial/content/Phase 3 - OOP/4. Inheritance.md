# 4. Inheritance

## Definition

**Inheritance** is an OOP mechanism that allows a class (child/subclass) to derive properties and behaviors from another class (parent/superclass). The child class can reuse, extend, or override the parent's attributes and methods.

```python
class Animal:          # Parent class
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "..."

class Dog(Animal):     # Child class — inherits from Animal
    def speak(self):
        return "Woof!"
```

## Why it exists

Inheritance exists to:
- **Reuse code** — avoid duplicating functionality across similar classes
- **Establish relationships** — model "is-a" relationships (a Dog *is an* Animal)
- **Create hierarchies** — organize code from general to specific
- **Enable polymorphism** — treat different subclasses through a common interface

## How it works

The child class inherits all attributes and methods from the parent. It can:
- **Add** new attributes/methods
- **Override** (replace) inherited methods
- **Extend** inherited methods using `super()`

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

    def move(self):
        return f"{self.name} moves"

class Dog(Animal):
    def speak(self):                        # Override
        return f"{self.name} says Woof!"

    def fetch(self):                        # New method
        return f"{self.name} fetches a ball"

dog = Dog("Rex")
print(dog.speak())    # Rex says Woof!     (overridden)
print(dog.move())     # Rex moves          (inherited)
print(dog.fetch())    # Rex fetches a ball (new method)
```

## Internal working

### Method Resolution Order (MRO)

When you call a method on an object, Python searches for it in this order:
1. The instance's class
2. Its parent classes (left to right)
3. Grandparent classes
4. ... up to `object`

```python
class A:
    def method(self):
        return "A"

class B(A):
    pass

class C(A):
    def method(self):
        return "C"

class D(B, C):
    pass

d = D()
print(d.method())        # "C" — B doesn't override, C does
print(D.__mro__)
# (<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>)
```

### `super()` function

`super()` returns a temporary proxy object that delegates method calls to the parent class. It's essential for cooperative multiple inheritance:

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # Call parent's __init__
        self.breed = breed
```

### `isinstance()` and `issubclass()`

```python
class Animal: pass
class Dog(Animal): pass

d = Dog()
print(isinstance(d, Dog))     # True
print(isinstance(d, Animal))  # True (Dog is an Animal)
print(issubclass(Dog, Animal))  # True
print(issubclass(Animal, Dog))  # False
```

## Real-world analogy

Think of inheritance like a family tree. A child inherits traits from parents (eye color, height potential) but can also develop unique characteristics. The child can override inherited traits (e.g., learn a different language) or acquire new ones not present in the parents. Just like a "Doberman is a Dog is a Mammal is an Animal," inheritance creates an "is-a" hierarchy.

## Examples (with code)

### Example 1: Single inheritance with `super()`

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        return f"Hi, I'm {self.name}, {self.age} years old."

class Student(Person):
    def __init__(self, name, age, student_id, major):
        super().__init__(name, age)
        self.student_id = student_id
        self.major = major

    def introduce(self):
        # Extend the parent's method
        base = super().introduce()
        return f"{base} I'm a {self.major} major."

    def study(self):
        return f"{self.name} is studying."

s = Student("Alice", 20, "S12345", "Computer Science")
print(s.introduce())
# Hi, I'm Alice, 20 years old. I'm a Computer Science major.
print(s.study())           # Alice is studying.
print(isinstance(s, Person))  # True
```

### Example 2: Method overriding

```python
class Vehicle:
    def __init__(self, brand, model, year):
        self.brand = brand
        self.model = model
        self.year = year

    def start(self):
        return "Vehicle engine started."

    def info(self):
        return f"{self.year} {self.brand} {self.model}"

class Car(Vehicle):
    def __init__(self, brand, model, year, doors):
        super().__init__(brand, model, year)
        self.doors = doors

    def start(self):
        return "Car engine purrs to life. Vroom!"

    def honk(self):
        return "Beep beep!"

class Motorcycle(Vehicle):
    def __init__(self, brand, model, year, has_sidecar=False):
        super().__init__(brand, model, year)
        self.has_sidecar = has_sidecar

    def start(self):
        return "Motorcycle engine revs. Vroom vroom!"

    def wheelie(self):
        return f"{self.brand} {self.model} pops a wheelie!"

car = Car("Toyota", "Camry", 2022, 4)
bike = Motorcycle("Harley", "Sportster", 2021)

print(car.start())    # Car engine purrs to life. Vroom!
print(bike.start())   # Motorcycle engine revs. Vroom vroom!
print(car.info())     # 2022 Toyota Camry
```

### Example 3: Multiple inheritance

```python
class Flyer:
    def fly(self):
        return "Flying through the air!"

    def move(self):
        return "Flyer moves through air"

class Swimmer:
    def swim(self):
        return "Swimming through water!"

    def move(self):
        return "Swimmer moves through water"

class Duck(Flyer, Swimmer):
    def quack(self):
        return "Quack!"

duck = Duck()
print(duck.fly())      # Flying through the air!
print(duck.swim())     # Swimming through water!
print(duck.quack())    # Quack!
print(duck.move())     # Flyer moves through air (Flyer first in MRO)
print(Duck.__mro__)
# Duck -> Flyer -> Swimmer -> object
```

### Example 4: Overriding `__init__` with different signatures

```python
class LibraryItem:
    def __init__(self, title, item_id):
        self.title = title
        self.item_id = item_id
        self.checked_out = False

class Book(LibraryItem):
    def __init__(self, title, item_id, author, pages):
        super().__init__(title, item_id)
        self.author = author
        self.pages = pages

class DVD(LibraryItem):
    def __init__(self, title, item_id, director, duration):
        super().__init__(title, item_id)
        self.director = director
        self.duration = duration

book = Book("1984", "B001", "George Orwell", 328)
dvd = DVD("Inception", "D001", "Christopher Nolan", 148)

print(book.title)     # 1984
print(dvd.director)   # Christopher Nolan
```

### Example 5: Using `super()` in multiple inheritance (cooperative MRO)

```python
class Base:
    def __init__(self, **kwargs):
        print(f"Base.__init__: {kwargs}")
        super().__init__(**kwargs)

class A(Base):
    def __init__(self, a, **kwargs):
        print(f"A.__init__: a={a}, kwargs={kwargs}")
        super().__init__(**kwargs)
        self.a = a

class B(Base):
    def __init__(self, b, **kwargs):
        print(f"B.__init__: b={b}, kwargs={kwargs}")
        super().__init__(**kwargs)
        self.b = b

class C(A, B):
    def __init__(self, c, **kwargs):
        print(f"C.__init__: c={c}, kwargs={kwargs}")
        super().__init__(**kwargs)
        self.c = c

obj = C(a=1, b=2, c=3)
# Output:
# C.__init__: c=3, kwargs={'a': 1, 'b': 2}
# A.__init__: a=1, kwargs={'b': 2}
# B.__init__: b=2, kwargs={}
# Base.__init__: kwargs={}
```

## Common mistakes

1. **Forgetting to call `super().__init__()`** — parent initialization is skipped, leaving attributes unset
2. **Circular inheritance** — a class cannot inherit from itself or create cycles
3. **Misunderstanding MRO with multiple inheritance** — method resolution follows C3 linearization; check `__mro__` if unsure
4. **Overusing inheritance** — prefer composition over inheritance (a Car *has a* Engine, not *is a* Engine)
5. **Deep inheritance hierarchies** — more than 2–3 levels becomes hard to maintain
6. **Not using `super()` in cooperative multiple inheritance** — passing arguments incorrectly can break other classes in the MRO chain

## Interview questions + answers

**Q1: What is inheritance?**
A: Inheritance allows a class to derive properties and methods from a parent class, enabling code reuse and establishing "is-a" relationships.

**Q2: What is the `super()` function?**
A: `super()` returns a proxy object that delegates method calls to the next class in the MRO chain, allowing child classes to call parent methods.

**Q3: What is MRO and how do you view it?**
A: Method Resolution Order defines the order in which Python searches for methods in a class hierarchy. View it with `ClassName.__mro__` or `ClassName.mro()`.

**Q4: What is method overriding?**
A: Redefining a method in a child class that already exists in the parent class, giving it new behavior specific to the child.

**Q5: What's the difference between `isinstance()` and `issubclass()`?**
A: `isinstance(obj, Class)` checks if an object is an instance of a class (or its subclass). `issubclass(Sub, Parent)` checks if one class inherits from another.

## Revision notes (bullet points summary)

- `class Child(Parent):` — syntax for single inheritance
- `super().__init__()` — calls parent constructor
- MRO determines method lookup order; check with `__mro__`
- Overriding: child redefines a parent method with the same name
- Extending: child calls parent method via `super()` and adds behavior
- Multiple inheritance: `class C(A, B):` — use cautiously
- `isinstance(obj, Class)` and `issubclass(Sub, Parent)` for type checking
- All classes inherit from `object`
- Prefer composition over inheritance when there's no true "is-a" relationship
- Use `**kwargs` pattern for cooperative multiple inheritance
