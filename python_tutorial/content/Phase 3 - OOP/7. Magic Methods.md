# 7. Magic Methods

## Definition

**Magic methods** (also called **dunder methods** — short for "double underscore") are special methods in Python that begin and end with double underscores (`__method__`). They allow you to define how objects behave with built-in operations like `print()`, `len()`, arithmetic, comparison, indexing, and more.

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

v1 = Vector(1, 2)
v2 = Vector(3, 4)
print(v1 + v2)  # Vector(4, 6)
```

## Why it exists

Magic methods exist to:
- **Integrate with Python's built-in functions** — make objects work naturally with `len()`, `str()`, `repr()`, etc.
- **Enable operator overloading** — define how `+`, `-`, `*`, `==`, etc. work with your objects
- **Implement collection protocols** — support indexing, iteration, `in` checks
- **Make objects callable** — allow instances to be used as functions
- **Provide string representations** — for debugging and user display

## How it works

Python internally translates operators and built-in functions into magic method calls:

| Expression | Magic Method Called |
|------------|-------------------|
| `str(obj)` | `obj.__str__()` |
| `repr(obj)` | `obj.__repr__()` |
| `len(obj)` | `obj.__len__()` |
| `obj + other` | `obj.__add__(other)` |
| `obj - other` | `obj.__sub__(other)` |
| `obj * other` | `obj.__mul__(other)` |
| `obj == other` | `obj.__eq__(other)` |
| `obj < other` | `obj.__lt__(other)` |
| `obj[key]` | `obj.__getitem__(key)` |
| `obj[key] = val` | `obj.__setitem__(key, val)` |
| `obj()` | `obj.__call__(...)` |

```python
# These are equivalent:
print(len([1, 2, 3]))
print([1, 2, 3].__len__())

# These are equivalent:
print(5 + 3)
print((5).__add__(3))
```

## Internal working

When Python encounters an operator, it looks up the corresponding magic method on the left operand's class (with the right operand as argument), then on the right operand's class (reflected operators like `__radd__`):

```python
# a + b
# 1. tries a.__add__(b)
# 2. if that returns NotImplemented, tries b.__radd__(a)
# 3. if both return NotImplemented, raises TypeError

class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __add__(self, other):
        if isinstance(other, Point):
            return Point(self.x + other.x, self.y + other.y)
        return NotImplemented  # Let Python try other.__radd__

    def __radd__(self, other):
        # Called when other.__add__(self) returned NotImplemented
        return Point(self.x + other, self.y + other) if isinstance(other, (int, float)) else NotImplemented

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

p = Point(1, 2)
print(p + p)      # Point(2, 4)  — p.__add__(p)
print(10 + p)     # Point(11, 12) — p.__radd__(10)
```

## Real-world analogy

Think of magic methods as adapters that make your custom objects compatible with Python's universal language. Just as an electrical adapter lets a foreign plug work in any socket, magic methods let your objects work with Python's built-in operators and functions. Without `__len__`, `len(your_obj)` fails; without `__str__`, `print(your_obj)` shows `<__main__.MyObj object at 0x...>`.

## Examples (with code)

### Example 1: `__init__`, `__str__`, `__repr__`, `__len__`

```python
class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages

    def __str__(self):
        # User-friendly — used by print() and str()
        return f"'{self.title}' by {self.author}"

    def __repr__(self):
        # Developer-friendly — used by repr() and in REPL
        return f"Book('{self.title}', '{self.author}', {self.pages})"

    def __len__(self):
        return self.pages

b = Book("1984", "George Orwell", 328)
print(str(b))       # '1984' by George Orwell
print(repr(b))      # Book('1984', 'George Orwell', 328)
print(len(b))       # 328

# In REPL, just typing b shows repr:
# >>> b
# Book('1984', 'George Orwell', 328)
```

### Example 2: Arithmetic operators — `__add__`, `__sub__`, `__mul__`

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)

    def __mul__(self, scalar):
        if isinstance(scalar, (int, float)):
            return Vector(self.x * scalar, self.y * scalar)
        return NotImplemented

    def __rmul__(self, scalar):
        return self.__mul__(scalar)

    def __neg__(self):
        return Vector(-self.x, -self.y)

v1 = Vector(1, 2)
v2 = Vector(3, 4)

print(v1 + v2)    # Vector(4, 6)
print(v2 - v1)    # Vector(2, 2)
print(v1 * 3)     # Vector(3, 6)
print(3 * v1)     # Vector(3, 6) — via __rmul__
print(-v1)        # Vector(-1, -2)
```

### Example 3: Comparison operators — `__eq__`, `__lt__`, `__gt__`

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __repr__(self):
        return f"Person('{self.name}', {self.age})"

    def __eq__(self, other):
        """Equality: == and != (Python auto-inverts !=)"""
        if not isinstance(other, Person):
            return NotImplemented
        return self.name == other.name and self.age == other.age

    def __lt__(self, other):
        """Less than: <"""
        if not isinstance(other, Person):
            return NotImplemented
        return self.age < other.age

    def __gt__(self, other):
        """Greater than: >"""
        if not isinstance(other, Person):
            return NotImplemented
        return self.age > other.age

    # __le__ (<=), __ge__ (>=) can also be defined

p1 = Person("Alice", 30)
p2 = Person("Bob", 25)
p3 = Person("Alice", 30)

print(p1 == p3)       # True (same name and age)
print(p1 == p2)       # False
print(p1 != p2)       # True (auto from __eq__)
print(p1 < p2)        # False (30 is not < 25)
print(p2 < p1)        # True
```

### Example 4: Indexing — `__getitem__`, `__setitem__`

```python
class CustomList:
    def __init__(self, *items):
        self._items = list(items)

    def __getitem__(self, index):
        return self._items[index]

    def __setitem__(self, index, value):
        self._items[index] = value

    def __len__(self):
        return len(self._items)

    def __repr__(self):
        return f"CustomList({self._items})"

    def __contains__(self, item):
        return item in self._items

cl = CustomList(10, 20, 30, 40)
print(cl[0])          # 10 — __getitem__
print(cl[1:3])        # [20, 30] — slicing works automatically
cl[2] = 99            # __setitem__
print(cl)             # CustomList([10, 20, 99, 40])
print(20 in cl)       # True — __contains__
print(len(cl))        # 4
```

### Example 5: `__call__` — making objects callable

```python
class Multiplier:
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, x):
        return x * self.factor

    def __repr__(self):
        return f"Multiplier({self.factor})"

double = Multiplier(2)
triple = Multiplier(3)

print(double(5))      # 10 — calls __call__(5)
print(triple(10))     # 30

# Useful for decorators, partial functions, callbacks
print(callable(double))  # True
```

### Example 6: Full-featured Vector class

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    # String representations
    def __str__(self):
        return f"({self.x}, {self.y})"

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

    # Arithmetic
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)

    def __mul__(self, other):
        if isinstance(other, (int, float)):
            return Vector(self.x * other, self.y * other)
        if isinstance(other, Vector):
            return Vector(self.x * other.x, self.y * other.y)
        return NotImplemented

    def __neg__(self):
        return Vector(-self.x, -self.y)

    # Comparison
    def __eq__(self, other):
        return isinstance(other, Vector) and self.x == other.x and self.y == other.y

    # Absolute value (magnitude)
    def __abs__(self):
        return (self.x ** 2 + self.y ** 2) ** 0.5

    # Boolean — zero vector is False
    def __bool__(self):
        return self.x != 0 or self.y != 0

    # Support iteration: Vector(1, 2) -> (1, 2)
    def __iter__(self):
        return iter((self.x, self.y))

    # Support indexing: v[0] -> x, v[1] -> y
    def __getitem__(self, index):
        if index == 0:
            return self.x
        if index == 1:
            return self.y
        raise IndexError("Vector index out of range")

v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(repr(v1))        # Vector(3, 4)
print(str(v1))         # (3, 4)
print(v1 + v2)         # Vector(4, 6)
print(v1 - v2)         # Vector(2, 2)
print(v1 * 2)          # Vector(6, 8)
print(v1 == Vector(3, 4))  # True
print(abs(v1))         # 5.0
print(bool(Vector(0, 0)))  # False
print(list(v1))        # [3, 4]
print(v1[0])           # 3
```

### Example 7: Context manager — `__enter__`, `__exit__`

```python
class ManagedFile:
    def __init__(self, filename, mode="r"):
        self.filename = filename
        self.mode = mode

    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.file.close()
        # Return False to propagate exceptions, True to suppress
        return False

# Usage
with ManagedFile("test.txt", "w") as f:
    f.write("Hello, World!")
```

## Common mistakes

1. **Defining `__str__` but not `__repr__`** — `__repr__` is the fallback for `__str__`. Always define at least `__repr__` for debugging.
2. **Returning the wrong type from magic methods** — e.g., `__add__` should return a new instance, not modify `self` and return `None`
3. **Forgetting `return NotImplemented`** (not `raise NotImplementedError`) — return `NotImplemented` to let Python try the reflected operator on the other operand
4. **Mutable default arguments in `__init__`** — creates shared state across instances
5. **Modifying `self` in `__add__`** — arithmetic operators should return new objects, not mutate the originals
6. **Not implementing `__eq__` when implementing `__hash__`** — if you define one, you should define both; if `__eq__` is defined without `__hash__`, the class becomes unhashable
7. **Confusing `__bool__` with `__len__`** — if `__bool__` is not defined, Python falls back to `__len__` (empty = False)

## Interview questions + answers

**Q1: What are magic/dunder methods?**
A: Special methods with double underscores (`__method__`) that define how objects behave with Python's built-in operations like `+`, `len()`, `str()`, indexing, etc.

**Q2: What's the difference between `__str__` and `__repr__`?**
A: `__str__` returns a user-friendly string (for `print()` and `str()`). `__repr__` returns a developer-friendly string (for debugging, REPL). `__repr__` should ideally be unambiguous and, if possible, valid Python code to recreate the object.

**Q3: What does `NotImplemented` do?**
A: It's a special singleton returned by magic methods to indicate that the operation is not implemented for the given operand types. Python then tries the reflected operation (e.g., `__radd__`) or raises `TypeError`.

**Q4: What does `__call__` do?**
A: It makes an object callable like a function. Instances of a class with `__call__` can be used with the `obj()` syntax.

**Q5: How do you make an object support the `with` statement?**
A: Implement `__enter__` (returns the context resource) and `__exit__` (cleans up) methods.

## Revision notes (bullet points summary)

- Dunder methods customize behavior for Python's built-in operations
- `__init__`: initializes object after creation
- `__str__`: user-friendly string (via `print()`, `str()`)
- `__repr__`: developer-friendly string (via `repr()`, REPL)
- `__len__`: length support (for `len()`)
- `__add__`, `__sub__`, `__mul__`: arithmetic operators
- `__eq__`, `__lt__`, `__gt__`: comparison operators
- `__getitem__`, `__setitem__`: indexing support
- `__call__`: makes objects callable
- `__enter__`, `__exit__`: context manager support (with statement)
- Always return new objects from arithmetic operators (immutable pattern)
- Return `NotImplemented` (not raise) for unsupported operand types
- Define `__repr__` on every class for better debugging
