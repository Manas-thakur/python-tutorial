# 2. Constructors

## Definition

A **constructor** in Python is the mechanism that creates and initializes objects. Python uses a two-step process: `__new__()` allocates memory (true constructor) and `__init__()` initializes the object's attributes (initializer). When people say "constructor" in Python, they usually mean `__init__`.

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

## Why it exists

Constructors exist to ensure objects start in a valid, usable state. Without `__init__`, you'd have to manually set every attribute after creation, which is error-prone and verbose:

```python
# Without constructor — tedious and error-prone
s = Student()
s.name = "Alice"  # Might forget this
s.age = 20        # Might forget this
```

## How it works

`__init__` is automatically called when you create an object. It receives the instance as `self` plus any arguments you pass to the class name.

```python
class Student:
    def __init__(self, name, age, grade=None):
        self.name = name
        self.age = age
        self.grade = grade   # Default value
        self.enrolled = True # Always set regardless of args

s1 = Student("Alice", 20, "A")
s2 = Student("Bob", 22)      # grade defaults to None

print(s1.name, s1.grade)     # Alice A
print(s2.name, s2.grade)     # Bob None
print(s1.enrolled)           # True
```

## Internal working

### `__new__` vs `__init__`

| Step | Method | Purpose |
|------|--------|---------|
| 1 | `__new__(cls, ...)` | Allocates memory, returns a new instance |
| 2 | `__init__(self, ...)` | Initializes the instance's attributes |

```python
class Demo:
    def __new__(cls, *args, **kwargs):
        print("1. __new__ called")
        instance = super().__new__(cls)
        return instance

    def __init__(self, value):
        print("2. __init__ called")
        self.value = value

d = Demo(42)
# Output:
# 1. __new__ called
# 2. __init__ called
```

**Important**: `__new__` is rarely overridden — it's mostly used in singleton patterns or when subclassing immutable types (like `int`, `str`).

### What happens internally

When you run `s = Student("Alice", 20)`:

1. Python calls `Student.__new__(Student, "Alice", 20)` → creates a bare object
2. If the returned object is an instance of `Student`, Python calls `Student.__init__(instance, "Alice", 20)`
3. `__init__` sets attributes on the instance
4. The instance is returned and assigned to `s`

## Real-world analogy

Think of `__new__` as ordering an empty coffee cup (the container), and `__init__` as filling it with the specific coffee and extras the customer ordered. The cup is always made the same way; what goes inside varies by order.

## Examples (with code)

### Example 1: Default values in `__init__`

```python
class Book:
    def __init__(self, title, author, pages=0, is_available=True):
        self.title = title
        self.author = author
        self.pages = pages
        self.is_available = is_available

b1 = Book("1984", "George Orwell", 328)
b2 = Book("Dracula", "Bram Stoker")  # pages=0, is_available=True
b3 = Book("Moby Dick", "Herman Melville", 635, False)

print(b2.pages)         # 0
print(b3.is_available)  # False
```

### Example 2: Multiple constructors via `@classmethod`

Python doesn't support method overloading, but you can create alternative constructors using `@classmethod`:

```python
class Employee:
    def __init__(self, name, emp_id, department):
        self.name = name
        self.emp_id = emp_id
        self.department = department

    @classmethod
    def from_string(cls, data_string):
        """Alternative constructor that parses a CSV string."""
        name, emp_id, department = data_string.split(",")
        return cls(name.strip(), int(emp_id.strip()), department.strip())

    @classmethod
    def from_email(cls, email):
        """Alternative constructor that extracts name from email."""
        name = email.split("@")[0].replace(".", " ").title()
        return cls(name, None, "Unassigned")

    def __repr__(self):
        return f"Employee({self.name}, {self.emp_id}, {self.department})"

e1 = Employee("Alice Smith", 101, "Engineering")
e2 = Employee.from_string("Bob Jones, 102, Marketing")
e3 = Employee.from_email("charlie.doe@company.com")

print(e1)  # Employee(Alice Smith, 101, Engineering)
print(e2)  # Employee(Bob Jones, 102, Marketing)
print(e3)  # Employee(Charlie Doe, None, Unassigned)
```

### Example 3: `__init__` with validation

```python
class BankAccount:
    def __init__(self, account_holder, initial_balance=0):
        if not account_holder.strip():
            raise ValueError("Account holder name cannot be empty")
        if initial_balance < 0:
            raise ValueError("Initial balance cannot be negative")

        self.account_holder = account_holder
        self.balance = initial_balance
        self.account_number = id(self)  # Unique identifier

try:
    a = BankAccount("", 100)  # Raises ValueError
except ValueError as e:
    print(e)  # Account holder name cannot be empty
```

### Example 4: Demonstrating `__new__` (singleton pattern)

```python
class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self):
        print("__init__ called")

s1 = Singleton()
s2 = Singleton()
print(s1 is s2)  # True (same instance)
```

### Example 5: Using `*args` and `**kwargs` in `__init__`

```python
class Config:
    def __init__(self, **kwargs):
        for key, value in kwargs.items():
            setattr(self, key, value)

    def __repr__(self):
        items = ", ".join(f"{k}={v}" for k, v in self.__dict__.items())
        return f"Config({items})"

cfg = Config(host="localhost", port=8080, debug=True)
print(cfg)          # Config(host=localhost, port=8080, debug=True)
print(cfg.host)     # localhost
print(cfg.port)     # 8080
```

## Common mistakes

1. **Returning a value from `__init__`** — `__init__` must return `None`. Returning anything else raises `TypeError`:
   ```python
   class Bad:
       def __init__(self):
           return 42  # TypeError: __init__ should return None
   ```

2. **Confusing `__new__` and `__init__`** — `__new__` creates the object (rarely overridden), `__init__` initializes it

3. **Using mutable default arguments** — they're shared across all instances:
   ```python
   class Bad:
       def __init__(self, items=[]):  # Wrong: shared list
           self.items = items

   class Good:
       def __init__(self, items=None):
           self.items = items if items is not None else []
   ```

4. **Forgetting to call `super().__init__()`** in inheritance hierarchies

5. **Not validating arguments** — always validate if the object state must be consistent

## Interview questions + answers

**Q1: What is `__init__` in Python?**
A: `__init__` is the initializer method that Python calls automatically when creating a new object. It sets up the initial state of the object by assigning values to its attributes.

**Q2: What is the difference between `__new__` and `__init__`?**
A: `__new__` is the actual constructor that creates and returns a new object (memory allocation). `__init__` is the initializer that sets up the object's attributes. `__new__` is called first, then `__init__`. `__new__` is rarely overridden.

**Q3: Can a Python class have multiple constructors?**
A: Not directly (no method overloading), but you can achieve the same effect using default parameter values or `@classmethod` alternative constructors.

**Q4: Can `__init__` return a value?**
A: No. `__init__` must return `None`. Returning a non-None value raises a `TypeError`.

**Q5: What happens if a class doesn't define `__init__`?**
A: Python calls the default `__init__` inherited from `object`, which does nothing. The object is still created but has no custom initialization.

## Revision notes (bullet points summary)

- `__init__` is the initializer, called automatically after object creation
- `self` is the instance being initialized; it's always the first parameter
- You can set default values for `__init__` parameters
- `__new__` is the true constructor — rarely overridden
- Use `@classmethod` for alternative constructors
- Never return a value from `__init__` (must be `None`)
- Avoid mutable default arguments; use `None` instead
- Validate inputs in `__init__` to ensure object consistency
- If no `__init__` defined, Python uses `object.__init__` (no-op)
